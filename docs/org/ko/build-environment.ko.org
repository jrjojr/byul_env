이 문서는 BYUL을 빌드하기 위한 최소 구성 요건과 설치 방법을 정리합니다.
빌드 후 테스트 실행 파일과 =byul.dll=/=libbyul.so=를 실행 파일에 연결하는 방법은
[[file:runtime-execution.ko.org][실행 방법]]을 확인합니다.

현재 권장 작업 환경은 Ubuntu입니다. Linux 빌드와 Windows MinGW 크로스 빌드는
대부분 Ubuntu에서 실행하고, MSVC 빌드는 Windows Visual Studio 환경에서
확인하는 구조입니다.

* 빌드 기준

BYUL은 CMake preset 중심으로 빌드합니다.

#+begin_src text
주 작업 환경: Ubuntu
주 빌드 명령: cmake --preset ... / cmake --build --preset ...
SDK ZIP target: byul_sdk_zip
BYUL Grid ZIP target: byul_grid_zip
GPU Compute Tester ZIP target: gpu_comp_tester_zip (기본 빌드와 분리)
최종 배포물: byul.dll 또는 libbyul.so, public headers
개발 산출물: 모듈 static library, 모듈별 test exe, 통합 test exe
#+end_src

=byul_sdk_zip=은 단독 실행 대상입니다. 먼저 =all= target을 최신 상태로 만든 뒤
임시 package layout에 설치하고
=byul-sdk-<version>-<platform>-<toolchain>-<configuration>.zip=을 생성합니다. BYUL
Grid는 별도 =byul_grid_zip= target으로
=byul-grid-<version>-<platform>-<configuration>.zip=을 생성합니다.
=gpu_comp_tester_zip= target으로 일반 빌드와 별도로
=gpu-comp-tester-<version>-<platform>-<configuration>.zip=을 생성할 수 있습니다.
일반 =all=/=ALL_BUILD=와 =build-*= preset은 ZIP을 만들지 않습니다. ZIP은 위 package
target을 명시적으로 실행할 때만 생성됩니다.

* Ubuntu 최소 요건

Ubuntu에서 Linux 빌드와 Windows MinGW 크로스 빌드를 모두 수행하려면 다음이
필요합니다.

#+begin_src text
필수:
  cmake
  make
  gcc
  g++
  libc 개발 헤더
  zip
  pkg-config

테스트/디버그 권장:
  gdb
  ctest는 cmake 패키지에 포함

Windows MinGW 크로스 빌드:
  gcc-mingw-w64-x86-64
  g++-mingw-w64-x86-64
  mingw-w64-tools

그래픽/SDL 관련 target을 빌드할 경우:
  libsdl2-dev 또는 프로젝트가 요구하는 SDL 패키지
  Vulkan/OpenGL 관련 개발 패키지
#+end_src

Ubuntu 설치 예:

#+begin_src bash
sudo apt update
sudo apt install -y \
  build-essential \
  cmake \
  pkg-config \
  zip \
  gdb \
  gcc-mingw-w64-x86-64 \
  g++-mingw-w64-x86-64 \
  mingw-w64-tools
#+end_src

SDL 또는 GPU 보조 실행 파일을 함께 빌드해야 하면 환경에 맞게 추가 설치합니다.

#+begin_src bash
sudo apt install -y libsdl2-dev libvulkan-dev mesa-common-dev
#+end_src

현재 =win_sdl_release= preset은 =external/SDL3-3.2.18/cmake= 경로를
=CMAKE_PREFIX_PATH=로 사용합니다. 이 preset을 사용하려면 해당 외부 SDL3
패키지가 같은 위치에 준비되어 있어야 합니다.

* Ubuntu Linux 빌드

Debug 빌드:

#+begin_src bash
cd byul_env/byul
cmake --preset linux-debug
cmake --build --preset build-linux-debug
#+end_src

Release 빌드:

#+begin_src bash
cd byul_env/byul
cmake --preset linux-release
cmake --build --preset build-linux-release
#+end_src

빌드 preset은 generator의 기본 =all=/=ALL_BUILD=를 실행합니다. Library와 테스트
실행 파일만 갱신하며 ZIP은 생성하지 않습니다.

* Ubuntu에서 Windows MinGW 크로스 빌드

Windows DLL용 MinGW64 크로스 빌드:

#+begin_src bash
cd byul_env/byul
cmake --preset win-release
cmake --build --preset build-win-release
#+end_src

이 preset은 다음 toolchain을 사용합니다.

#+begin_src text
byul/cmake/toolchain-mingw64.cmake
#+end_src

현재 toolchain은 다음 컴파일러 이름을 기대합니다.

#+begin_src text
x86_64-w64-mingw32-gcc
x86_64-w64-mingw32-g++
x86_64-w64-mingw32-windres
#+end_src

즉 Ubuntu에서는 MinGW 크로스 컴파일러가 PATH에서 보여야 합니다.

* Windows native MinGW 빌드

다음 preset은 Ubuntu용이 아니라 Windows MSYS2 MinGW shell에서 실행하는
구성입니다.

#+begin_src text
win-native
win-native-debug
#+end_src

Windows MSYS2에서 필요한 패키지 예:

#+begin_src bash
pacman -S --needed \
  mingw-w64-x86_64-cmake \
  mingw-w64-x86_64-gcc \
  mingw-w64-x86_64-make \
  mingw-w64-x86_64-pkgconf \
  mingw-w64-x86_64-zip
#+end_src

실행 예:

#+begin_src bash
cd /path/to/byul_env/byul
cmake --preset win-native
cmake --build --preset build-win-native
#+end_src

* MSVC 빌드 분석

현재 preset은 다음과 같습니다.

#+begin_src text
win-msvc
#+end_src

이 preset은 Visual Studio 2022 generator를 사용하고, =cmakeExecutable=도 Windows
경로를 가리킵니다.

#+begin_src text
Visual Studio 17 2022
C:/Program Files/Microsoft Visual Studio/2022/Community/.../cmake.exe
#+end_src

따라서 현재 구성 그대로는 Ubuntu에서 직접 실행할 수 없습니다. Ubuntu에는
Visual Studio generator와 MSVC compiler toolchain이 없기 때문입니다.

Ubuntu에서 가능한 선택지는 다음입니다.

#+begin_src text
가능:
  MinGW64 크로스 컴파일로 Windows DLL/EXE 생성
  Windows VM, Windows 머신, GitHub Actions Windows runner에서 MSVC preset 실행
  원격 Windows 빌드 서버에서 MSVC preset 실행

현재 비권장/미구성:
  Wine 위에서 Visual Studio CMake를 실행하는 방식
  Ubuntu clang-cl로 MSVC ABI를 흉내 내는 방식
#+end_src

MSVC 산출물을 정식으로 보장하려면 Windows 환경 또는 Windows CI가 필요합니다.
Ubuntu에서 =win-msvc-*= preset을 직접 돌리는 것은 현재 목표가 아닙니다.

* 테스트 실행

CTest 자체는 build하지 않습니다. 먼저 해당 build preset 또는 =all=/=ALL_BUILD=를
실행한 뒤 CTest를 실행합니다.

Linux Debug:

#+begin_src bash
ctest --test-dir ../build_debug --output-on-failure
#+end_src

Linux Release:

#+begin_src bash
ctest --test-dir ../build_release --output-on-failure
#+end_src

Windows MinGW 크로스 빌드 산출물은 Ubuntu에서 바로 실행하지 않습니다. Wine을
쓰면 일부 실행은 가능할 수 있지만, 현재 공식 검증 경로로 문서화하지 않습니다.

MSVC multi-config 빌드는 Windows에서 configuration을 지정합니다.

#+begin_src bash
ctest --preset test-win-msvc-release
ctest --preset test-win-msvc-debug
#+end_src

* 주요 target

#+begin_src text
all                 library, 보조 실행 파일, 테스트 실행 파일 빌드
byul                최종 shared library target만 빌드
test_core           core 모듈 테스트 실행 파일 빌드

test                등록된 CTest 테스트 실행

byul_sdk_zip         all 갱신, package layout 설치, byul-sdk ZIP 생성
byul_grid_zip        현재 byul shared library로 portable BYUL Grid ZIP 생성

byul_help           configure/build/test/install/package CMake CLI 도움말 출력
#+end_src

* CMake CLI 도움말

Preset 목록과 BYUL 전용 도움말은 다음 명령으로 확인합니다.

#+begin_src bash
cmake --list-presets
cmake --build --list-presets
ctest --list-presets
cmake --build <build-directory> --target byul_help --config Release
#+end_src

=byul_help=는 configure, 일반 build, test, install, clean과 SDK/Grid/GPU package
명령을 출력합니다. =--config=는 Visual Studio 같은 multi-config generator에서
사용하고 single-config generator에서는 configure preset의 build type을 따릅니다.
