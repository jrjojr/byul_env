#+TITLE: BYUL Public C Header 코멘트 규칙
#+LANGUAGE: ko
#+OPTIONS: toc:3 num:t

* 문서 목적

이 문서는 BYUL public C header에서 선언만으로 표현할 수 없는 정보를 일관된
Doxygen comment로 작성하는 규칙을 정의한다.

선언과 명명 규칙은 [[file:header-declaration-rules.org][BYUL Public C Header 선언 규칙]]이 우선한다.
Wrapper 생성은 [[file:wrapper-rules.org][BYUL Python Wrapper 자동 생성 규칙]]을 따른다.

Comment는 선언을 대신하지 않는다. 메모리 할당 여부를 일반 설명문에서 추측하게
만들지 않는다. Create/copy/get/take/release 같은 핵심 의미는 함수 이름과 선언에
먼저 반영한다.

* 파일 수준 Doxygen 문서

[[file:header-declaration-rules.org][선언 규칙]]에 따라 라이선스 블록 바로 아래에 파일 수준 Doxygen 문서를 둔다.
라이선스 블록은 일반 comment이며 parser가 건너뛴다.

필수 항목은 다음과 같다.

- =@file=: 확장자를 포함한 실제 파일명
- =@brief=: header의 핵심 책임을 설명하는 영어 한 문장
- 상세 설명: 대표 type과 외부 제공 기능을 설명하는 영어 1개 이상의 문장

파일 수준 문서는 header 전체의 책임만 설명하며 API symbol로 변환하지 않는다.

* Comment가 제공해야 하는 추가 정보

선언만으로 알 수 없는 다음 정보를 comment로 제공한다.

- 함수 목적과 중요한 의미
- NULL 허용 여부
- NULL 반환 의미
- 숫자 범위와 단위
- 실패 조건과 status 의미
- precondition과 postcondition
- side effect
- thread safety
- callback 호출 시점과 thread
- callback 재진입 가능 여부
- pointer 유효 기간
- buffer 길이 관계
- 문자열 encoding
- deprecation과 replacement
- 성능상 중요한 복잡도

* 기본 형식

Public type과 함수는 Doxygen block comment를 사용한다.

#+begin_src c
/**
 * @brief Creates a coordinate with the specified values.
 *
 * @param[in] x X coordinate in grid cells.
 * @param[in] y Y coordinate in grid cells.
 * @return Newly allocated coordinate owned by the caller.
 * @retval NULL Allocation failed.
 *
 * @byul.nullable return true
 * @byul.thread_safety thread-safe
 */
BYUL_API coord_t* coord_create_full(int x, int y);
#+end_src

=@brief=와 일반 설명은 사람을 위한 문장이다. =@byul.*= tag는 선언만으로 알 수
없는 정보를 generator와 linter가 읽기 위한 구조화 metadata다.

* 언어와 문장 규칙

- Public header comment의 기준 언어는 영어다.
- 첫 문장은 동사로 시작하는 간결한 현재형을 사용한다.
- 함수명을 그대로 반복하는 문장만 작성하지 않는다.
- “maybe”, “usually”, “somehow” 같은 모호한 표현을 피한다.
- MUST/does/does not으로 실제 계약을 명확히 쓴다.
- 구현 세부보다 caller가 알아야 할 계약을 설명한다.

* Type comment

** Opaque object

#+begin_src c
/**
 * @brief Dynamic list of coordinates.
 *
 * Instances are created by coord_list_create() and destroyed by
 * coord_list_destroy().
 *
 * @byul.thread_safety not-thread-safe
 */
typedef struct s_coord_list coord_list_t;
#+end_src

Type comment에는 다음을 설명한다.

- 개념과 역할
- 생성/해제 함수
- thread safety 기본값
- 중요한 불변 조건

** Public value struct

#+begin_src c
/**
 * @brief Integer coordinate in a two-dimensional grid.
 *
 * @byul.value_semantics true
 */
typedef struct s_coord {
    int x; /**< X coordinate in grid cells. */
    int y; /**< Y coordinate in grid cells. */
} coord_t;
#+end_src

각 public field에는 단위와 의미를 작성한다.

* 함수 comment 필수 항목

모든 public 함수는 최소한 다음을 포함한다.

- =@brief=
- 모든 parameter의 =@param=
- non-void 반환의 =@return=
- 실패 반환이 있으면 =@retval= 또는 =@byul.error=

필요에 따라 다음을 추가한다.

- =@pre=
- =@post=
- =@note=
- =@warning=
- =@deprecated=
- =@byul.*=

* Parameter 방향

Doxygen 방향 표기를 반드시 사용한다.

#+begin_example
@param[in]     src
@param[out]    out_value
@param[in,out] object
#+end_example

Pointer라고 모두 out이 아니다. =const T*=는 일반적으로 =[in]=이다.

#+begin_src c
/**
 * @param[in] coord Coordinate to inspect. Must not be NULL.
 * @param[out] out_x Receives the X coordinate. Must not be NULL.
 * @param[out] out_y Receives the Y coordinate. Must not be NULL.
 */
BYUL_API void coord_fetch(
    const coord_t* coord,
    int* out_x,
    int* out_y);
#+end_src

* NULL 규칙

** Parameter NULL

구조화 tag:

#+begin_example
@byul.nullable <parameter-name> true|false
#+end_example

예:

#+begin_example
@byul.nullable coord false
@byul.nullable userdata true
#+end_example

MUST NOT be NULL이라는 일반 문장만으로 generator가 판단하게 하지 않는다.

** 반환 NULL

#+begin_example
@byul.nullable return true
@retval NULL No item exists at the requested index.
#+end_example

NULL이 실패인지 정상적인 “없음”인지 =@retval NULL=에 명확히 쓴다.

예:

#+begin_src c
/**
 * @brief Returns the coordinate stored at index without allocating a copy.
 *
 * @param[in] list List to inspect.
 * @param[in] index Zero-based index.
 * @return Internal coordinate owned by list.
 * @retval NULL index is outside [0, coord_list_length(list)).
 *
 * @byul.nullable list false
 * @byul.nullable return true
 * @byul.lifetime return parent:list
 */
BYUL_API const coord_t* coord_list_get(
    const coord_list_t* list,
    int index);
#+end_src

* Lifetime tag

선언 이름으로 기본 소유권을 판단한 뒤, 정확한 유효 기간을 comment로 보완한다.

형식:

#+begin_example
@byul.lifetime <target> <policy>
#+end_example

Policy:

#+begin_example
caller-owned
static
call-only
parent:<parameter-name>
until-next-call:<function-name>
until-unbind
until-destroy:<parameter-name>
transferred-to:<parameter-name>
#+end_example

예:

#+begin_example
@byul.lifetime return caller-owned
@byul.lifetime return parent:list
@byul.lifetime callback until-unbind
#+end_example

Lifetime tag가 함수 이름의 create/get/take/release 의미와 충돌하면 lint 오류다.

** Pointer invalidation

Container 또는 object mutation이 기존 내부 pointer를 무효화하면 반드시 구조화
tag로 명시한다.

#+begin_example
@byul.invalidates <operation> <target>
#+end_example

Target vocabulary:

#+begin_example
all-internal-pointers
element-pointers
iterators
views
callback-handles
#+end_example

예:

#+begin_example
@byul.invalidates push_back element-pointers
@byul.invalidates clear all-internal-pointers
#+end_example

기존 pointer가 유지되면 명시할 수 있다.

#+begin_example
@byul.preserves push_back element-pointers
#+end_example

같은 operation과 target에 invalidates/preserves를 동시에 쓰면 lint 오류다.

* Type storage와 copy metadata

** Storage kind

선언 구조로 판정한 type 분류를 comment metadata로 확인한다.

#+begin_example
@byul.storage basic-value
@byul.storage resource-value
@byul.storage opaque-object
@byul.storage view
#+end_example

Comment tag가 실제 struct 선언과 충돌하면 lint 오류다.

** Zero initialization

#+begin_example
@byul.zero_valid true|false
#+end_example

=false=이면 유효한 object가 되기 위해 호출해야 할 init 함수를 일반 설명과 함께
기록한다.

#+begin_example
@byul.zero_valid false
@byul.requires_init type_init
#+end_example

** Copy semantics

#+begin_example
@byul.copy_semantics trivial-copy
@byul.copy_semantics deep-copy
@byul.copy_semantics non-copyable
#+end_example

Deep copy type은 assign 또는 copy 함수가 선언되어 있어야 한다. Non-copyable type에
copy/assign 함수가 있으면 lint 오류다.

* Memory 크기 comment

=<type>_sizeof()=는 object 본체 크기만 반환한다는 전역 선언 규칙을 따른다.

=<type>_get_memory_size()=에는 포함 범위를 명시한다.

#+begin_example
@byul.memory includes object-storage
@byul.memory includes owned-buffers
@byul.memory excludes allocator-overhead
@byul.memory excludes shared-objects
@byul.memory exact|estimate
#+end_example

예:

#+begin_src c
/**
 * @brief Returns the total native memory directly owned by list.
 *
 * The result includes the coord_list_t object and its element buffer. It does
 * not include allocator bookkeeping overhead.
 *
 * @param[in] list List to inspect.
 * @return Owned native memory size in bytes.
 *
 * @byul.unit return bytes
 * @byul.memory includes object-storage
 * @byul.memory includes owned-buffers
 * @byul.memory excludes allocator-overhead
 * @byul.memory exact
 */
BYUL_API size_t coord_list_get_memory_size(
    const coord_list_t* list);
#+end_src

* Error 규칙

** 구조화 error tag

#+begin_example
@byul.error none
@byul.error null-return
@byul.error false-return
@byul.error negative-status
@byul.error nonzero-status
@byul.error enum:<error-enum-type>
#+end_example

각 오류 code의 의미는 =@retval=로 작성한다.

#+begin_src c
/**
 * @return Status code.
 * @retval 0 Success.
 * @retval -1 object is NULL.
 * @retval -2 capacity is negative.
 *
 * @byul.error negative-status
 */
#+end_src

** errno와 global error

Public ABI가 =errno= 또는 global last-error를 사용한다면 type/module comment에 한 번
명시하고 모든 함수가 같은 규칙을 따라야 한다. 함수별로 다른 숨은 error channel을
사용하지 않는다.

* 숫자 범위와 단위

** Range

#+begin_example
@byul.range <parameter> <minimum> <maximum>
#+end_example

범위 끝 포함 여부가 필요하면 수학 구간 문법을 사용한다.

#+begin_example
@byul.range angle_deg [0,360)
@byul.range capacity [0,INT_MAX]
#+end_example

** Unit

#+begin_example
@byul.unit <parameter-or-return> <unit>
#+end_example

권장 unit vocabulary:

#+begin_example
seconds
milliseconds
meters
meters-per-second
kilometers-per-hour
radians
degrees
grid-cells
unitless
#+end_example

예:

#+begin_example
@byul.unit interval_sec seconds
@byul.unit return degrees
#+end_example

Parameter 이름에도 가능한 경우 unit suffix를 포함한다.

* Precondition과 postcondition

** Precondition

#+begin_example
@pre object was successfully initialized.
@pre index is non-negative.
#+end_example

Generator가 검사 코드를 생성해야 하는 조건이면 구조화 tag도 추가한다.

#+begin_example
@byul.requires initialized:object
@byul.requires nonnegative:index
#+end_example

** Postcondition

#+begin_example
@post out contains an independent copy of src.
@post list length increases by one on success.
#+end_example

소유권 이동 후 source가 무효가 되는 경우 반드시 쓴다.

* Side effect

형식:

#+begin_example
@byul.side_effect none
@byul.side_effect mutates:<parameter>
@byul.side_effect consumes:<parameter>
@byul.side_effect invokes-callback:<name>
@byul.side_effect blocks
@byul.side_effect writes-stdout
#+end_example

여러 side effect가 있으면 tag를 반복한다.

#+begin_example
@byul.side_effect mutates:list
@byul.side_effect consumes:item
#+end_example

* Thread safety

** 함수 또는 type 기본값

#+begin_example
@byul.thread_safety thread-safe
@byul.thread_safety read-only-thread-safe
@byul.thread_safety externally-synchronized
@byul.thread_safety not-thread-safe
#+end_example

Type comment에 기본값을 선언하고 예외 함수에 별도로 덮어쓴다.

** Blocking

#+begin_example
@byul.blocking false
@byul.blocking true
@byul.blocking conditional
#+end_example

Conditional이면 조건을 일반 설명으로 이어서 쓴다.

* Callback comment

** Callback typedef

#+begin_src c
/**
 * @brief Receives a coordinate selected by the native search loop.
 *
 * @param[in] coord Coordinate valid only for the duration of this call.
 * @param[in] userdata User data supplied when the callback was bound.
 *
 * @byul.nullable coord false
 * @byul.nullable userdata true
 * @byul.lifetime coord call-only
 * @byul.callback_thread caller
 * @byul.reentrant false
 */
typedef void (*move_func)(
    const coord_t* coord,
    void* userdata);
#+end_src

** Callback thread

#+begin_example
@byul.callback_thread caller
@byul.callback_thread worker
@byul.callback_thread event-loop
#+end_example

=caller=는 callback을 발생시킨 native 함수를 호출한 thread를 뜻한다.

** Callback retained 기간

#+begin_example
@byul.lifetime callback call-only
@byul.lifetime callback until-unbind
@byul.lifetime callback until-destroy:object
#+end_example

** Reentrancy

#+begin_example
@byul.reentrant true|false
#+end_example

Callback 내부에서 같은 object API를 다시 호출해도 되는지 설명한다.

** Callback exception

C에는 Python exception 개념이 없지만 wrapper generator에는 실패 반환 규칙이
필요하다.

#+begin_example
@byul.callback_failure return-null
@byul.callback_failure return-false
@byul.callback_failure ignore
@byul.callback_failure stop-operation
#+end_example

* Buffer와 array comment

#+begin_src c
/**
 * @brief Copies route coordinates into a caller-provided buffer.
 *
 * @param[in] route Route to export.
 * @param[out] out_coords Destination array.
 * @param[in] capacity Number of coord_t elements available in out_coords.
 * @return Number of coordinates written.
 * @retval -1 route is NULL.
 * @retval -2 capacity is negative.
 *
 * @byul.buffer out_coords
 * @byul.capacity out_coords capacity
 * @byul.count return out_coords
 * @byul.error negative-status
 */
#+end_src

구조화 tag:

#+begin_example
@byul.buffer <parameter>
@byul.capacity <buffer> <capacity-parameter>
@byul.count return|<out-count-parameter> <buffer>
#+end_example

Byte capacity인지 element capacity인지 parameter 설명과 tag vocabulary로
구분한다.

* 문자열 comment

#+begin_example
@byul.encoding return utf-8
@byul.termination return nul
@byul.lifetime return static
#+end_example

Caller buffer 문자열:

#+begin_example
@byul.encoding buffer utf-8
@byul.capacity buffer buffer_size_bytes
@byul.termination buffer nul
#+end_example

Owned 문자열이면 destroy 함수를 comment가 아니라 선언 naming 규칙으로 찾을 수
있어야 하며 lifetime tag는 이를 확인한다.

* 성능과 복잡도

Caller가 알고리즘 선택에 필요할 때만 작성한다.

#+begin_example
@complexity Average O(1), worst-case O(n).
#+end_example

성능 수치를 근거 없이 작성하지 않는다. 구현 변경으로 보장이 깨지면 comment와
test를 함께 갱신한다.

* Deprecated comment

#+begin_src c
/**
 * @deprecated Use coord_next_to_goal() instead. Removal is planned for ABI 2.
 */
#+end_src

규칙:

- replacement symbol을 명시한다.
- 제거 예정 ABI version을 가능한 경우 명시한다.
- deprecated 함수의 기존 계약도 계속 정확히 설명한다.

** BYUL 구조화 tag

#+begin_example

@byul.error

새 tag를 임의로 추가하지 않는다. 필요한 tag가 없으면 먼저 이 문서를 변경하고

* Comment parser 규칙

Generator는 일반 문장에서 의미를 추론하지 않는다. 구조화 tag만 기계 판독한다.

Parser는 다음을 오류로 처리한다.

- 존재하지 않는 parameter를 tag에서 참조
- 같은 대상에 상충하는 nullable/lifetime 지정
- buffer인데 capacity tag 없음
- callback인데 thread/lifetime 정보 없음
- unit vocabulary에 없는 단위
- 선언 naming과 lifetime tag 충돌
- =@retval= 값과 =@byul.error= 규칙 충돌
- 알 수 없는 =@byul.*= tag

* Comment lint 필수 조건

** 모든 public 함수

- =@brief= 존재
- 모든 parameter의 =@param= 존재
- non-void 반환의 =@return= 존재
- pointer parameter의 nullable 명시
- pointer 반환의 nullable과 lifetime 명시
- 실패 가능 함수의 error 명시

** Callback

- 모든 parameter 방향과 nullable
- callback thread
- retained lifetime
- reentrant 여부
- 실패 처리

** Buffer/string

- capacity/count 관계
- encoding
- null termination
- 반환/출력 lifetime

* 선언과 comment 충돌 조건

다음 충돌은 lint 오류다.

- =create= 반환에 =parent-owned= lifetime 지정
- =get/peek= 반환에 =caller-owned= lifetime 지정
- =const= receiver 함수에 receiver mutation 지정
- 선언의 parameter와 존재하지 않는 comment 대상 참조
