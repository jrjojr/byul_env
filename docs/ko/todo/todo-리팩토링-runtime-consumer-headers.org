* 목적과 실행 위치

Navsys와 Math Engine이 완료된 뒤 BYUL root/config/core, console과 tick의 public
ABI·lifecycle·callback·install/export를 정리한다. 이 문서는 수학 공식, numeric kernel,
Numal, number theory 또는 RNG를 소유하지 않는다.

실행 순서는 다음과 같다.

#+begin_src text
Navsys DONE
-> Math Engine canonical handoff DONE
-> Runtime/Foundation consumer header
-> Simulation consumer header
-> root/wrapper/package integration
#+end_src

Math Engine의 기존·신규 header/API 이름, tolerance, status와 계산 source를 이 문서에서
재설계하거나 복제하지 않는다.

* 기존 Header 6개

| Header | Consumer 책임 |
|--------+---------------|
| =byul/byul.h= | root module entry, umbrella, version/ABI query |
| =byul/byul_config.h= | export/import/static macro와 version encoding |
| =byul/core/strset.h= | opaque resource, callback, allocation/lifecycle |
| =byul/console/console.h= | output side effect와 formatting contract |
| =byul/console/dstar_lite_console.h= | Navsys diagnostic compatibility와 wrapper |
| =byul/byul_tick/byul_tick.h= | tick resource, callback/userdata와 wrapper lifetime |

=byul/aerial/aerial.h=는 projectile/guidance를 소비하는 orchestration이므로
[[file:todo-리팩토링-simulation-consumer-headers.org][Simulation consumer TODO]]로
이동한다.

* 명명과 ownership Gate

6개 initial asset의 =asset_id/current_path/current_owner/disposition/compatibility_path=를
[[file:header-refactor-current/header-role-manifest.json][단일 ownership manifest]]에
기록한다. Rename/split이면 canonical path와 forwarding 종료 version을 갖고, public
basename은 SDK 전체에서 고유해야 한다.

이 문서는 Math Engine header/source의 =canonical_owner=나 =design_todo=가 될 수 없다.
Root umbrella는 승인된 Math Engine public aggregate를 소비할 수 있지만 Math Engine
component header를 BYUL root 소유로 재분류하지 않는다.

* 현재 위험

- =byul.h=의 빈 parameter prototype와 version string lifetime/ABI fingerprint가 불명확하다.
- =byul_config.h=의 generic debug macro와 version/export 분기가 consumer와 충돌할 수 있다.
- =strset_pop=은 library allocation을 반환하지만 대응 destroy API가 없다.
- =strset.h=은 =bool= direct include와 callback/mutation 계약이 불완전하다.
- D* console의 두 선언은 구현이 없는데 wrapper가 호출하며 override가 충돌을 숨긴다.
- Tick callback/userdata retention, unbind, thread/reentrancy와 wrapper close lifetime이
  닫히지 않았다.
