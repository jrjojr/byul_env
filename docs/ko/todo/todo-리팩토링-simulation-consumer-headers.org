* 목적과 실행 위치

Math Engine의 54개 기존 계산 관련 header extraction이 canonical handoff를 마친 뒤
Balix, Entity, Projectile, Ground와 Aerial의 기존 public ABI, DTO/resource lifecycle,
callback/dispatch, storage, legacy forwarding과 install/export를 정리한다.

이 문서는 공식·상수·solver·geometry·physics·control·prediction·sampling kernel을
설계하거나 구현하지 않는다. 모든 계산은 Math Engine public API를 소비한다.

* 단일 ownership Gate

[[file:header-refactor-current/header-role-manifest.json][단일 ownership manifest]]에서 initial
header와 Math Engine successor를 별도 asset으로 기록한다.

- Math Engine: canonical formula/kernel/value/result/header/source/target
- Simulation consumer: 기존 DTO/resource/lifecycle/callback/storage/adapter
- Legacy shim: 기존 include/symbol에서 canonical Math Engine API로 forwarding

한 symbol/source/header에 design owner가 둘 이상이면 진행하지 않는다. Adapter 내부에
공식, solver, tolerance 또는 numeric heuristic를 다시 작성하지 않는다.

* P0 ABI·lifecycle 결함

- MPC init/assign 두 함수의 Windows export hole
- Projectile result =free=/=destroy=의 caller/heap storage 충돌
- Projectile tick assign의 trajectory shallow copy와 누락 state/null cleanup
- Ground table/height copy/free 비대칭과 allocation failure를 숨기는 =void= assign
- Trajectory fresh-init이 uninitialized =samples=를 읽는 문제
- Controller/filter public impl/vtable와 inline layout access
- Environ callback 문서/signature, retention/thread/reentrancy 불일치
- Projectile/Ground의 owning pointer와 callback public layout

이 결함은 수학 추출과 무관한 lifecycle 작업이지만 실행 순서는 Math Engine handoff 뒤로
고정한다. Math Engine API를 임시로 복제해 우회하지 않는다.
