* 대상과 기준

대상은 =greedy_best_first.h/.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]],
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]],
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/AStarVisitor.html][AStar Visitor]]와
[[https://www.boost.org/doc/libs/release/libs/graph/doc/using_property_maps.html][Property Maps]]의
algorithm/visitor/state 분리를 직접 비교한다. 이 algorithm은 =h(n)=만 사용하는 비최적
best-first search임을 capability에 명시하고 A*와 같은 최단 cost 보장을 붙이지 않는다.

  consumer assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 구현/소비 분석

- =find_greedy_best_first= 하나가 module/root/SDK/CFFI/dispatcher에 노출된다.
- Header는 heuristic이 required라고 하지만 구현은 NULL이면 =default_heuristic=을 선택한다.
- Heuristic userdata를 전달할 통로가 없고 항상 NULL이다. NaN/Inf/negative 결과도 검사하지 않는다.
- =max_retry <= 0=이면 loop가 한 번도 돌지 않아 failed/empty route가 되지만 invalid/limit/no-path를
  구분하지 않는다.
- 발견 시 visited mark를 하므로 더 좋은 경로를 다시 고려하지 않는다. 이는 greedy traversal에는
  가능하나 completeness, tie order와 partial endpoint를 계약해야 한다.
