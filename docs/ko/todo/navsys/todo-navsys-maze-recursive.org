* 대상과 명명 판정

- 대상: =maze_recursive.h/.cpp=.
- 실제 algorithm은 generic recursion이 아니라 randomized depth-first backtracker다.
  [[https://weblog.jamisbuck.org/2010/12/27/maze-generation-recursive-backtracking.html][Jamis Buck reference]]를 기준으로 visited neighbor와 backtrack 종료를 검증한다.
- 공통 status/options는 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 사용한다.

* 책임 경계와 실제 검증 파일

- 이 문서는 recursive-backtracker/iterative DFS algorithm state·materialization과 RNG 소비,
  seed/replay, budget/cancel 구현을 소유한다.
- resource/extent/enumeration/overlay primitive는 =maze_core=를 사용하고 재구현하지 않는다.
  enum selection/options routing/capability 합류는 =maze= dispatcher가 소유하며 이 문서는
  direct entry와 검증된 capability record만 제공한다.
- 실제 maze native 검증 파일은 =byul/navsys/maze/tests/test_module_maze.cpp=,
  =byul/navsys/maze/tests/test_module_maze_discrete.cpp=,
  =byul/navsys/maze/tests/test_module_maze_room.cpp=다. Algorithm별 case는 이 공용 세 파일
  중 책임에 맞는 파일에 둔다.

* P0 구현 결함

| 부분 | 현재 코드 | 영향 |
|------+----------+------|
| validation | 없음 | width/height <3, even, negative에서 OOB/huge allocation/invalid topology |
| global RNG | =srand(time)=, =rand()= | process-global state 오염, thread race, 재현 불가 |
| blocked init loops | =for(y=y0; y<height)=, =for(x=x0; x<width)= | origin이 loop bound로 섞여 nonzero origin에서 cell 수가 축소됨 |
| coordinate insert | ={out->x0+x,out->y0+y}= | x/y가 이미 origin에서 시작해 nonzero origin을 이중 가산 |
| visited | =malloc h rows/calloc w= 실패 미검사 | partial allocation cleanup 없음 |
| start | 항상 local =(1,1)= | small/even input OOB 가능 |
| traversal | 모든 in-bounds cell에서 step 2 | parity/border rule이 header의 odd >=3 주장과 구현상 강제되지 않음 |
| recursion | cell 수만큼 call stack | large maze stack overflow, cancellation 없음 |
| result | removal/insert 실패 무시 | incomplete maze도 성공 반환 |

* 목표 구현·계약

- explicit stack 기반 iterative DFS를 기본으로 하여 O(V) heap workspace와 bounded native stack을 보장한다.
- logical odd/odd interior cell과 world origin 변환을 한 함수에서만 수행한다.
- current cell의 unvisited neighbors를 unbiased shuffle/select하고, 없으면 stack pop한다.
- passage graph는 perfect maze이며 DFS 특유 long corridor는 characteristic이지 stable exact guarantee는 아니다.
- seed/RNG state는 per-call, max cells/steps와 cancel은 loop에서 bounded poll한다.
