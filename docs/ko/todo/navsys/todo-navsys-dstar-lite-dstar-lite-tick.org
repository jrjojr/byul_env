* Build compile-definition 불변식

- D* Lite module static은 =BYUL_STATIC=으로 tick controller와 callback adapter를 컴파일한다.
- root shared는 동일 source/header inventory를 =BYUL_EXPORTS=로 재컴파일·export하고
  =byul_tick= callback ABI와 wrapper load/exclusion을 별도로 검사한다.

* 문서 책임과 P0 판정

=byul/navsys/dstar_lite/dstar_lite_tick.h=의 tick 등록, 시간 적분, 이동과 planner lifetime을
독립적으로 재설계한다. 공통 lifecycle/cancellation 원칙은
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다. 현행 공개 struct와 attach/detach 구현에는 attached 상태
destroy UAF, double attach, 자동 detach 미실행, 미초기화 create/copy가 있어 ABI 2 opaque
전환 전에도 안전 수정이 필요하다.

* 실행 소유권과 테스트 위치

- 이 문서는 tick subscription, deterministic =advance(dt)=, 이동 step, controller state와
  planner/tick lifetime adapter만 소유한다.
- Planner lifecycle/config, key/open-list, edge update와 replan correctness는
  [[file:todo-navsys-dstar-lite-dstar-lite.org][D* Lite core 문서]]가 소유한다. Tick은
  공개 planner operation을 호출할 뿐 g/rhs, key, queue 또는 reconstruction을 직접
  변경하지 않는다.
- Tick controller의 실제 native test 파일은
  =byul/navsys/dstar_lite/tests/test_module_dstar_lite_tick.cpp=다. Planner core regression은
  =byul/navsys/dstar_lite/tests/test_module_dstar_lite.cpp=에 두며 tick 문서가 중복하지 않는다.
- Python 지원/제외와 lifetime assertion은 기존 shared test
  =tools/python/byul_wrapper/tests/test_dstar_lite.py=에 추가한다.

* 현재 구현 전수 분석

- =create_full=은 =max_time=만 설정하고 =base=와 나머지 필드를 정상 초기화하지 않는다.
- =copy=는 내용 복제가 아니라 사실상 미초기화 새 object를 반환한다. callback 등록을
  복제하는 것 또한 안전한 의미가 아니므로 controller는 non-copyable이어야 한다.
- =destroy/reset=은 tick system에서 callback을 detach하지 않는다. tick callback snapshot이
  남아 있으면 object free 뒤 UAF가 가능하다.
- update는 완료 시 =ticked=false=만 설정하고 =tick_request_detach= 또는 equivalent를
  호출하지 않는다. 문서의 자동 detach와 다르다.
- controller가 =tick_t*= 또는 subscription token을 저장하지 않아 destroy가 정확한 등록을
  해제할 수 없다.
- =prepare= double attach, 기존 =real_route= leak와 partial initialization을 허용한다.
- =speed_sec <= 0=, =unit_m <= 0=, NaN/Inf, negative =dt=를 검증하지 않는다.
- 완료 판정은 Numal =scalar_equal=에 의존한다. Tick 상태 전이는 Navsys lifecycle
  책임이므로 외부 Math Engine tolerance가 아니라 명시적인 Navsys 조건으로 교체한다.
- multi-step while에서 local start를 갱신하지 않아 같은 origin에서 반복 이동할 수 있다.
- =MAX_STEP=은 namespace 없는 macro이고 구현은 별도 literal 64를 사용한다.

* 현행 선언 전수 목록과 처분

| 선언 | 현재 의미/결함 | 처분 |
|------+----------------+------|
| =MAX_STEP= | 전역 macro, 구현 literal과 중복 | deprecate-forward |
| =dstar_lite_tick_t= | public mutable layout와 raw base | abi-major |
| =dstar_lite_tick_create= | base borrowed/retained 불명, OOM 채널 없음 | deprecate-forward |
| =dstar_lite_tick_create_full= | 실제 미초기화 P0 | deprecate-forward |
| =dstar_lite_tick_destroy= | attached callback을 해제하지 않음 | keep |
| =dstar_lite_tick_copy= | 미초기화 복제, registration 의미 불가능 | deprecate-forward |
| =dstar_lite_tick_reset= | detach 없이 ticked=false | deprecate-forward |
| =dstar_lite_tick_prepare= | void attach, double attach/error 없음 | deprecate-forward |
| =dstar_lite_tick_prepare_full= | config+attach 원자성 없음 | deprecate-forward |
| =dstar_lite_tick_update= | public/manual 호출 금지와 실제 export 충돌 | deprecate-forward |
| =dstar_lite_tick_complete= | caller가 tick handle을 다시 알아야 함 | deprecate-forward |

새 config/create/start/stop/advance/query API는 =add=다. struct opaque 전환과 copy symbol
제거는 =abi-major=다. ABI 1 =copy=는 안전한 detached config snapshot만 복제하도록
정의할 수 있는지 검토하고, 불가능하면 항상 실패를 표현할 새 API로 migration한다.

* 업계 lifecycle 기준

- [[https://docs.libuv.org/en/v1.x/handle.html][libuv 공식 handle lifecycle]]처럼 long-lived handle은 close 요청과 callback quiescence
  이후에만 storage를 해제한다.
  tick system이 snapshot iteration을 한다면 단순 unregister가 즉시 quiescence를 보장하는지
  tick contract를 먼저 확인한다.
- controller 상태를 =DETACHED/ATTACHED/RUNNING/COMPLETED/CANCELLED/FAILED=의 explicit
  enum으로 관찰한다. bool =ticked=만으로 registration과 실행 결과를 합치지 않는다.
- create는 configuration만 만들고 =start=가 registration을 수행한다. =stop=은 저장된
  subscription을 해제하므로 caller가 =tick_t*=를 다시 넘기지 않는다.
- deterministic unit test를 위해 tick callback과 분리된 =advance(dt)= operation을
  canonical state machine으로 만들고 callback은 이를 호출하는 adapter가 된다.

* 시간·이동 계약

- parameter 이름은 =tile_size_m=, =speed_m_per_sec=, =max_duration_sec=로 단위를 드러낸다.
- 모두 finite이고 tile/speed는 >0, duration은 >=0, =dt>=0=이어야 한다.
- step interval은 =tile_size_m / speed_m_per_sec=. 누적 시간이 여러 step을 포함하면
  =max_steps_per_update=까지만 처리하고 남은 누적 시간을 보존한다.
- 각 step 성공 뒤 local/current start를 즉시 갱신한다. goal/no-path/cancel/limit/callback
  failure를 서로 다른 status/state로 기록한다.
- max duration과 step budget의 경계(=, >)를 명시하고 wall clock 대신 전달된 monotonic
  delta만 사용한다.

ABI 2 =destroy=는 detached/quiescent object만 받거나 내부에서 synchronous stop을 완료해야
한다. tick system이 quiescence를 보장하지 못하면 reference-counted subscription/token API를
tick module에 먼저 추가해야 하며 이를 숨겨서 free하지 않는다.

* planner ownership과 thread 규칙

- controller가 planner를 retain할 방법이 현재 없다. 권장은 controller lifetime 동안
  caller가 planner를 보존하는 borrowed 계약과 destroy-order runtime assertion을 먼저
  제공하고, 장기적으로 planner retain/release 또는 shared owner를 설계하는 것이다.
- 같은 controller의 start/stop/advance/destroy 동시 호출은 지원하지 않거나 내부
  serialization을 제공한다. callback thread와 user thread stop race는 반드시 test한다.
- callback 안에서 planner/callback이 controller를 destroy하는 reentrancy를 금지하고
  deferred close를 제공한다.

* wrapper, build와 test 영향

- 이 header는 wrapper generator manifest에 등록되지 않았다. 지원하지 않을 것이면
  명시적 exclusion reason, 지원하면 state/status/lifecycle wrapper를 추가한다.
- 기존 native test는 public field를 직접 수정해 create/copy 결함을 가린다. opaque API만
  쓰는 black-box test로 바꾼다.
- static D* Lite와 root shared가 같은 tick source를 사용하고 =byul_tick= dependency와
  callback signature가 두 경로에서 일치해야 한다.
