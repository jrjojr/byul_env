* 문서 책임과 역할 판정

대상은 =byul/navsys/route_finder/route_finder_core.h=다. 이 파일은 finder lifecycle을
소유하지 않고 cost/heuristic callback과 기본 평가 함수만 제공하므로 public-foundation
=core=가 아니다. [[file:todo-navsys-industry-api-baseline.org][공통 업계 기준]]에 따라
=route_finder_evaluation.h= public-component를 새 canonical header로 만들고 기존 파일은
source compatibility를 위한 deprecated forwarding header로 둔다.

* 구현/소비 현황

- =route_finder_core.cpp=는 3개 cost와 6개 heuristic을 구현한다. NULL 좌표는
  =numeric_limits<float>::max()=로 숨기고 callback failure channel은 없다.
- 모든 search 구현은 callback의 =userdata=에 =nullptr=를 넘긴다.
- =route_finder.py=와 =route_finder_common.py=가 두 typedef, 9함수와 상수를 CFFI/registry에
  등록한다. Python callback을 직접 안전하게 bind하는 수명 모델은 없다.
- A*, Dijkstra, IDA*, Fringe, Greedy, RTA*, SMA*, Weighted A*와 FMM가 이 header에
  transitive 의존한다.

* 공식 비교 기준

- [[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]]는 edge
  weight가 non-negative이고 heuristic/distance가 같은 algebra에서 compare/combine됨을
  전제한다.
- [[https://www.boost.org/doc/libs/latest/libs/graph/doc/AStarHeuristic.html][Boost AStarHeuristic]]는
  goal estimate의 호출 계약을 별도 concept로 둔다.
- [[https://recastnav.com/classdtQueryFilter.html][Detour dtQueryFilter]]는 pass filter와 traversal
  cost를 query filter object로 묶고 query 동안의 수명을 정한다.
- [[https://doi.org/10.1109/TSSC.1968.300136][Hart/Nilsson/Raphael A* 원 논문]]의
  admissibility/consistency 조건과 최적성 주장을 API guarantee와 test에 연결한다.

현재 typedef parameter에는 이름이 없고 callback thread, nullable, reentrancy, retained
lifetime와 failure propagation metadata가 없다. 기본 Euclidean은 4방향 unit cost에는
admissible하지만, 대각선도 cost 1인 현재 8방향 model에서는 한 diagonal step에
=sqrt(2) > 1=을 추정해 과대평가한다. 사용자/0/non-uniform cost에서도 보장을 추측할 수
없다. Default를 이름만으로 최적성 보장처럼 사용하지 않는다.

- Callback은 synchronous caller thread, call 동안 같은 finder 재진입 금지, pointer는
  call-only다. =userdata=는 native가 역참조/해제하지 않는다.
- Cost는 finite, non-negative이고 heuristic과 같은 cost unit을 사용한다. 0-cost edge 허용과
  cycle 처리, overflow/NaN/Inf를 status로 분리한다.
- Heuristic descriptor에는 admissible/consistent 주장, topology와 minimum step cost 전제를
  기록한다. 잘못된 claim은 최적성 test에서 실패한다.
- =route_finder_bind_cost_func=와 =bind_heuristic_func=가 callback+userdata를 원자적으로
  보관하며 개별 raw setter는 사용하지 않는다.
