* 대상과 원전 기준

대상은 =rta_star.h/.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]]과
Korf의 [[https://doi.org/10.1016/0004-3702(90)90054-4][Real-Time Heuristic Search]] 및
[[https://cdn.aaai.org/AAAI/1987/AAAI87-024.pdf][AAAI Real-Time Heuristic Search]]를
algorithm identity 기준으로 삼는다.

* 실행 소유권과 테스트 위치

- 이 문서는 RTA*/LRTA* 구현·stateful conformance와 lookahead/learning typed
  config/default/validation만 소유한다.
- 공통 evaluator와 callback binding은
  [[file:todo-navsys-route-finder-route-finder-core.org][evaluation 문서]], facade/options/
  capability와 config 합류는
  [[file:todo-navsys-route-finder-route-finder.org][dispatcher 문서]]가 소유한다.
- RTA* direct/conformance assertion의 실제 파일은 shared native test
  =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=다.
  Dispatcher config 합류와 root consumer assertion은 각각
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=와
  =byul/navsys/tests/test_module_navsys.cpp=에서 해당 owner가 추가한다.
- RTA* 문서는 Python에 필요한 config/state/status 기대값을 제공한다. 실제 wrapper
  consumer assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* P0 선언/구현 및 Algorithm Identity 결함

- =rta_star_config_create()=와 =rta_star_config_create_full()=,
  =rta_star_config_destroy()=는 public 선언/CFFI에는 있으나 =rta_star.cpp=에 정의가 없다.
  Root shared symbol intersection에서 즉시 실패해야 한다.
- Dispatcher는 config type을 사용하지 않고 =void* typedata=를 =int*=로 해석한다.
- 현재 =rta_iterative_eval=은 한 greedy path를 depth만큼 따라가며 neighbor별 minimax/lookahead
  tree와 heuristic backup/learning을 구현하지 않는다. 외부 loop는 visited node 재방문을
  금지한다. Korf RTA*/LRTA*와 같은 guarantee를 주장할 수 없다.
- Lookahead 중 debug visited를 result route에 섞고, =depth_limit <= 0=, callback userdata,
  cost/heuristic numeric failure와 cancel/OOM을 검증하지 않는다.
- Header example은 현재 =route_finder_create_full= signature와 맞지 않고 type 이름도
  =route_finder=처럼 잘못 사용한다.

* 전체 Public Symbol Inventory와 disposition

=RTA_STAR_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_RTA_STAR_H=로 교체한다.

| symbol | 종류/현재 상태 | disposition | 목표 |
|--------+----------------+-------------+------|
| =rta_star_config_t= | public one-field value type | =ABI 1.x: keep; companion: add -> ABI-major: keep= | versioned typed config companion 추가 |
| =depth_limit= | public =int= field | =ABI 1.x: keep; companion: add -> ABI-major: keep= | positive range와 expansion unit 명시 |
| =rta_star_config= | pointer typedef alias | =ABI 1.x: keep -> ABI-major: internalize= | canonical value/config-view에서 제거 |
| =rta_star_config_create= | 선언만 있고 정의 없음 | =ABI 1.x: keep -> ABI-major: internalize= | P0 legacy symbol 구현; canonical value에는 allocation 불필요 |
| =rta_star_config_create_full= | 선언만 있고 정의 없음 | =ABI 1.x: keep; companion: add -> ABI-major: internalize= | P0 구현 후 =init/default= value API로 전환 |
| =rta_star_config_destroy= | 선언만 있고 정의 없음 | =ABI 1.x: keep -> ABI-major: internalize= | legacy allocation 쌍 구현 후 canonical value에서 제거 |
| =find_rta_star= | one-shot route pointer | =ABI 1.x: deprecate-forward; conformance-block -> ABI-major: internalize= | stateful RTA*/LRTA* conformance 전 facade 지원 금지 |

* 목표 상태형 Signature 후보

RTA*는 planning과 action을 interleave하므로 full route one-shot보다 stateful step을 우선 검토한다.

#+begin_src c
BYUL_API navsys_status_t route_finder_bind_rta_star_config(
    route_finder_t* finder,
    const rta_star_config_t* config);

BYUL_API navsys_status_t route_finder_rta_star_begin(
    route_finder_t* finder,
    const coord_t* start,
    const coord_t* goal,
    const navsys_search_options_t* options);

BYUL_API navsys_status_t route_finder_rta_star_step(
    route_finder_t* finder,
    coord_t* out_next,
    navsys_search_stats_t* out_stats);
#+end_src

Heuristic learning table owner, reset/snapshot, per-action lookahead expansion bound와 execution 실패를
정한다. Full route =find_rta_star=는 step을 반복하는 compatibility adapter일 뿐 실시간 latency
보장을 대신하지 않는다.
