* 대상과 원전 기준

대상은 =weighted_astar.h/.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]],
Pohl의 [[https://doi.org/10.1016/0004-3702(70)90007-X][Heuristic Search Viewed as Path
Finding in a Graph]]과 [[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]]의
weight/heuristic precondition을 기준으로 bounded-suboptimal claim을 검증한다.

* 구현/소비와 결함

- =find_weighted_astar= 하나가 module/root/SDK/CFFI/dispatcher에 공개된다.
- =weight <= 0=을 1로 silent 교정하고 NaN/Inf는 거르지 않는다. Dispatcher는 raw
  =float*= typedata에서 0.1..10만 수용하고 기본 1.5를 사용해 direct API와 정책이 다르다.
- Header의 1.2..2.5/5.0 추천은 workload 근거이며 stable API 계약이 아니다.
- cost/heuristic userdata를 전달하지 않고 callback failure와 invalid numeric result를 구분하지 않는다.
- =w= suboptimal bound는 admissible heuristic, non-negative cost와 올바른 graph-search 조건이
  있어야 하는데 header/test는 이를 검증하지 않는다. Route final cost도 기록하지 않는다.
- Header는 선언에 사용하지 않는 =scalar.h=를 include한다. Weighted A*의 weight와
  heuristic 정책은 Navsys 책임이며 Numal/Math Engine dependency가 아니다.

* 전체 Public Symbol Inventory와 disposition

=WEIGHTED_ASTAR_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_WEIGHTED_ASTAR_H=로 교체한다.

| symbol | 현재 signature | wrapper | disposition |
|--------+------------------+---------+-------------|
| =find_weighted_astar= | =route_t*(grid,start,goal,cost,heuristic,float weight,int,bool)= | generated CFFI | =ABI 1.x: deprecate-forward; conformance-block -> ABI-major: internalize= |

Weight는 finite이고 승인 범위(기본적으로 >=1)를 status로 검증한다. =w= bound를 보장할 수
없는 heuristic/cost이면 descriptor/stats가 “unbounded”를 명시하며 silent 최적성 주장을 하지 않는다.
