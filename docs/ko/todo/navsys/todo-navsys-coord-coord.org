* Build compile-definition 불변식

- module static target은 =BYUL_STATIC=으로 이 header/source를 컴파일한다.
- root shared target은 같은 inventory를 =BYUL_EXPORTS=로 다시 컴파일하고 public symbol을
  export한다. 두 경로의 성공을 서로 대신하는 증거로 사용하지 않는다.

* 문서 책임

=byul/navsys/coord/coord.h= 하나만을 대상으로 공개 C ABI, 수치 의미, 오류 모델,
소유권, 문자열 변환과 언어 바인딩을 함수 단위로 재설계한다. 공통 정책은
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다. 이 문서는 구현 변경 자체가 아니라 변경 전
승인해야 할 계약과 검증 순서를 고정한다.

* 조사한 현재 경로와 역할

- 선언: =byul/navsys/coord/coord.h=
- 정의: =byul/navsys/coord/coord.cpp=
- 모듈: =byul/navsys/coord/CMakeLists.txt=의 static =coord= 및 root shared가 동일
  source/header inventory를 컴파일해야 한다.
- 소비자: Navsys 전 모듈, =coord.hpp=, C/C++ 테스트, 설치 SDK, handwritten Python
  =c_coord=.
- 현재 공개 layout은 =int x; int y;=이다. ABI 1.x에서 field type/order를 바꾸지 않는다.
- 현재 구현은 좌표를 [-200000000, 200000000]으로 암묵 wrapping하고, 곱셈 중간값은
  signed overflow가 가능하며, 0 나눗셈과 NULL을 정상값/무동작으로 숨긴다.

* 업계 기준과 적용 판정

- Vulkan/SQLite류 C ABI처럼 실패 가능한 연산은 status와 out parameter를 분리한다.
- caller buffer는 용량과 실제/필요 길이를 함께 받으며 query-only 호출을 허용한다.
- C++/Python의 associative key는 equality와 hash가 일치하고 key로 쓰이는 동안 값이
  바뀌지 않아야 한다. Python 공식 기준은
  [[https://docs.python.org/3/reference/datamodel.html#object.__hash__][Data model의 hash 계약]]이다.
- 좌표 범위 초과를 자동 보정하는 정책은 일반적인 수치 API의 숨은 부작용이다.
  기본 API는 checked, wrapping이 필요하면 이름에 =wrapped=를 드러낸다.

* 함수군별 상세 결함과 목표 계약

** 범위와 산술

- =COORD_MIN/MAX=는 저장 가능한 C =int= 범위가 아니라 프로젝트 정책 범위다. 이름을
  =BYUL_COORD_COMPONENT_MIN/MAX=로 바꾸고 exact numeric ABI manifest에 기록한다.
- checked 연산은 64-bit 중간값으로 계산하고 범위 초과 시
  =NAVSYS_STATUS_INVALID_ARGUMENT= 또는 별도 =OUT_OF_RANGE= 결정 전까지 출력과 원본을
  보존한다.
- legacy 연산은 기존 wrapping을 재현하는 adapter로 남기되 deprecated annotation과
  문서로 silent policy를 노출한다. 새 코드가 legacy를 호출하지 않게 lint한다.
- =coord_div_checked=는 scalar 0을 거부하고 C 정수 나눗셈의 truncation-toward-zero를
  명시한다. =INT_MIN / -1=도 checked 경로에서 잡는다.

** 비교, 거리와 방향

- canonical compare는 정확히 -1, 0, 1만 반환하고 lexicographic =(x,y)=를 사용한다.
- 거리 차분은 먼저 64-bit 또는 double로 승격한다. Euclidean 결과는 =double=,
  Manhattan은 =int64_t= out을 우선 후보로 한다.
- 동일한 두 점은 방향이 정의되지 않은 입력으로 취급한다. 기존 0 반환은 legacy에서만
  보존하고 새 함수는 status를 반환한다.
- =coord_next_to_goal=의 이웃 topology를 8-neighbor로 고정하는지, 대각선 tie가
  x/y 중 어느 축 우선인지 golden test로 선언한다.

** allocation, format와 I/O

- =create/copy=는 C++ =new= 예외가 C 경계를 넘지 않게 catch하고 status/out handle을
  제공한다. 작은 value type은 caller storage 초기화가 canonical이다.
- 문자열 변환은 =(object, buffer, capacity, out_required)= 순서와 two-call 규칙을 쓴다.
  NUL 포함 필요 크기인지 제외 길이인지 하나로 고정한다.
- =coord_print=는 library core에서 stdout을 만지지 않는다. debug/log sink 층으로
  옮기고 ABI 1 함수는 =coord_format= 결과를 출력하는 deprecated adapter로만 둔다.

=coord_add_checked(out, out, b)= 같은 aliasing을 허용하는지 모든 산술 함수에서 동일하게

* ABI, wrapper, build와 test 영향

- ABI 1: 현행 symbol과 =coord_t= layout을 유지하고 새 checked symbol만 추가한다.
- ABI 2: field type 명시 또는 opaque 전환은 external consumer 조사와 별도 migration
  guide 없이는 진행하지 않는다.
- Python =c_coord=는 mutable한데 =__hash__=가 있어 dict/set key가 된 뒤 hash가 바뀐다.
  immutable value wrapper와 mutable wrapper를 분리하거나 mutable wrapper의
  =__hash__ = None=을 적용한다. =close()= 뒤 모든 접근도 예외로 막는다.
- module static과 root shared 양쪽의 symbol/export를 비교하며 설치 SDK C17/C++17
  consumer에서 새 선언을 직접 include/link한다.
- test는 NULL, 경계값, 산술 overflow, 0 나눗셈, aliasing, 동일점 각도, 모든 사분면,
  two-call format, OOM/예외 경계, layout size/align/offset을 포함한다.
