- root shared는 같은 source/private helper를 =BYUL_EXPORTS=로 재컴파일한다. ABI 1 기간의

* 문서 책임과 결론

=byul/navsys/dstar_lite/dstar_lite_pqueue.h=의 D* Lite open list를 원 논문 연산과
대조한다. 공통 계약은 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이다. public generic container가 아니라
D* Lite implementation detail에 가까우므로 ABI 2에서는 internalize가 권장된다. ABI 1에는
atomic by-value API를 먼저 추가하고 기존 pointer API를 deprecated한다.

* 원 알고리즘이 요구하는 연산

Koenig/Likhachev [[https://publications.ri.cmu.edu/d-lite][D* Lite]]의 =ComputeShortestPath=
에서는 =TopKey()=를 현재 시작 key와 비교하고, =k_old=와 vertex를 같은 top entry에서
꺼낸 뒤 stale key면 새 key로 재삽입한다. 따라서 queue가 제공해야 하는 최소 primitive는
다음이다.

- top =(key, coord)=의 원자적 snapshot
- pop =(key, coord)=의 원자적 제거
- coord당 current key를 갱신하는 upsert/remove
- empty/size와 deterministic exact lexicographic ordering

현재처럼 key/coord를 별도 조회하면 두 호출 사이 mutation과 소유권 오류가 생기고
=k_old=를 잃기 쉽다.

* 현재 구현 결함

- heap key pointer를 key로 둔 map, heap coord pointer vector, coord→key hash를 중복 저장한다.
- 같은 coord를 다시 push하면 reverse map은 최신 key만 가리키지만 이전 queue entry는
  stale로 남는다.
- grouping은 approximate equality loop, map ordering은 exact pointer comparator라 key
  equivalence가 일치하지 않는다.
- =pop=은 owned coord allocation, =peek=은 borrowed coord, =get_key_by_coord=은 borrowed
  mutable key, =top_key=는 owned key copy다. 이름만으로 구분할 수 없다.
- wrapper는 =pop=을 non-owning으로 감싸 leak하고, borrowed =get_key_by_coord=를 own=True로
  감싸 double-free/UAF를 만든다.

=internalize=는 ABI 2 최종 처분이다. ABI 1.x에서 symbol을 즉시 제거하지 않는다. 아래
status/by-value 선언은 public compatibility를 위해 =add=한 뒤 D* implementation이 먼저
사용하고, ABI 2에서는 private C++ detail API로 축소한다.

* 목표 storage와 invariant

- key와 coord를 by-value 저장하고 exact canonical key comparator를 쓴다.
- 권장 invariant는 coord당 live entry 정확히 하나다. =upsert=가 기존 entry를 제거하고
  새 key로 삽입한다.
- lazy duplicate heap을 선택한다면 generation을 entry에 넣고 pop에서 stale을 건너뛰며,
  size가 live count인지 physical count인지 명시한다. 메모리 상한 때문에 unique-entry
  구조를 우선한다.
- equal key tie는 coord lexicographic 또는 stable insertion sequence로 deterministic하게
  고정한다. 알고리즘의 최적성에는 영향을 주지 않아도 reproducibility에는 필요하다.
- copy는 planner snapshot에 필요하지 않으면 제거한다. 필요하면 complete independent
  deep copy와 status를 제공한다.

* D* Lite 통합 영향

- =ComputeShortestPath=는 반드시 =peek_min=에서 loop condition을 판정하고
  =pop_min=으로 받은 동일 entry의 =k_old=를 보존한다.
- =UpdateVertex=는 coord가 inconsistent하면 =upsert=, consistent하면 =remove=한다.
- queue exact key ordering과 D* key equality/hash가 동일 relation을 사용한다.
- g/rhs/frontier public getter로 queue를 외부에서 mutate하게 하는 API는 main D* 문서에
  따라 ABI 2에서 internalize한다.

* wrapper, build와 test 영향

- pqueue를 standalone Python public class로 유지할 근거가 없으면 ABI 2에서 wrapper도
  제거한다. ABI 1 기간에는 by-value entry만 노출한다.
- parent retention, owned/borrowed flag 추론을 모두 없앤다.
- test는 duplicate coord update, stale-key branch, exact tie, empty, remove, copy,
  randomized reference queue, OOM과 sanitizer를 포함한다.
- static D* module과 root shared는 같은 pqueue source/private helper를 컴파일한다.
