* P0 Algorithm Identity 결함

현재 구현은 각 threshold iteration마다 =cost_coord_pq_t= priority queue, =cost_so_far=,
=came_from=과 =visited= hash를 새로 만든다. 깊이 우선 재귀/명시 stack과 path-local memory를
사용하는 IDA*가 아니므로 header의 “very low memory”, “DFS 기반”, “optimal” 설명을 현재
구현에 적용할 수 없다. Operational capability는 conformance가 끝날 때까지 false로 두거나
현재 구현을 다른 이름의 custom threshold search로 분류해야 한다.

추가 결함:

- cost/heuristic userdata가 없고 callback에 NULL을 전달한다.
- =max_retry=는 모든 threshold round가 공유하지만 expansion인지 retry인지 불명확하다.
- =best_f=가 전체 실행에서 감소만 추적해 partial endpoint 선택이 goal progress와 일치하지 않는다.
- Header의 760/88 iteration과 Manhattan 추천 수치는 특정 workload 근거가 없는 경험값이다.
- Dispatcher는 IDA* 실행 전에 finder heuristic을 Manhattan으로 영구 변경한다.
