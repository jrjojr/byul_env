* 대상과 비교 기준

대상은 =dijkstra.h/dijkstra.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]]과
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/dijkstra_shortest_paths.html][Boost.Graph Dijkstra]]의
non-negative edge weight, predecessor/distance output, decrease-key/color/visitor와 O((V+E)logV)
계약을 비교한다.

* 실행 소유권과 테스트 위치

- 이 문서는 Dijkstra search 구현·conformance와 Dijkstra 전용 config/default/validation만
  소유한다.
- 공통 evaluator와 callback binding은
  [[file:todo-navsys-route-finder-route-finder-core.org][evaluation 문서]], facade/options/
  capability와 config 합류는
  [[file:todo-navsys-route-finder-route-finder.org][dispatcher 문서]]가 소유한다.
- Dijkstra direct/conformance assertion의 실제 파일은 shared native test
  =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=다.
  Dispatcher 및 root consumer assertion은 각각
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=와
  =byul/navsys/tests/test_module_navsys.cpp=에서 해당 owner가 추가한다.
- Dijkstra 문서는 Python에 필요한 config/status 기대값을 제공한다. 실제 wrapper consumer
  assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 구현/소비와 결함

- =find_dijkstra= 하나가 module/root/SDK/CFFI/dispatcher에 공개된다.
- cost callback에 userdata를 전달할 parameter가 없고 구현은 항상 =nullptr=를 넘긴다.
- negative/NaN/Inf cost를 수용해 algorithm 전제가 깨져도 success route처럼 보일 수 있다.
- decrease-key 대신 duplicate entry를 push하고 stale-pop/expanded/reopened 통계를 제공하지 않는다.
- NULL 또는 nonpositive budget은 NULL이며 partial/no-path/OOM과 충돌한다. 최종 누적 cost를
  =route_t.cost=에 기록하지 않는다.

* 전체 Public Symbol Inventory와 disposition

=DIJKSTRA_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_DIJKSTRA_H=로 교체한다.

| symbol | 현재 signature | wrapper | disposition |
|--------+------------------+---------+-------------|
| =find_dijkstra= | =route_t*(grid,start,goal,cost_func,int,bool)= | generated CFFI | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= |

Cost callback+userdata는 한 call-only view 또는 finder binding이고, cost는 같은 unit의 finite
non-negative 값이어야 한다. Stats에 final cost, expanded/generated/reopened/stale-pop과 peak
workspace를 기록한다.
