* Build compile-definition 불변식

- D* Lite module static은 =BYUL_STATIC=으로 key C ABI를 컴파일한다.
- root shared는 동일 key source/header를 =BYUL_EXPORTS=로 재컴파일·export하고 exact
  comparison/hash symbol과 value layout을 별도로 검사한다.

* 문서 책임과 P0 결함

=byul/navsys/dstar_lite/dstar_lite_key.h=의 D* Lite lexicographic key value API를
재설계한다. 공통 기준은 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이다. 현재 approximate equality,
exact ordering, bitwise hash가 서로 다른 동치관계를 사용하여 ordered/unordered container와
Python hash 계약을 동시에 위반한다.

* 알고리즘·언어 기준

- Koenig/Likhachev의 [[https://publications.ri.cmu.edu/d-lite][D* Lite 원 논문]]에서 key는
  =(k1,k2)= lexicographic ordering으로 open list 순서를 정한다. 근사 동등성은 queue의
  equivalence가 아니다.
- C++ comparator는 strict weak ordering이어야 한다. C++ 표준
  [[https://eel.is/c++draft/associative.reqmts][associative requirements]]와
  [[https://wiki.sei.cmu.edu/confluence/display/cplusplus/CTR57-CPP.+Provide+a+valid+ordering+predicate][SEI CTR57-CPP]]를 적용한다.
- equal object는 같은 hash를 가져야 한다. 근사 비교는 비추이적일 수 있으므로 별도
  =is_close= 진단 함수로만 제공한다.

* 현행 구현 근거

- =equal=은 scalar tolerance 비교다.
- 구현은 Numal =scalar.h=의 =scalar_equal=을 호출한다. 이 결합은 D* Lite key의
  동치관계를 외부 범용 tolerance 정책에 맡기는 구조이므로 제거한다.
- =compare=는 먼저 approximate equal이면 0을 반환하고, 아니면 exact
  lexicographic을 사용한다. A≈B, B≈C, A!≈C가 가능하다.
- =hash=는 exact float bit 조합이므로 approximate-equal key가 다른 hash를 가질 수
  있다. =-0/+0=과 NaN도 canonical policy가 없다.
- 작은 8-byte value를 create/copy에서 heap allocation하며 C++ =new= 실패 채널이 없다.

* 현행 선언 전수 목록과 처분

| 선언 | 현재 의미/결함 | 처분 |
|------+----------------+------|
| =dstar_lite_key_t= | 공개 ={float k1,k2}= value layout | keep |
| =dstar_lite_key_create= | heap zero-key, OOM 채널 없음 | deprecate-forward |
| =dstar_lite_key_create_full= | NaN/Inf 검증 없는 heap 생성 | deprecate-forward |
| =dstar_lite_key_copy= | tiny value heap 복제 | deprecate-forward |
| =dstar_lite_key_destroy= | heap family 종결 | keep |
| =dstar_lite_key_equal= | approximate, hash/order와 불일치 | deprecate-forward |
| =dstar_lite_key_compare= | 비추이 approximate equivalence | deprecate-forward |
| =dstar_lite_key_hash= | equal⇒same-hash 위반 | deprecate-forward |

새 init/exact/is_close/metadata 함수는 =add=다. =float=를 =double=로 직접 바꾸는 후보와
field layout 변경은 =abi-major=다. D* Lite 내부 전용으로 완전히 숨기는 선택도 ABI 2
consumer 조사 뒤 별도 =internalize= 결정을 내린다.

* 목표 수치 계약

- queue에 들어가는 canonical key는 finite 또는 algorithm이 사용하는 positive infinity만
  허용할지 결정한다. D* Lite는 초기 g/rhs infinity가 key에 나타날 수 있으므로
  =+Inf=를 valid maximum sentinel로 허용하고 NaN, =-Inf=를 거부하는 안을 우선한다.
- =-0.0f=는 =+0.0f=로 canonicalize한다.
- exact equality는 canonical field equality, compare는 exact lexicographic -1/0/1,
  hash는 두 canonical field와 exact equality를 공유한다.
- approximate comparison은 =abs_tol=와 =rel_tol=를 caller가 명시하며 queue/hash/equality,
  termination 조건에는 사용하지 않는다.
- ABI 1 public layout의 size/align/offset을 manifest로 고정한다.

정렬용 compare가 NULL을 받는 것은 invalid precondition이다. public exact compare가 status를
반환하지 않는다면 NULL debug assertion과 release fallback을 ABI contract에 적거나,
status+out_compare 대안을 최종 API review에서 결정한다.

* wrapper, build와 test 영향

- Python wrapper의 현재 approximate =__eq__=와 exact =__hash__= 조합을 즉시 제거한다.
  immutable exact key로 노출하거나 approximate equality를 유지하면 unhashable로 만든다.
- =dstar_lite_key.hpp=의 세 comparator와 =std::hash=는 별도 문서에 따라 internalize한다.
- static D* Lite와 root shared가 같은 exact implementation을 사용하고 installed SDK의
  layout/symbol을 검사한다.
- test는 transitivity, antisymmetry, total lexicographic ordering, equal-hash, ±0,
  adjacent floats, tolerance chain, Inf/NaN과 OOM을 포함한다.
