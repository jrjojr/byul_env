* 대상과 원전 기준

대상은 =fast_marching.h/.cpp=다. 공통 형태는
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]에 연결한다. 수치/algorithm
conformance는 Sethian의 [[https://doi.org/10.1073/pnas.93.4.1591][A Fast Marching Level Set
Method for Monotonically Advancing Fronts]]에 정의된 positive speed, upwind Eikonal update,
FAR/TRIAL/KNOWN causality와 O(N log N)를 기준으로 한다.

* 실행 소유권과 테스트 위치

- 이 문서는 Fast Marching search/field 구현·conformance와 FMM 전용 typed
  config/default/validation만 소유한다.
- 공통 cost evaluator와 callback binding은
  [[file:todo-navsys-route-finder-route-finder-core.org][evaluation 문서]], facade/options/
  capability와 config 합류는
  [[file:todo-navsys-route-finder-route-finder.org][dispatcher 문서]]가 소유한다.
- Fast Marching direct/conformance assertion의 실제 파일은 shared native test
  =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=다.
  Dispatcher 및 root consumer assertion은 각각
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=와
  =byul/navsys/tests/test_module_navsys.cpp=에서 해당 owner가 추가한다.
- FMM 문서는 Python에 필요한 config/status 기대값을 제공한다. 실제 wrapper consumer
  assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 현재 구현/소비와 P0 결함

- Header가 cell value type, hash copy hook, owning distance-field layout와 one-shot path API를 한
  파일에 섞고 전부 SDK/CFFI에 노출한다.
- =fmm_cell_copy/destroy=는 public header 선언인데 =BYUL_API=가 없고 generic =void*= hash
  hook이다. 외부 API가 아니라 implementation detail이다.
- Update는 4축 이웃 값만 사용하는데 expansion/path descent는 =navgrid_copy_neighbors=의
  4/8방향 mode를 따른다. Grid spacing, speed/cost reciprocal 관계와 unit이 없다.
- =find_fast_marching=는 radius를 직선거리의 1.5배로 고정해 더 긴 detour가 존재해도 goal을
  도달 불가로 만들 수 있다. Fallback은 goal에 가장 가까운 cell이 아니라 visit-order의
  마지막 cell이다.
- =MAX_RADIUS=를 넘거나 nonpositive radius는 1e6으로 silent 교정하고 NaN/Inf/negative cost를
  검사하지 않는다. Cost callback userdata도 항상 NULL이다.
- unlimited =max_retry <= 0=에서는 counter가 증가하지 않아 stats가 0이 된다.
- =fmm_dump_ascii=는 library 함수가 stdout을 직접 쓰고, public =fmm_grid_t=는 hash/list owner와
  navgrid width/height layout을 노출한다.

Distance field와 route extraction을 분리한다. 지원 topology/speed model이 원전 전제를 만족하지
않으면 =UNSUPPORTED=다. Field export는 two-call caller-buffer 모델을 사용하고 internal hash/
visit list pointer는 반환하지 않는다.
