* 대상과 비교 기준

대상은 =bfs.h/bfs.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]]과
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/breadth_first_search.html][Boost.Graph BFS]]의
queue/color map, visitor event와 O(V+E) 계약을 비교한다. BFS는 동일 edge cost에서만
최소 step path를 보장하며 weighted shortest path API로 설명하지 않는다.

* 실행 소유권과 테스트 위치

- 이 문서는 BFS traversal 구현·conformance와 BFS 전용 config/default/validation만
  소유한다.
- 공통 evaluator/callback binding은
  [[file:todo-navsys-route-finder-route-finder-core.org][evaluation 문서]], facade/options/
  capability와 config 합류는
  [[file:todo-navsys-route-finder-route-finder.org][dispatcher 문서]]가 소유한다.
- BFS direct/conformance assertion의 실제 파일은 shared native test
  =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=다.
  Dispatcher 및 root consumer assertion은 각각
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=와
  =byul/navsys/tests/test_module_navsys.cpp=에서 해당 owner가 추가한다.
- BFS 문서는 Python에 필요한 config/status 기대값을 제공한다. 실제 wrapper consumer
  assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 구현/소비와 결함

- =find_bfs=는 module/root/SDK와 generated CFFI에 등록되고 dispatcher/native test가 사용한다.
- NULL 또는 =max_retry <= 0=이면 NULL이다. 다른 algorithm의 “0 이하 unlimited” 설명과
  다르다.
- 예산 소진 시 마지막 탐색 node까지 route를 반환하고 success=false로 두지만 limit/no-path/
  OOM을 구분하지 않으며 route cost는 0이다.
- neighbor 순서가 tie path를 결정하지만 deterministic 계약이 없다. 방문 event는 discover
  count만 저장하고 callback/observer/cancel은 없다.
- allocation 실패와 NULL neighbor/container를 검사하지 않으며 legacy tests는 성공/강제실패
  bool만 확인한다.

* 전체 Public Symbol Inventory와 disposition

=BFS_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_BFS_H=로 교체한다.

| symbol | 현재 signature | wrapper | disposition |
|--------+------------------+---------+-------------|
| =find_bfs= | =route_t*(grid,start,goal,int max_retry,bool debug)= | generated CFFI | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= |

* 목표 Signature 후보

#+begin_src c
BYUL_API navsys_status_t route_finder_find_bfs_ex(
    const navgrid_t* grid,
    const coord_t* start,
    const coord_t* goal,
    const navsys_search_options_t* options,
    route_t** out_route,
    navsys_search_stats_t* out_stats);
#+end_src

=max_retry=는 =max_expansions=로 의미를 고정한다. Status는 complete/no-path/limit/cancel/OOM을
분리하고 stats는 level, expanded/generated, peak frontier와 endpoint를 기록한다. Weighted
grid에서는 step-optimal만 보장하거나 =UNSUPPORTED=를 반환한다.
