* 구현 감사: 현재 lattice가 다른 generator와 불일치

| 구간 | 현재 코드 | 판정 |
|------+----------+------|
| 초기 row | =set_id[0][odd x]=, =grid[0][odd x]=PASSAGE= | 외곽 y=0을 passage로 열어 다른 generator의 odd/odd interior와 충돌 |
| row loop | =y=0; y<h; y+=2= | logical cell row가 even coordinate에 배치됨 |
| vertical join | =y+1,y+2= passage | row representation 자체는 일관되나 public border-wall 설명과 불일치 |
| final row | =y=h-1=에서 set merge | odd height이면 마지막 외곽 row를 passage/merge에 사용 |
| RNG | =mt19937(time(nullptr))=, =rng()%size= | seed 재현 불가; modulo bias가 있을 수 있음 |
| allocation | maze/vector/map/hash | create failure/exception/hash insert 실패가 C ABI로 새거나 무시됨 |
| topology test | print와 non-NULL | boundary opening, connectivity, no cycle 미검증 |

현재 결과가 의도한 wall raster인지 바꾸기 전에 fixture로 동결한다. 신규 API는 다른 maze
family와 동일하게 logical cell을 odd/odd interior에 두고 외곽을 blocked로 유지하는 방향을
기본 후보로 한다. 이는 legacy 결과를 바꾸므로 legacy symbol은 compatibility adapter 또는
별도 legacy implementation을 유지할 수 있다.

* 목표 알고리즘 계약

- working memory는 O(logical width) row set이며 full 2D =set_id=는 제거 후보이다.
- 각 non-final row에서 서로 다른 adjacent set을 임의 horizontal join하고, 각 set마다 최소
  한 vertical join을 보장한다. final row는 남은 서로 다른 adjacent set을 모두 연결한다.
- 결과 passage graph는 connected/acyclic, edge = vertex-1, outer border blocked다.
- exact seed replay와 RNG draw order, row streaming 가능 여부를 capability에 기록한다.
- dimension, max cells/steps, cancellation과 allocation failure는 =navsys_status_t=로 구분한다.
