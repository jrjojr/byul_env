* Build compile-definition 불변식

- D* Lite module static은 =BYUL_STATIC=으로 algorithm source/header를 컴파일한다.
- root shared는 동일 inventory를 =BYUL_EXPORTS=로 재컴파일·export한다. module static test와
  root runtime/export/wrapper load는 서로 독립된 release gate다.

* 문서 책임과 최종 방향

=byul/navsys/dstar_lite/dstar_lite.h= 하나의 공개 ABI, D* Lite 알고리즘 구현, callback,
동적 재계획과 wrapper를 전수 재설계한다. 공통 C ABI는
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다. 목표는 다음 세 층을 분리하는 것이다.

- public planner: opaque lifecycle, config/binding, edge change 통지, non-blocking replan,
  route/result/stats/cancel.
- internal algorithm: g/rhs/open list, CalculateKey/UpdateVertex/ComputeShortestPath와 route
  reconstruction.
- orchestration: 이동, sleep/interval, changed-source polling과 tick 등록. core planner에서
  제거하고 =dstar_lite_tick= 또는 application 층이 담당한다.

* 실행 소유권과 Child 합류 Gate

- 이 문서는 planner lifecycle/config/binding, =CalculateKey/UpdateVertex/
  ComputeShortestPath= 제어 흐름, edge-change 통지, reconstruction과 replan
  status/result/stats만 소유한다.
- Key domain·exact compare/hash는
  [[file:todo-navsys-dstar-lite-dstar-lite-key.org][dstar_lite_key.h 문서]], C++ comparator의
  private 전환은
  [[file:todo-navsys-dstar-lite-dstar-lite-key-cpp.org][dstar_lite_key.hpp 문서]],
  open-list storage와 atomic queue operation은
  [[file:todo-navsys-dstar-lite-dstar-lite-pqueue.org][dstar_lite_pqueue.h 문서]]가 소유한다.
  이 문서는 세 child의 완료 API를 소비하며 key 또는 queue 구현을 중복 수정하지 않는다.
- Tick 등록, 시간 적분, 이동과 subscription lifetime은
  [[file:todo-navsys-dstar-lite-dstar-lite-tick.org][dstar_lite_tick.h 문서]]가 소유한다.
  Planner core는 sleep, movement와 tick callback을 수행하지 않는다.
- Core algorithm의 실제 native test 파일은
  =byul/navsys/dstar_lite/tests/test_module_dstar_lite.cpp=다. Root one-shot consumer는
  =byul/navsys/tests/test_module_navsys.cpp=, Python planner/lifetime은
  =tools/python/byul_wrapper/tests/test_dstar_lite.py=에서 검증한다.

* 현재 build와 ABI 위치

- 선언/정의: =dstar_lite.h= / =dstar_lite.cpp=.
- =dstar_lite/CMakeLists.txt=는 module static과 root shared 양쪽에 source를 넘긴다. root
  shared만 public runtime이고 두 target의 source/header inventory는 동일해야 한다.
- header는 =navgrid/coord/list/hash/route/key/pqueue/route_finder=를 직접 포함하며 내부
  storage type까지 public include DAG로 끌어온다.
- 구현은 g/rhs와 key 판정에서 Numal =scalar_equal=을 반복 호출한다. D* Lite의
  수치 비교는 planner invariant의 일부이며 Math Engine으로 위임하지 않는다.
- handwritten Python =c_dstar_lite=가 있으나 선언과 실제 호출이 크게 어긋난다.
- public =dstar_lite_t= layout은 모든 state/owning pointer/callback/userdata/counter/bool을
  노출하므로 field 추가/재배치조차 ABI를 깨뜨린다. ABI 2에서 opaque로 전환한다.

* 업계·원 논문 기준

- [[https://publications.ri.cmu.edu/d-lite][Koenig/Likhachev D* Lite publication]] 및
  [[https://www.cs.cmu.edu/~motionplanning/papers/sbp_papers/integrated3/koenig_dstarlite_aaai02b.pdf][원 논문 알고리즘]]을 canonical reference로 둔다.
- 원 논문 Initialize는 =rhs(goal)=0 뒤 =goal=을 =CalculateKey(goal)=로 open list에 넣는다.
- =ComputeShortestPath= loop 조건은 =TopKey < CalculateKey(start)= 또는
  =rhs(start) != g(start)=이다. pop 전 =k_old=를 저장하고 =k_old < CalculateKey(u)=이면
  updated key로 재삽입한다.
- robot start가 이동하면 =km += h(s_last,s_start)=, =s_last=s_start=, 변경된 directed
  edge와 predecessor를 갱신한 뒤 재계획한다.
- public API는 SQLite/libuv/Vulkan류 opaque lifecycle, atomic callback+userdata,
  explicit status/options/result, cooperative cancellation과 caller-visible stats를 적용한다.

* 구현과 원 논문의 구체적 차이

** Initialize와 key

- 현재 init은 =start=의 key를 계산하고 그 key로 =goal=을 push한다. 원 논문은
  =CalculateKey(goal)=을 push한다.
- g/rhs default infinity와 goal rhs=0, open-list uniqueness를 init 후 invariant로 검사한다.
- approximate D* key equality를 termination/open-list ordering에 쓰지 않는다.

** ComputeShortestPath

- 현재 구현은 pop 전 저장된 =k_old=를 보존하지 않고 pop 뒤 key를 다시 계산한다.
- =k_old < CalculateKey(u)=이면 새 key로 재삽입하는 원 논문 분기가 빠져 있다.
- loop guard가 =TopKey < CalculateKey(start) || rhs(start) != g(start)=와 동일한지 각
  sub-condition을 독립 test한다.
- retry count로 알고리즘 종료를 성공처럼 숨기지 않고 =LIMIT_REACHED=와 stats를 반환한다.

** 변경 통지와 이동

- D* Lite 이론 입력은 방향 있는 edge cost 변화다. 현재 =changed coord + max_range=는 어떤
  edge가 old/new cost로 바뀌었는지 표현하지 못해 증분 보장을 주장할 수 없다.
- start 이동마다 이전 start 기준 heuristic으로 km를 증가시키고 stored start를 원자적으로
  갱신해야 한다. 현행 normal loop는 local start만 움직여 km/reference가 stale하다.
- core는 sleep/move callback/poll changed list를 하지 않는다. application이 explicit edge
  update와 새 start를 전달하고 =replan=을 호출한다.

** reconstruction과 result

- 현행 reconstruct는 retry limit 전에 goal에 도달하지 않아도 성공으로 표시할 수 있다.
- =(0,0)= sentinel로 predecessor 부재를 표현해 정상 좌표와 충돌한다.
- =find=는 reconstruct bool을 무시하고 =find_full/find_loop=는 결과/status가 없다.
- canonical result는 =OK/NO_PATH/CANCELLED/LIMIT_REACHED/CALLBACK_FAILED=를 구분하고
  owned route와 stats의 유효 조건을 명시한다.

BYUL_API void dstar_lite_request_cancel(
    dstar_lite_t* planner);
BYUL_API bool dstar_lite_is_cancel_requested(
    const dstar_lite_t* planner);
#+end_src

- =out_route=는 success 시 새 owned =route_t*=이고 =route_destroy=로 해제한다. no-path/error
  시 NULL, partial route를 허용하면 명시적 status/flag와 endpoint를 추가한다.
- planner/navgrid 관계는 immutable snapshot, retained handle, caller-borrowed 중 하나로
  확정한다. 현 상태처럼 copy는 grid를 복제하고 destroy는 해제하지 않는 조합은 금지한다.
- cancellation은 atomic flag 또는 callback이며 compute/update/reconstruct의 bounded
  interval마다 poll한다. request 후 reset 전까지 sticky인지 명시한다.

* internalize할 algorithm API

- =calc_key/update_vertex/compute_shortest_route/reconstruct/fetch_next=는 private C++ detail로
  이동하고 status를 내부에서도 보존한다.
- g/rhs는 typed =coord -> scalar= table로 바꾸어 generic =void*= ownership을 없앤다.
- frontier는 atomic by-value entry queue를 사용하고 외부 getter/setter를 제거한다.
- invariant checker를 debug/test build에 제공한다: goal rhs, g/rhs domain, open-list iff
  locally inconsistent, unique coord, key exact order.
- =find_loop/interval/move/changed polling=은 core에서 제거한다. tick/application orchestration
  문서가 explicit replan API를 호출한다.

* callback, thread와 reentrancy

- callback과 userdata를 한 operation에서 교체하고 callback이 받는 userdata identity를
  test한다. 현 구현처럼 저장값 대신 NULL/visited_count를 넘기지 않는다.
- cost는 finite non-negative 또는 blocked sentinel/status를 반환하고 NaN/negative를
  reject한다. heuristic은 finite non-negative, admissible/consistent 요구와 zero fallback을
  문서화한다.
- 같은 planner의 mutation/replan 동시 호출은 busy status로 거부하거나 serialize한다.
  callback 안에서 destroy/rebind/replan은 금지하고 runtime guard를 둔다.
- user callback에서 나온 C++/Python exception은 ABI를 넘지 않고
  =CALLBACK_FAILED=로 변환한다.

* 현재 wrapper의 선언 불일치

handwritten wrapper는 다음 존재하지 않는 native symbol을 호출한다.

- =dstar_lite_get_compute_max_retry= / =set_compute_max_retry=
- =dstar_lite_get_debug_mode_enabled=
- =dstar_lite_calculate_key= (실제 =dstar_lite_calc_key=)
- =dstar_lite_get_update_count_table= 및 add/get/clear 계열
- =dstar_lite_set_map=, =dstar_lite_set_is_blocked_func=

또한 =dstar_lite_reconstruct_route=의 bool을 owned pointer로 감싸고,
=set_frontier=에 Python-owned queue를 넘긴 뒤 양쪽이 destroy하며, borrowed route/grid/frontier
child가 parent를 retain하지 않는다. changed-coords callback의 반환 list finalizer도 native
destroy와 충돌한다. constructor는 start 또는 goal 하나만 주면 해당 값을 조용히 무시한다.

canonical wrapper는 generated header manifest와 native export의 exact intersection만 호출하고,
opaque planner, atomic callback binding, owned route result, typed status/stats만 노출한다.

* 테스트 설계

** 원 논문 trace와 작은 exhaustive grid

- Initialize에서 inserted goal key가 =CalculateKey(goal)=인지 검사한다.
- stale =k_old= 재삽입, overconsistent/underconsistent 두 branch를 독립 fixture로 만든다.
- 작은 directed weighted grid를 exhaustive/reference Dijkstra 재계산과 비교한다.
- edge increase/decrease, obstacle add/remove, start move, goal change마다 incremental 결과와
  full recomputation의 route cost를 비교한다.

** 결과와 failure

- start==goal, blocked endpoint, unreachable, zero-cost edge 정책, duplicate/reversed edge
  update, NaN/Inf/negative callback cost를 검사한다.
- expansion/memory limit, cancel at init/queue/reconstruct/near-complete, callback failure와 OOM을
  주입하고 output preservation을 확인한다.
- reconstruct가 goal에 도달하지 않으면 절대 OK를 반환하지 않는지 검사한다.

** ABI/lifetime

- ABI 1 public struct size/align/모든 field offset과 ABI 2 opaque version mismatch를 검사한다.
- create/copy/reset/navgrid/frontier/route/callback owner graph를 ASan/LSan으로 검사한다.
- static D* module, root shared, installed C17/C++17 SDK, Python wrapper와 export signature를
  동일 manifest로 교차 검사한다.
