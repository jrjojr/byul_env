* Build compile-definition 불변식

- coord module static은 =BYUL_STATIC=으로 resource 구현과 header를 컴파일한다.
- root shared는 동일 source/header inventory를 =BYUL_EXPORTS=로 재컴파일·export하며
  allocator/callback ABI를 root runtime에서 별도로 검증한다.

* 문서 책임과 핵심 판정

=byul/navsys/coord/coord_hash.h=의 generic =coord_t -> void*= map API를 선언 하나씩
재설계한다. 공통 기준은 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이다. 가장 먼저 고칠 P0는
=coord_hash_set=의 소유권 파괴, =coord_hash_copy=의 NULL 치환, 무용량 export와 반환
allocation의 해제 경로 부재다.

* 현재 구현과 업계 비교

- 구현은 =std::unordered_map<coord_t, void*>=와 copy/destroy callback을 가진다.
- =set=은 copy callback을 거치지 않고 raw pointer를 저장하고 기존 값을 destroy하지
  않지만, map 파괴 시에는 configured destroy를 적용한다. leak, invalid delete,
  double-free가 모두 가능하다.
- =insert=는 기존 key의 값을 교체하고 =replace=도 missing key를 삽입한다. 이름의
  insert-only/replace-only 기대와 다르다.
- GLib의 [[https://docs.gtk.org/glib/struct.HashTable.html][GHashTable]]은 opaque table,
  create-time destroy notifier와 insert/replace 의미를 명시한다. iterator는
  [[https://docs.gtk.org/glib/struct.HashTableIter.html][GHashTableIter]]처럼 mutation invalidation을
  분명히 해야 한다.
- 현재 =hash/equal=은 value 내용을 비교하지 않는다. =equal=은 key 집합과
  NULL/non-NULL 여부만 비교해 이름이 의미를 과장한다.

signature 교체는 ABI 2에서만 하며 ABI 1 함수는 안전하게 표현 가능한 동작만 forwarding한다.

* 소유권 모델 결정

** 권장 모델

- create 시 value copy/destroy/equal callback과 userdata를 원자적으로 bind한다.
- entry operation을 =insert_copy=, =replace_copy=, =upsert_copy=로 구분한다.
- 소유권 이전이 정말 필요하면 =insert_take=를 별도 이름으로 제공하고 성공/실패 시
  누가 value를 소유하는지 각각 명시한다. raw =set=이라는 이름은 재사용하지 않는다.
- lookup의 기본은 borrowed =const void*=이며 map의 다음 mutation/destroy 전까지만
  유효하다. NULL value 저장을 허용하면 found bool/status를 별도 반환한다.
- generic =void*= API가 wrapper에서 안전하게 표현되지 않으면 public generic family를
  ABI 2에서 internalize하고 D* Lite용 typed table을 별도 제공하는 선택을 먼저 검토한다.

** copy와 equality

- deep copy는 value copy callback이 있을 때만 허용한다. 없는 경우
  =NAVSYS_STATUS_UNSUPPORTED=로 실패하며 destination을 만들지 않는다.
- key-only 비교는 =coord_hash_equal_keys=로 이름을 좁힌다. full equality는 value_equal
  callback이 bind된 경우에만 제공한다.
- table hash를 제공한다면 equality와 같은 key/value semantics, order-independent 조합,
  process-local 용도임을 명시한다. 그렇지 않으면 현행 =coord_hash_hash=를 deprecated한다.

** iteration과 export

- mutation generation을 table/iterator에 저장하고 stale iterator는
  =NAVSYS_STATUS_CORRUPT_STATE= 대신 별도 invalidated status 후보로 반환한다.
- callback 중 table mutation/reentrant destroy를 금지하거나 snapshot iteration으로
  구현한다. 금지 시 runtime generation check와 문서가 함께 있어야 한다.
- keys/entries export는 caller buffer two-call로 통합한다. 포인터 values snapshot은
  요소의 소유권을 이전하지 않는다.

=coord_hash_insert_take=를 추가한다면 callback binding과 동일 allocator domain의 value만
받을 수 있는지 명시해야 하므로 별도 승인 항목으로 둔다.

* wrapper와 테스트 결함

- Python =values()=는 현재 malloc된 pointer array를 해제하지 않는다.
- =export()=는 caller list에 append하는 native 동작 때문에 기존 key를 중복시킬 수 있다.
- iterator wrapper가 parent를 retain하지 않아 parent close 후 UAF가 가능하다.
- generic Python object handle을 =set=으로 넣으면 native destroy callback과 Python handle
  lifetime이 충돌한다. typed codec/registry 없이 자동 wrapping하지 않는다.
- 기존 test에는 예상 밖 분기에서도 =CHECK(true)=로 통과하는 부분과 crash를 주석으로
  회피한 부분이 있어 failure oracle을 전면 교체한다.
