* 함수 단위 감사

| 단계 | 현재 | 결함/결정 |
|------+-----+-----------|
| validation | odd >=3 후 maze 생성 | maze allocation/hash failure 미검사 |
| start | logical interior 범위 uniform pick | 적절하나 seed를 외부가 지정 못함 |
| kill | shuffled 4 dirs 중 첫 unvisited valid | neighbor uniform, 한 edge씩 tree 확장 |
| hunt | row-major 첫 candidate를 선택, visited neighbor만 random | hunt cell selection은 deterministic first라 미적 bias 존재; 문서에 없음 |
| termination | hunt candidate가 없으면 종료 | visited count가 total인지 postcondition 검사 없음 |
| RNG | =mt19937(time(nullptr))=, =rng()%adj.size= | seed collision/modulo bias |
| result | WALL cell hash insertion | insertion/OOM failure 무시 |
| test | count 범위와 print | phase transition, perfect topology, seed/cancel 없음 |

* 목표 계약과 선택점

- hunt selection policy를 =ROW_MAJOR_FIRST= 또는 =RANDOM_ELIGIBLE= typed option으로 노출할지
  결정한다. 기존 adapter는 row-major first를 유지한다.
- 결과는 perfect maze이나 UST가 아니며 long corridor/bias는 guarantee가 아니라 characteristic로 기록한다.
- visited count가 logical cell count와 같아야 성공한다. 그렇지 않으면 corrupt-state status다.
- cancel은 kill step과 hunt scan chunk마다 poll하며 max_steps에는 neighbor probe/hunt scan 중 무엇을
  세는지 명시한다.
