* 대상과 원전 기준

대상은 =sma_star.h/.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]],
Russell의 [[https://people.eecs.berkeley.edu/~russell/publications.html][Efficient Memory-Bounded

forgotten child, backed-up f-value, regeneration과 explicit memory bound를 기준으로 한다.

* P0 Algorithm Identity 결함

현재 구현은 일반 A* frontier에서 =cost_coord_pq_trim_worst()=로 일부 queue item만 제거한다.
=cost_so_far=와 =came_from= hash는 계속 증가하고 forgotten child의 f-value를 parent에 backup하거나
재생성하지 않는다. 따라서 total memory가 =memory_limit=으로 제한되지 않으며 원전 SMA*의
completeness/optimality 조건을 주장할 수 없다. Conformance 전 operational capability는 false로 둔다.

추가 차이:

- =memory_limit=은 node/byte/전체 workspace 중 무엇인지 불명확하고 frontier length만 센다.
- cost/heuristic userdata가 없고 numeric/callback failure를 구분하지 않는다.
- Header의 map-size별 권장값과 수식은 stable 근거가 아닌 tuning 정보다.
- Budget 소진 시 frontier의 peek를 partial endpoint로 쓰지만 best-known/goal-nearest 보장이 없다.
- NULL, nonpositive memory/retry, OOM/no-path/limit가 NULL/false에 합쳐진다.

Node count와 byte workspace 중 normative limit를 하나로 정하거나 둘의 precedence를 명시한다.
Stats는 current/peak nodes와 bytes, pruned/regenerated count, incumbent/lower bound와 종료 이유를
포함한다. Caller workspace/allocator가 bound를 우회하지 않아야 한다.
