* 대상과 설계 방향

- 대상: =byul/navsys/obstacle/obstacle.h/.cpp=과 obstacle tests, maze/route-carver consumers.
- 이 header는 rectangle/random/line/ring/enclosure/cross/spiral/polygon raster generator 모음이다.
  shape geometry와 grid rasterization policy를 명시적으로 분리해야 한다.
- 공통 =navsys_status_t=, options, cancel과 output strong guarantee는
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이 개별 raw =int= 제안보다 우선한다.
- raster 기준은 [[https://gdal.org/en/stable/api/gdal_alg.html][GDAL rasterize C API]]의 center/all-touched 구분과
  [[https://www.boost.org/doc/libs/1_87_0/libs/geometry/doc/html/geometry/reference/algorithms/covered_by/covered_by_2.html][Boost.Geometry covered_by]]의 boundary-inclusive predicate를 비교 축으로 삼는다.

* 함수별 구현 결함과 목표 계약

** Rectangle/random

- filled rectangle은 width/height >0만 검사하고 multiplication/hash insert failure를 무시한다.
- random rectangle은 probability =0=을 invalid/NULL로 처리하고 >1을 조용히 1로 clamp한다.
  Target은 =[0,1]=을 모두 valid로 하며 outside/NaN은 INVALID_ARGUMENT로 거부한다.
- =srand(time)/rand=는 global/thread-unsafe/non-replay다. per-call RNG와 seed provenance로 교체한다.
- 현재 test는 probability 0에서 NULL을 요구하면서 NULL blocked set length도 호출한다. 신규 contract와
  논리 assertion으로 교정한다.

** Beam/line

- extent를 =goal-start=로 만들어 inclusive endpoint와 width/height가 불일치하고 reverse line은
  negative extent가 된다.
- range 0은 start를 제외하고 goal까지 block한다. range >0은 =range-1=로 neighborhood를 구해
  radius 1에서 centerline을 제외하고 주변 이웃만 block하는 등 이름과 다른 hollow 결과가 가능하다.
- Target은 start/end inclusive Bresenham/supercover 선택, radius 0=centerline, radius N의 metric
  (Chebyshev/Manhattan/Euclidean disk)을 typed option으로 고정한다.

** Rect outline/enclosure/cross

- =torus=는 3D torus가 아니라 axis-aligned rectangular ring이다. canonical 명명만 고치고 legacy 유지한다.
- enclosure는 문서의 minimum interior 크기를 검사하지 않는다. open side corner가 다른 side 조건으로
  다시 막힐 수 있어 aperture 범위가 불분명하다.
- thickness/range는 실제로 cell center 주변 square radius여서 선 폭은 =2r+1=이다. thickness가
  one-sided width인지 radius인지 모든 shape에서 통일한다.

** Spiral

- radius는 output metadata extent에만 쓰이고 실제 step 종료에는 사용되지 않는다. =turns*4=와 증가하는
  leg length가 radius 밖 좌표를 block할 수 있다.
- =gap=은 ring spacing이 아니라 step index에 따라 entire leg를 skip한다. direction unknown은 silently CW다.
- Target descriptor는 max radius, pitch/ring spacing, turns 또는 endpoint 중 authoritative parameter,
  raster radius와 clip policy를 분리한다.

** Triangle/polygon

- triangle barycentric dot product가 =int=에서 overflow하고 float division으로 large coordinate precision을 잃는다.
- degenerate triangle은 NULL이 아니라 empty obstacle로 성공한다. boundary inclusion도 불명확하다.
- polygon은 non-const list, self-intersection/duplicate/null vertex/orientation/fill/boundary contract가 없다.
- ray crossing은 even-odd처럼 동작하지만 boundary cell을 안정적으로 포함하지 않는다.
- outline legacy 이름 =*_torus=는 의미가 틀리며 line dilation이 square radius임을 밝히지 않는다.
- Target은 checked 64-bit/wider orientation, validity status, explicit fill/raster rule과 contiguous
  =vertices+count= input을 사용한다. self-intersection support 여부는 capability로 보고한다.
