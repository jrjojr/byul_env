* 대상과 핵심 판정

- 대상: =navgrid.h/.cpp=, navgrid static/root inventory, tests, =byul_wrapper/navgrid.py=와
  route-finder/D* Lite/console/maze/obstacle consumers.
- =navgrid_t=는 owning hash와 retained callback을 가진 resource다. 업계식 목표는 opaque handle이다.
  ABI 1.x에서는 accessor/bind/buffer API만 additive하고 ABI-major에서 public layout을 숨긴다.
- 공통 status/options/callback/two-call buffer 모델은
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 선행 decision gate로 삼는다.

* 현행 구현 P0/P1 결함

| 영역 | 구체 사실 | 영향 |
|------+-----------+------|
| create | userdata field를 초기화하지 않음 | public layout에 indeterminate pointer 노출 |
| callback call | =navgrid_copy_neighbors=가 항상 userdata =nullptr= 전달 | 저장 field와 Python userdata contract가 작동하지 않음 |

| copy binding | Python callback pointer만 native copy | Python closure lifetime/owner와 copied grid가 분리될 수 있음 |
| bounds | width/height 0은 unbounded, negative도 반대 방향 extent로 취급 | origin 없이 negative dimension 의미가 비표준/모호 |
| resize | set width/height가 field만 변경 | out-of-bounds cells를 남기고 no status |
| unblock | NORMAL cell을 map에 replace | 기존 terrain/height 소실, sparse map 증가, 다른 overlay provenance 제거 |
| set cell | =coord_hash_replace= result를 무시하고 true 반환 | OOM/replace failure 은폐 |
| fetch | =out=NULL= 미검사 | absent와 invalid output이 모두 unsafe/=-1= |
| built-in query | fetch result를 무시 | corrupt/failed cell lookup을 traversability로 오인 |
| neighbor | allocation/push failure 미검사 | partial list success |

| diagonal | 8-way에서 corner-cut policy 없음 | path가 두 blocked orthogonal cell 사이를 통과 가능 |
| angles | degree zero/clockwise/axis가 coord implementation에 암묵적 | SDK contract 불충분 |
| range | Chebyshev square에서 각 cell의 neighbors union | 이름/주석의 range=0 semantics 외 metric/center inclusion 불명확 |

* 업계 비교와 목표 모델

- [[https://docs.godotengine.org/en/stable/classes/class_astargrid2d.html][Godot AStarGrid2D]]는 region, solid, weight, diagonal mode를 분리하고 네 가지 corner
  정책을 명시한다. 단순 4/8 enum만으로 traversal topology가 완성되지 않는다.
- [[https://recastnav.com/classdtNavMeshQuery][Detour query API]]는 caller buffer/capacity/count와 status/partial을 쓴다. neighbor마다
  heap list를 반환하는 API 대신 two-call/caller-buffer가 canonical이어야 한다.
- [[https://www.glfw.org/docs/latest/window.html][GLFW user pointer/callback]]은 userdata 초기값과 object lifetime을 명시한다. callback과
  userdata는 하나의 bind operation으로 교체해야 한다.
- [[https://www.sqlite.org/c3ref/sqlite3.html][SQLite opaque handle]]처럼 owning map/callback layout을 SDK에서 숨긴다.

=out=NULL,capacity=0= count query와 undersized =INCOMPLETE= 의미는 공통 문서와 동일해야 한다.

않는다. callback thread/reentrancy 및 callback 중 mutation/destroy 금지를 명시한다.
