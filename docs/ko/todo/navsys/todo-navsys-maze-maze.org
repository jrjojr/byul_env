* 대상과 결론

- 대상: =byul/navsys/maze/maze.h=, =maze.cpp=, maze CMake inventory와 native consumer.
- 역할: 11개 generator를 선택하는 public C aggregate/dispatcher이다.
- 판정: enum ABI와 기존 =maze_make= symbol은 보존하되, silent fallback을 제거할 수 있는
  status/out-parameter/options 기반 API를 먼저 추가한다. 기존 함수의 signature를 직접
  바꾸지 않는다.
- =maze_type_t=의 현재 값 0..10은 binary ABI fixture로 동결한다. 새 enum member를 중간에
  삽입하지 않는다.
- 공통 status/options/callback 계약은
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이 우선한다. 아래
  =navsys_status_t=와 symbol prefix는 공통 결정 전의 구체 검토안이다.

* 책임 경계와 실제 검증 파일

- 이 문서와 =maze.cpp=는 enum-to-generator selection, 공통 options 검증·routing,
  unsupported/fallback 판정과 11개 generator capability의 합류를 소유한다.
- 개별 algorithm state machine, RNG 소비·seed/replay와 generation loop 구현은 각 generator
  전용 TODO가 소유한다. dispatcher는 승인된 context를 전달할 뿐 그 구현을 복제하지 않는다.
- =maze_core.cpp=의 resource/extent/enumeration/overlay primitive는
  [[file:todo-navsys-maze-core.org][maze core TODO]]가 소유하며 dispatcher가 재구현하지 않는다.
- 실제 maze native 검증 파일은
  =byul/navsys/maze/tests/test_module_maze.cpp=,
  =byul/navsys/maze/tests/test_module_maze_discrete.cpp=,
  =byul/navsys/maze/tests/test_module_maze_room.cpp=다. 이 문서는 세 파일의 dispatcher
  selection/capability 통합 case를 합류 검증한다.

* 업계 관행과 비교 기준

- [[https://www.boost.org/doc/libs/1_48_0/libs/graph/doc/random_spanning_tree.html][Boost.Graph random_spanning_tree]]는 graph, caller 제공 RNG, root와 output map을
  분리하고 연결성 precondition 및 stuck failure를 명시한다. BYUL dispatcher는 RNG와 실패
  채널이 모두 숨겨져 있다.
- [[https://recastnav.com/classdtNavMeshQuery][Recast Detour query API]]는 status, caller buffer/capacity, partial result와 sliced
  progress를 구분한다. 긴 maze generation에도 status/cancel/progress 계약을 적용한다.
- [[https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2016/p0205r0.html][WG21 P0205R0]]는 deterministic seed sequence와 nondeterministic seed source를 별도
  개념으로 다룬다. =time(NULL)=을 각 구현에 숨기는 현재 방식은 재현·병렬성 관행과 맞지 않는다.

Options 후보에는 =struct_size=, =abi_version=, =uint64_t seed=, seed 사용 여부,
=uint64_t max_steps=, cancel callback와 =void* userdata=를 둔다. 성공 때만
=*out_maze=가 caller-owned가 되며 실패 때는 NULL이어야 한다. callback은 호출 thread,
retention 없음, reentrancy 금지/허용을 명시한다.

* 호환성과 ABI-major 경로

1. 현 enum numeric fixture와 export/signature를 먼저 동결한다.
2. additive =byul_maze_generate=와 support query를 구현한다.
3. =maze_make=는 새 함수로 forwarding하되 기존 nondeterministic seed 정책을 유지하고
   invalid enum은 과거 fallback 호환 여부를 release policy로 결정한다. 동작 변경이
   허용되지 않으면 legacy adapter만 Kruskal fallback을 유지하고 명확히 deprecate한다.
4. wrapper는 새 status API를 기본으로 노출하고 legacy는 compatibility namespace에 둔다.
5. enum storage와 =maze_t= opaque 전환은 ABI-major에서만 한다.
