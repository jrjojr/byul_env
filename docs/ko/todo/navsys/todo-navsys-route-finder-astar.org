* 대상과 비교 기준

대상은 =byul/navsys/route_finder/astar.h=와 =astar.cpp=다. 공통 API 형태는
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다.
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]]의
non-negative weight, distance/predecessor map, visitor event와 complexity 계약 및
[[https://doi.org/10.1109/TSSC.1968.300136][Hart/Nilsson/Raphael 원 논문]]의 admissible/
consistent heuristic 조건을 correctness 기준으로 사용한다.

* 실행 소유권과 테스트 위치

- 이 문서는 A* search 구현·conformance와 A* 전용 config/default/validation만 소유한다.
- 공통 evaluator와 callback binding은
  [[file:todo-navsys-route-finder-route-finder-core.org][evaluation 문서]], facade/options/
  capability와 config 합류는
  [[file:todo-navsys-route-finder-route-finder.org][dispatcher 문서]]가 소유한다.
  이 문서는 받은 evaluation/options를 소비하고 공통 callback 또는 facade를 다시 정의하지
  않는다.
- A* direct/conformance assertion의 실제 파일은 shared native test
  =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=다.
  Dispatcher 및 root consumer assertion은 각각
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=와
  =byul/navsys/tests/test_module_navsys.cpp=에서 해당 owner가 추가한다.
- A* 문서는 Python에 필요한 config/status 기대값을 제공한다. 실제 wrapper consumer
  assertion은 dispatcher/root 통합이
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 현재 구현/소비 분석

- module static/root shared/SDK install과 generated CFFI =route_finder.py=가 이 header의
  유일한 함수를 공개 ABI로 소비한다. High-level wrapper는 직접 호출하지 않고 dispatcher를
  경유한다.
- 구현은 cost/heuristic userdata를 받을 parameter 자체가 없어 항상 =nullptr=를 전달한다.
- =max_retry <= 0=, NULL input은 모두 NULL을 반환해 invalid/limit/OOM을 구분하지 못한다.
- 예산 소진 시 마지막 pop node까지 partial route를 만들지만 header에는 endpoint,
  optimality와 ownership 외에 종료 이유가 없다. =route.cost=도 계산하지 않는다.
- 기본 =default_cost=는 8방향 모두 1이고 기본 Euclidean heuristic은 대각선 한 step의 실제
  cost 1보다 sqrt(2)를 추정한다. 8방향 grid에서 heuristic이 과대평가되어 현재 default의
  최단 cost 보장은 성립하지 않는다.
- negative/NaN/Inf cost, inconsistent heuristic, stale queue entry, OOM과 callback failure를
  검사하지 않는다. =new/delete=와 BYUL destroy family도 혼용한다.

* 전체 Public Symbol Inventory와 disposition

=ASTAR_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_ASTAR_H=로 교체한다.

| 현재 symbol | 현재 signature | wrapper | disposition |
|-------------+------------------+---------+-------------|
| =find_astar= | =route_t*(grid,start,goal,cost_fn,heuristic_fn,int,bool)= | generated CFFI 등록 | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= |

=find_astar=는 unprefixed root DLL symbol이다. ABI 1에서는 유지하되 canonical generic
=route_finder_run_ex= 또는 아래 explicit adapter를 호출하게 하고 ABI 2에서 direct algorithm
header를 SDK에서 제외한다.

* 목표 Signature 후보

#+begin_src c
BYUL_API navsys_status_t route_finder_find_astar_ex(
    const navgrid_t* grid,
    const coord_t* start,
    const coord_t* goal,
    const route_finder_evaluation_view_t* evaluation,
    const navsys_search_options_t* options,
    route_t** out_route,
    navsys_search_stats_t* out_stats);
#+end_src

- =evaluation=은 cost/heuristic와 각 userdata의 call-only view다. Finder에 bind한 형태를
  canonical으로 선택하면 explicit 함수는 제공하지 않고 generic run만 공개한다.
- status는 complete/no-path/partial/limit/cancel/callback/numeric/OOM을 구분한다.
- 최적성은 모든 edge cost가 finite/non-negative이고 heuristic이 같은 unit에서 admissible,
  graph-search 재개방 정책과 consistent 조건을 만족할 때만 주장한다.
- =out_route=는 usable outcome에서 caller-owned이고 =route_destroy=로 해제한다.
