* 대상과 기준

- 대상: =maze_wilson.h/.cpp=.
- primary 기준은 [[https://dl.acm.org/doi/10.1145/237814.237880][Wilson, Generating Random Spanning Trees More Quickly than the Cover Time]]과
  established implementation인 [[https://www.boost.org/doc/libs/1_48_0/libs/graph/doc/random_spanning_tree.html][Boost.Graph random_spanning_tree]]다.
- 목표 결과는 rectangular 4-neighbor graph의 uniform spanning tree다. 단순히 random walk를
  저장하는 것이 아니라 loop-erased random walk임을 구현/검증해야 한다.
- 공통 status/options/cancellation은
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따르며 아래 signature보다 우선한다.

* 현행 loop-erasure 감사

| 단계 | 현재 | 위험 |
|------+-----+------|
| seed | =mt19937(time(nullptr))= | 재현 불가/동시 호출 seed collision |
| root | all_cells 중 random 하나를 visited | 적절하나 provenance 없음 |
| path | =unordered_map<Cell,Cell>= successor overwrite | loop가 생길 때 stale successor를 명시적으로 erase하지 않아 canonical loop-erased path인지 증명/검사가 필요 |
| walk | 매 step 4 directions shuffle, 첫 valid | 균등 neighbor지만 max step/cancel 없음 |
| commit | =path[walk]=로 다음 cell 조회 | missing key면 default Cell 삽입 가능; invariant 위반을 숨김 |
| hash | custom =std::hash<Cell>= | collision 허용은 되나 deterministic iteration에 의존하면 안 됨 |
| allocation | maze/vector/set/map/hash insert | 모든 failure/exception이 C ABI 밖으로 전파될 수 있음 |
| test | non-NULL/block count/print | loop erasure, UST, exception barrier를 검사하지 않음 |

* 목표 구현 선택

- current walk를 ordered vector와 =cell -> position= index map으로 유지한다. 재방문 시 해당
  position 뒤를 즉시 truncate하여 loop erasure를 구조적으로 표현한다.
- walk가 기존 tree를 만났을 때만 path를 commit한다. commit 전에는 =visited=와 maze output을
  변경하지 않아 cancel/OOM rollback이 가능해야 한다.
- C++ allocation/exception은 catch되어 defined negative status로 변환되어야 한다.
- topology/seed/cancel/step budget은 Aldous-Broder와 같은 공통 options를 사용한다.
