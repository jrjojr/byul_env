* 대상과 구조 판정

- 대상: =byul/navsys/maze/maze_core.h=과 =maze_core.cpp=.
- 현재 =maze_t=는 origin/extent와 owning =coord_hash_t*=를 public field로 노출한다.
- 목표는 업계식 opaque owned handle이다. 다만 field 제거는 binary/source ABI break이므로
  accessor 기반 additive 이행 후 ABI-major에서만 =typedef struct maze maze_t;=로 전환한다.
- 내부적으로 obstacle과 중복된 rectangular coordinate-mask engine을 공유할 수 있지만 public
  maze/obstacle type과 symbol을 합치지 않는다.
- 공통 status, opaque lifecycle, caller-buffer와 overlay 계약은
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이 우선한다. 아래
  =navsys_status_t= signature는 그 공통 결정에 종속된다.

* 현행 구현 불변식 감사

| 영역 | 현행 | 구체 결함 |
|------+-----+-----------|
| allocation | =malloc(maze_t)= 후 =coord_hash_create= | hash allocation 실패를 검사하지 않아 반쯤 생성된 handle 반환 |
| dimensions | 어떤 signed 값도 허용 | generator 요구 odd >=3 및 overflow 계약과 불일치 |
| clear | blocked set만 비움 | origin/extent 유지 여부는 문서 없음 |
| copy | 새 hash를 만들었다가 destroy하고 다시 copy | 임시 allocation 낭비, create/copy 실패 dereference 가능 |
| origin setter | x0/y0 field만 변경 | 이미 저장된 world blocked coordinate를 translate하지 않아 metadata와 payload 분리 |
| width/height | getter만 있으나 public field로 수정 가능 | blocked 좌표와 bounds invariant를 강제할 수 없음 |
| blocked getter | borrowed internal hash 반환 | lifetime/invalidation/thread 계약과 안전한 iteration 없음 |
| apply | navgrid가 작으면 width/height만 확대 | nonzero origin을 무시하고 부분 실패/status가 없으며 기존 terrain을 덮음 |
| remove | 모든 blocked coordinate를 normal로 설정 | 다른 obstacle/source가 막은 셀까지 제거할 provenance 위험 |
| equality/hash | origin/extent/hash 결합 | NULL과 hash collision/stability/version 계약 없음 |

* 업계 관행 비교

- [[https://www.sqlite.org/c3ref/sqlite3.html][SQLite connection handle]]은 =typedef struct sqlite3 sqlite3= 형태의 opaque object와
  명시적 constructor/destructor를 사용한다. public owning hash field는 이 ABI 안정성 관행과 반대다.
- [[https://recastnav.com/classdtNavMeshQuery][Detour query API]]는 status와 caller buffer/capacity, partial/buffer-too-small을 명시한다.
  raw internal hash를 빌려 주는 대신 count/fetch API가 필요하다.
- [[https://docs.godotengine.org/en/stable/classes/class_astargrid2d.html][Godot AStarGrid2D]]는 region, solid state, update와 query를 구분하고 diagonal 정책도
  명시한다. maze apply가 navgrid 크기를 몰래 바꾸는 동작은 분리해야 한다.

=fetch=는 capacity가 작아도 required count를 반환하고 destination beyond-count를 건드리지
않는다. =translate=는 overflow 때 원본을 보존한다. apply는 all-or-nothing 또는 명시적
partial-result flag 중 하나를 택한다. overlay token은 navgrid 생존 기간에만 유효하다.

* Storage·copy·thread 계약 목표

| 계약 | 목표 |
|------+------|
| zero | public opaque handle에는 zero-init 사용 금지; =NULL=만 empty handle |
| create/destroy | library allocation은 =maze_destroy=만 해제 |
| copy | origin/extent/blocked set deep copy, source와 독립 mutation |
| borrowed data | 신규 API는 internal =coord_hash_t*=를 반환하지 않음 |
| invalidation | mutation 후 이전 iterator/view/overlay 준비 상태의 유효성 명시 |
| thread | 서로 다른 handle은 병렬 사용 가능; 같은 mutable handle은 외부 동기화 |
| hash | 동일 process/version 내 equality와 일치; 저장·network ABI fingerprint로 사용 금지 |
| layout | ABI-major 전 size/align/offset fixture 보존; 이후 opaque size query 불필요 |
