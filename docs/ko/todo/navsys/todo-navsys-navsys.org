* 문서 책임

대상은 =byul/navsys/navsys.h= 하나다. 구현 =navsys.cpp=, 12개 one-shot helper가
호출하는 route-finder/D* Lite, module/root CMake inventory, =byul/byul.h=, native test,
README와 설치 SDK consumer를 함께 다룬다. 공통 status/options/stats, cancel,
allocator/workspace, capability와 ABI/static/shared/wrapper gate는
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다.

현재 header는 module umbrella와 search facade 역할을 동시에 한다. 리팩터링의 핵심은
=navsys.h=를 작은 안정 search facade로 만들고, 모든 public component를 한꺼번에 포함하는
편의가 정말 필요하면 별도 =navsys_all.h= 같은 명시적 umbrella로 분리하는 것이다.

* 실행 소유권과 선행 Gate

- 이 문서는 42개 선행 header의 승인 결과를 합치는 facade/aggregate owner다. Route Finder,
  D* Lite, Maze와 collection 구현을 직접 고치거나 미완료 child의 API 결정을 선점하지 않는다.
- 상위 orchestration TODO 0~5와 공통 Status/Result/Capability harness가 =DONE=인 뒤에만 이
  문서의 TODO 1을 시작한다.
- 기존 native 기준 test는 =byul/navsys/tests/test_module_navsys.cpp=다. Facade 동작과
  legacy/canonical pair assertion은 이 파일에 추가하고, installed SDK C/C++ fixture와
  wrapper test는 각각 담당 test tree에서 별도 실행한다.
- 선행 algorithm이 =conformance-block=이면 facade capability도 unsupported/experimental로
  합류한다. Aggregate가 구현 유무나 보장 수준을 독자적으로 승격하지 않는다.

* 조사한 현재 소비·배포 경로

- =byul/navsys/CMakeLists.txt=가 =navsys.cpp=를 module static에 넣고 =byul/CMakeLists.txt=가
  같은 source를 root shared에 직접 넣는다. =navsys.h=는 module 설치 header이며
  =byul/byul.h=가 다시 include하므로 root SDK에서도 넓게 전파된다.
- =byul/navsys/tests/test_module_navsys.cpp=는 12개 helper를 각각 한 번 호출해 route를
  출력/파괴한다. status 분류, option boundary, large grid, NULL/OOM/cancel, algorithm
  correctness와 shared-library 경로는 다루지 않는다.
- =docs/ko/README.ko.md=는 =#include "navsys.h"=와 =navsys_find_astar=를 입문 예제로
  고정한다. source migration을 위해 canonical 예제를 먼저 추가해야 한다.
- Python wrapper는 현재 =navsys.h=와 12개 helper를 등록하지 않는다. Python 사용자는
  lower-level =RouteFinder=만 사용하므로 root public facade와 wrapper capability가 다르다.
- repository 외부 consumer는 =navsys.h=의 transitive include에 의존할 수 있다. include를
  줄이는 것은 binary ABI 변화가 아니어도 source compatibility 변화이므로 install-consumer
  fixture와 migration window가 필요하다.

* 업계 기준과 직접 비교

- [[https://recastnav.com/classdtNavMeshQuery.html][Detour dtNavMeshQuery]]는 query object,
  =dtQueryFilter=, node pool과 caller result buffer를 분리하고 상세 =dtStatus=를 반환한다.
  BYUL one-shot facade도 algorithm, policy, budget, workspace와 결과/status를 서로 독립된
  versioned input/output으로 전달해야 한다.
- [[https://recastnav.com/DetourStatus_8h_source.html][Detour status]]는 invalid parameter,
  out-of-memory/out-of-nodes, partial result와 buffer-too-small을 no-path와 분리한다.
  현재 owned route 또는 NULL 하나인 반환 계약을 교체할 직접 기준이다.
- [[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]]와
  [[https://www.boost.org/doc/libs/latest/libs/graph/doc/dijkstra_shortest_paths.html][Dijkstra]]는
  weight/heuristic, predecessor/distance map, visitor와 color/workspace를 명시적 parameter로
  다룬다. 알고리즘 이름만 다른 12개 wrapper에 숨은 fixed default를 복제하지 않는다.
- [[https://idm-lab.org/bib/abstracts/papers/aaai02b.pdf][Koenig & Likhachev, D* Lite]]는
  변화된 edge cost를 반영하는 incremental replanning algorithm이다. 한 번 create/find/destroy하는
  =navsys_find_dstar_lite=는 주 사용 모델을 잃으므로 one-shot convenience와 persistent
  replanner를 capability/documentation에서 구분한다.
- [[https://curl.se/libcurl/c/][libcurl easy interface]]처럼 편의 facade라도 handle/result
  ownership, option lifetime와 오류 code를 안정 C ABI로 노출하고 C++ exception이 C 경계를
  통과하지 않게 한다.

* 확인된 P0/P1 구현·계약 결함

1. A*부터 weighted A*까지 11개 helper가 stack public =route_finder_t=,
   =route_finder_init_full=와 fixed =MAX_RETRY= 1000을 복제한다. map 규모와 관계없이 expansion
   budget이 잘리고 limit/no-path/OOM이 owned route/NULL로 합쳐진다.
2. start가 NULL이면 조용히 =(0,0)=으로 바뀌지만 goal NULL은 실패한다. implicit origin은 typo를
   성공한 query로 만들며 start/goal의 validation 대칭성이 없다.
3. grid가 NULL이거나 goal이 NULL인 경우 NULL, unreachable/invalid config/allocation/callback
   failure도 NULL 또는 unsuccessful route다. caller는 실패 원인과 partial route의 usability를
   판별할 수 없다.
4. A* default가 Euclidean heuristic이고 current 8-direction grid에서 모든 move cost가 1이면
   diagonal distance를 과대평가할 수 있다. 이름만 =astar=인 helper가 최적성을 보장하지 않는다.
5. Fringe/RTA/SMA/Weighted helper는 scalar config 한 개를 =new=로 heap allocation한 뒤
   raw =typedata=로 넘긴다. 불필요한 allocation이며 =new= failure가 C ABI 밖으로 예외를
   던질 수 있다.
6. =route_finder_init_full=이 userdata를 저장하지 않고 algorithm adapters도 NULL userdata를
   전달하는 lower-level P0가 one-shot helper에도 그대로 전파된다.
7. D* Lite helper만 grid/start/goal NULL을 먼저 검사하지 않고 =dstar_lite_create_full=의
   NULL 반환도 확인하지 않은 채 =dstar_lite_find(dsl)=를 호출한다. crash와 OOM이 가능하다.
8. RTA*/SMA*/Fringe helper의 실제 algorithm 구현은 각 원전의 completeness/memory/realtime
   보장을 충족한다고 보기 어렵다. facade가 algorithm 이름을 unconditional support처럼
   export하여 capability와 semantic honesty가 어긋난다.
9. =navsys_find_fringe_search/rta_star/weighted_astar= Doxygen은 존재하지 않는 parameter =a=와
   =userdata=를 설명하고 실제 =ng/start/goal/config=의 ownership/range를 설명하지 않는다.
   SMA 수식도 =/===/=, =/x/=, =/<==/= 표기가 섞여 executable contract가 아니다.
10. query input grid constness/borrowed duration, returned route owner/destroy 함수, thread safety,
    callback thread, reentrancy, deterministic tie, cost/heuristic 단위와 partial result가 모두
    문서에 없다.
11. header guard =NAVSYS_H=는 generic하고 file-level role/comment 및 12개 함수의 일관된
    documentation이 없다. facade가 내부 header를 transitively include해 compile coupling과
    SDK surface를 과도하게 키운다.
12. Python wrapper에 facade declaration/status/config/capability가 없어 native API와 언어 API가
    서로 다른 지원 집합과 default를 갖게 될 위험이 있다.

** 12개 함수별 disposition

모든 legacy helper는 ABI 1.x에서 유지한다. canonical =navsys_find_path=에 완전히 표현 가능한
helper는 deprecate-forward하고, 구현 보장이 이름과 다른 algorithm은 forwarding 전에 해당
header의 correctness gate를 통과하거나 capability를 experimental/unsupported로 낮춘다.

| # | 현재 함수 | disposition | 목표 forwarding/config 계약 |
|---+-----------+-------------+------------------------------|
| 1 | =navsys_find_astar= | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= | =algorithm=ASTAR; movement model에 맞는 admissible default 또는 명시 heuristic |
| 2 | =navsys_find_bfs= | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= | =algorithm=BFS; unweighted/unit-edge capability를 validation |

| 4 | =navsys_find_dijkstra= | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= | =algorithm=DIJKSTRA; finite non-negative edge cost 검증 |

=conformance-block=은 legacy export 제거 처분이 아니다. 위 6개 함수는 ABI 1.x에서
=deprecate-forward=로 보존하되 capability query, 문서와 wrapper가 algorithm guarantee를
과장하지 않으며 correctness gate 전에는 canonical supported path로 승격하지 않는다.

* 목표 facade와 Signature 후보

#+begin_src c
typedef struct s_navsys_path_query {
    uint32_t struct_size;
    uint32_t struct_version;
    navgrid_t* grid;                  /* borrowed for call duration */
    coord_t start;
    coord_t goal;
    route_finder_type_t algorithm;
    const navsys_algorithm_config_t* algorithm_config;
    navsys_search_options_t options;
} navsys_path_query_t;

BYUL_API navsys_status_t navsys_path_query_init(
    navsys_path_query_t* query,
    size_t query_size);

BYUL_API bool navsys_is_algorithm_supported(
    route_finder_type_t algorithm);
BYUL_API navsys_status_t navsys_get_algorithm_descriptor(
    route_finder_type_t algorithm,
    navsys_algorithm_descriptor_t* out_descriptor);

BYUL_API navsys_status_t navsys_find_path(
    const navsys_path_query_t* query,
    route_t** out_route,

=navsys_find_path=는 grid/start/goal을 모두 필수로 하고 implicit origin을 허용하지 않는다.
호출 동안 grid/config를 borrowed하고 반환 =*out_route=는 caller가 =route_destroy=하는 owning
handle이다. 실패 전 =*out_route=는 NULL이어야 하며, partial output이 허용되는 status만
명시적으로 route를 전달한다. options는 max expansions/time, cancel token, trace policy,
allocator/workspace와 deterministic tie를 표현한다.

=navsys_algorithm_config_t=는 untyped =void*=가 아니라 tag와 =struct_size/version=을 가진
typed configuration family다. algorithm과 config tag 불일치는 =INVALID_ARGUMENT=,
unknown future tail은 size/version 규칙에 따라 무시 또는 =UNSUPPORTED_VERSION=으로 처리한다.
descriptor는 supported/experimental, complete/optimal/bounded-suboptimal claim, required config,
grid connectivity/cost restrictions와 incremental 여부를 machine-readable하게 반환한다.

* Legacy default와 의미 migration

- legacy start NULL → origin은 ABI 1 forwarding에서만 유지하고 deprecation diagnostic 및
  migration note를 제공한다. canonical query는 start를 value로 필수화한다.
- legacy =MAX_RETRY= 1000도 forwarding compatibility profile에만 남긴다. canonical default는
  unlimited가 아니라 documented budget profile/query이며 종료 stats에 applied limit를 기록한다.
- legacy helper의 owned route/NULL 계약은 유지하되 thread-local error 같은 숨은 채널을 새로
  만들지 않는다. 새 status API가 canonical이고 wrapper는 legacy helper를 사용하지 않는다.
- A*/FMM/IDA*/RTA*/SMA*/Fringe의 의미 교정으로 path가 바뀔 수 있음을 behavior change로
  release note에 기록하고 old/new deterministic fixture를 함께 제공한다.

* ABI, static/shared와 Wrapper 전환

- ABI 1.x에서 12개 symbol/signature를 보존하고 =navsys_find_path=와 capability/descriptor를
  additive export한다. legacy 구현은 중복 setup을 제거하고 하나의 compatibility config
  constructor를 통해 canonical core로 forwarding한다.
- facade type은 version/size를 첫 field로 두고 reserved field 0 규약을 사용한다. ABI 2 전까지
  기존 helper signature나 returned owner를 바꾸지 않는다.
- module static와 root shared가 동일 source/header/compile-definition inventory를 사용하도록
  shared object/source inventory를 통합하고 12 legacy + canonical symbol의 exact-once export를
  검사한다.
- wrapper declaration은 public header에서 생성하고 capability matrix/enum/status/config를 함께
  등록한다. high-level =find_path=는 owned Route와 typed result/exception을 반환하며 Python
  default가 native canonical default와 fixture로 일치해야 한다.
- umbrella 분리는 source compatibility migration을 거친다. =byul/byul.h=가 facade만 포함할지
  explicit =navsys_all.h=까지 포함할지는 compile-time/SDK coupling을 측정해 별도 decision으로
  확정한다.
