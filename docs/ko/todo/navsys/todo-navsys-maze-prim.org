* 구현 결함 감사

| 위치/단계 | 현재 | 위험 |
|-----------+-----+------|
| precondition | validation 없이 =maze_create_full= 및 =vector(h,w)= | 음수→huge size, even/small input, exception/C ABI 탈출 |
| empty start | maze allocation 후 =odd_x/odd_y.empty=면 NULL | 이미 생성한 maze leak |
| RNG | =mt19937(time(nullptr))= | 재현 불가/초 단위 collision |
| frontier | wall cell vector, random index 후 O(n) erase | duplicate walls 가능, O(E²) worst-case |
| opposite test | passage/WALL 양쪽 검사 | frontier 의미는 맞지만 logical visited set이 없어 duplicate/ambiguous state |
| neighbor enqueue | =nx=by+dy, =ny=bx+dx= 후 인자를 다시 swap | x/y 교차 코드는 검증이 어렵고 rectangular grid에서 오류 가능성이 높음 |
| result | blocked insert failure 무시 | partial/incomplete maze 성공 반환 |
| test | block count 범위 | connectivity/no cycle/rectangular orientation/invalid input 없음 |

* 목표 알고리즘

- logical visited cell set과 frontier edge =(visited, unvisited, wall)=를 명시적으로 둔다.
- random frontier edge를 선택해 unvisited vertex를 정확히 하나 추가한다. duplicate frontier는
  indexed bag/set으로 제거하거나 stale edge를 skip하며 stats로 관찰한다.
- result는 perfect maze, =V-1= accepted edges다. UST/minimum-weight claim은 금지한다.
- seed/RNG/cancel/max-frontier-pops/OOM status와 rectangular width != height test를 필수화한다.
