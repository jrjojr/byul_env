- 대상: =maze_binary.h/.cpp=.

  선택하고 경계에서는 가능한 방향을 강제하여 perfect maze를 만든다고 설명한다.
- 현 구현은 east/south bias를 사용한다. header 설명의 방향과 결과 root/border corridor를
  public option/contract로 명시해야 한다.
- 공통 status/options/RNG 계약은
  [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따르며 아래 signature보다 우선한다.

* 함수 구현 감사

| 단계 | 현재 | 결함/결정 |
|------+-----+-----------|
| validation | odd >=3 | 최대 크기/overflow/OOM status 없음 |
| RNG | =mt19937(random_device{})= | 다른 maze들과 seed 방식 불일치, 재현 불가 |
| initialization | 모든 grid coordinate를 blocked hash에 insert | insert 실패 무시; O(width*height) hash churn |
| cell traversal | odd cell마다 passage, east/south 선택 | east/south bias는 일관되며 edge cell에서 강제 방향 사용 |
| topology | implicit perfect tree | connected/acyclic/edge count test 없음 |
| cancellation | 없음 | 대형 grid materialization 중 취소 불가 |
| tests | 생성 성공과 print뿐 | bias, exact edge/border/seed 검증 없음 |

* 목표 계약

- cell graph와 wall raster를 분리하여 =N=((w-1)/2)*((h-1)/2)= cell, passage edge
  정확히 =N-1=을 보장한다.
- bias enum은 explicit numeric이고 지원 query/invalid status를 가진다.
- 결과는 perfect이나 uniform spanning tree가 아니며 강한 diagonal bias와 두 boundary corridor가
  있음을 숨기지 않는다.
- RNG는 caller seed를 사용하고 각 two-choice가 unbiased여야 한다. 경계 forced choice는 RNG를
  소비할지 여부까지 golden contract에 포함한다.
