* 문서 책임

대상은 =byul/navsys/route_finder/route_finder.h= 하나다. 선언을 제공하는
=route_finder.cpp=, 11개 algorithm dispatcher, module/root CMake inventory, native test,
Python =route_finder.py=와 설치 SDK consumer까지 함께 다룬다. 공통 status/options/stats,
cancel, allocator/workspace와 two-call result 원칙은
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다.

* 실행 소유권과 합류 Gate

- 이 문서는 finder lifecycle facade, 공통 search options/stats, 42개 type의
  capability/name query, typed algorithm config의 dispatcher 합류와 legacy dispatch만
  소유한다.
- Cost/heuristic evaluator, callback typedef, callback+userdata의 원자적 binding과 lifetime은
  [[file:todo-navsys-route-finder-route-finder-core.org][route_finder evaluation 문서]]가
  소유한다. 이 facade는 완성된 binding을 보관·조회하는 API를 노출할 수 있지만 계약과
  구현을 다시 정의하지 않는다.
- 각 algorithm의 search 구현, conformance와 config schema/validation은 해당
  =todo-navsys-route-finder-<algorithm>.org=가 소유한다. Dispatcher는 승인된 config를
  선택하고 공통 options/evaluation/result를 전달하는 합류점일 뿐이다.
- Dispatcher/facade의 실제 native test 파일은
  =byul/navsys/route_finder/tests/test_module_route_finder.cpp=다. Algorithm별 direct
  conformance는 =byul/navsys/route_finder/tests/test_module_route_finder_discrete.cpp=에서
  담당하고, root one-shot consumer 전환은
  =byul/navsys/tests/test_module_navsys.cpp=에서 검증한다.
- Python facade/capability/binding assertion은
  =tools/python/byul_wrapper/tests/test_route_finder.py=에 추가한다.

* 조사한 현재 소비 경로

- =byul/navsys/route_finder/CMakeLists.txt=와 =byul/navsys/CMakeLists.txt=가 module static에,
  =byul/CMakeLists.txt=가 같은 source를 root shared에 직접 compile하고 header를 SDK에
  설치한다.
- =navsys.cpp=의 11개 one-shot helper와 route-finder native test가 stack
  =route_finder_t= 및 public field semantics에 의존한다.
- =tools/python/byul_wrapper/byul_wrapper/route_finder.py=가 이 header 전체를 CFFI에
  등록하고 public enum 값을 Python에 다시 손으로 복제한다.
- Python wrapper는 임시 =ffi.new_handle()=을 native retained pointer로 넘기고 보관하지
  않으며, header에 없는 =route_finder_set_userdata=를 호출한다.
- Native/Python test는 정상 경로를 출력하는 수준이다. callback userdata identity,
  unsupported enum, no-op setter, NULL/OOM/cancel/partial result와 wrapper handle lifetime을
  검증하지 않는다.

* 업계 기준과 원전

- [[https://www.boost.org/doc/libs/latest/libs/graph/doc/astar_search.html][Boost.Graph A*]]는 graph, weight,
  heuristic, predecessor/distance maps와 visitor를 독립 계약으로 두고 non-negative weight,
  distance algebra와 event를 명시한다.
- [[https://recastnav.com/classdtNavMeshQuery][Detour dtNavMeshQuery]]는 reusable query object,
  max-node workspace, synchronous/sliced operation과 caller buffer를 분리한다.
- [[https://recastnav.com/DetourStatus_8h_source.html][Detour status]]는 success, in-progress,
  partial, invalid parameter, out-of-memory, out-of-nodes를 구분한다.
- [[https://curl.se/libcurl/c/][libcurl C API]]처럼 handle lifecycle, options, callback과
  userdata retention을 명시하고 callback pair를 원자적으로 설정하는 방식을 기준으로 삼는다.

* 확인된 P0 구현 결함

1. =route_finder_set_max_retry()=는 =a->max_retry;=만 실행해 입력값을 저장하지 않는다.
2. =route_finder_init_full()=은 =cost_fn_userdata=와 =heuristic_fn_userdata=를 저장하지 않는다.
3. 모든 algorithm adapter는 저장된 두 userdata 대신 =nullptr=를 callback에 전달한다.
4. =route_finder_create_full()=은 NULL =start/goal=을 즉시 역참조하고 allocation 실패를
   구분하지 않는다.
5. =route_finder_clear()=는 NULL을 검사하지 않고 borrowed grid, callback과 설정 전체를
   0으로 지워 object를 설명 없이 invalid 상태로 만든다.
6. =route_finder_copy()=는 grid, typedata와 callback userdata를 shallow-copy한다. retained
   lifetime도 없고 실행 상태 copy 정책도 없다.
7. =route_finder_run_ida_star()=는 실행 중 finder의 heuristic을 Manhattan으로 영구 변경한다.
8. enum 42개 중 dispatcher가 실행하는 값은 11개뿐이며 unsupported 값은 NULL 하나로
   invalid argument/OOM/no-path와 합쳐진다.

=run_ex=는 =OK/NO_PATH/INCOMPLETE/CANCELLED/LIMIT_REACHED=와 negative runtime error를
구분한다. 반환 route가 complete/partial 중 무엇인지, start에서 어느 endpoint까지인지,
cost와 optimality 보장을 stats에 기록한다. Callback은 호출 thread에서 동기 호출하며
userdata는 native가 역참조/해제하지 않고 unbind/destroy까지 caller가 살려 둔다.

* ABI와 Wrapper 전환

- ABI 1.x에서 enum 숫자와 struct layout, 모든 legacy symbol을 유지하고 canonical symbol을
  additive하게 export한다. Legacy setter/run은 새 API로 forwarding한다.
- ABI 1.x wrapper는 =ffi.new_handle= cdata를 binding lifetime 동안 강하게 보관하고,
  존재하지 않는 =route_finder_set_userdata= 호출을 제거한다.
- ABI 2에서 opaque layout, status형 setter/destroy signature와 typed config를 적용한다.
  ABI fingerprint mismatch 시 old CFFI layout으로 DLL을 load하지 않는다.
- Algorithm enum과 Python =IntEnum=은 생성 원본 하나에서 나오며 42개 numeric fixture와
  capability matrix를 비교한다.
