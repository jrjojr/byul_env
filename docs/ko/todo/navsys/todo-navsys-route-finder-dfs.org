* 대상과 비교 기준

대상은 =dfs.h/dfs.cpp=다. [[file:todo-navsys-industry-api-baseline.org][공통 기준]]과
[[https://www.boost.org/doc/libs/latest/libs/graph/doc/depth_first_search.html][Boost.Graph DFS]]의
color map, discover/tree/back/finish event, O(V+E)와 traversal semantics를 비교한다.

* 구현/소비와 결함

- =find_dfs= 하나가 module/root/SDK/CFFI/dispatcher에 노출된다. Header는 사용하지 않는
  =scalar.h=와 evaluation core까지 include한다.
- 구현은 iterative LIFO이지만 neighbor iteration/insert 순서가 path를 결정한다. “fast”라는
  설명 외에 completeness, non-optimal, deterministic order와 partial endpoint가 없다.
- NULL 또는 =max_retry <= 0=은 NULL이며 budget 소진/no-path/OOM이 구분되지 않는다.
- =debug_mode_enabled=는 방문 순서를 route 내부에 누적해 memory budget에 영향을 주지만
  stats/observer 계약이 아니다.

* 전체 Public Symbol Inventory와 disposition

=DFS_H= include guard는 현재 macro inventory에 포함하고 ABI 영향 없이
=BYUL_NAVSYS_ROUTE_FINDER_DFS_H=로 교체한다.

| symbol | 현재 signature | wrapper | disposition |
|--------+------------------+---------+-------------|
| =find_dfs= | =route_t*(grid,start,goal,int max_retry,bool debug)= | generated CFFI | =ABI 1.x: deprecate-forward; companion: add -> ABI-major: internalize= |

Capability descriptor는 “reachable goal 발견 가능, shortest/least-cost 보장 없음”을 명시한다.
Tie/neighbor order, cycle detection, maximum expansion/depth, partial path와 cancel semantics를
고정한다.
