* 대상과 역할 재설계

- 대상: =maze_room_blend.h/.cpp=.
- 이 header는 단일 fixed maze algorithm이라기보다 room placement, corridor connection,
  backtracker fill을 결합한 dungeon generator다. =maze= family에 유지할지 별도 dungeon
  component로 분리할지 role decision이 필요하다.
- 공통 status/options/lifecycle은 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]을 따른다.
- 비교 implementation은 established roguelike C/C++ library인 [[https://github.com/libtcod/libtcod][libtcod 공식 저장소]]의
  map/path/BSP style 책임 분리와 Jamis Buck backtracker reference다.

| 단계 | 현재 | 구체 문제 |
|------+-----+-----------|
| includes | =coord_hash.h=와 aggregate =maze.h= | component→aggregate 역의존; core direct include로 교정 |
| public room | =room_t= 선언 | implementation은 private =Room=만 사용해 dead ABI surface |
| size validation | width/height >=9만 확인 | parity, max size, allocation failure 없음 |
| room size | =3 + rand()%3*2 + 1= | 실제 4/6/8로 even; =room_min/max= 명칭/주석 의도와 불일치 |
| room position | =rand()%((w-rw-1)/2)= | w=9,rw=8이면 divisor 0으로 UB/crash |
| RNG | global =srand/rand= + fill의 =thread_local mt19937(random_device)= | 한 결과에 두 독립 RNG, seed replay 불가능, process-global race |
| attempts | hardcoded 30 | grid 크기와 무관하고 관찰/설정 불가 |

| fill | each remaining odd component를 DFS carve | 기존 room/corridor와 연결된다는 guarantee가 없음; 여러 disconnected maze island 가능 |
| result | blocked hash inserts | OOM failure 무시 |
| test | non-NULL, blocked>0, ASCII | rooms/count/connectivity/zero-divisor/seed 미검증 |

* 목표 algorithm 계약

- placement 후보 범위를 checked arithmetic으로 계산하고 후보가 없으면 skip 또는 explicit
  invalid-config status로 처리한다. modulo 0은 절대 호출하지 않는다.
- room size parity를 목적에 맞게 명시한다. cell rectangle은 half-open world extent, padding과
  boundary clearance를 options에 둔다.
- 단일 per-call RNG stream 또는 명시적으로 분리된 named streams를 사용하고 replay metadata를 남긴다.
- room connection graph를 sequential chain, MST, nearest-neighbor 중 typed policy로 정한다.
  기본 legacy adapter는 placement-order chain을 유지한다.
- fill component가 room/corridor network에 연결되는지 검사하고 필요하면 connector edge를
  추가한다. 결과 connectivity/cycle/room reachability claim을 명확히 한다.
- cancel은 placement attempt, corridor cell, DFS step, materialization마다 bounded poll한다.
