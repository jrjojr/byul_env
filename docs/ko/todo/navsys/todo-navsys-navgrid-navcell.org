* 대상과 type 역할

- 대상: =byul/navsys/navgrid/navcell.h/.cpp=, navgrid storage와 generated Python CFFI.
- =navcell_t=는 owning resource가 없는 작은 public value type이므로 무조건 opaque heap handle로
  만들지 않는다. explicit layout/value semantics를 안정 ABI로 유지하는 편이 업계 관행에 맞다.
- 단, enum storage/field 의미 변경은 ABI break다. 현재 layout을 snapshot하고 새 확장이 필요하면
  versioned value 또는 opaque cell-properties handle을 별도로 추가한다.
- 공통 status는 [[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]의 결정을 사용한다.

* 구현·소비 감사

| 영역 | 현재 사실 | 결함/불일치 |
|------+-----------+-------------|
| allocation | C++ =new navcell_t{}= / =delete= | paired API라 allocator boundary는 닫히지만 exception이 C ABI로 탈출 가능 |
| init | null만 검사, enum/height 어떤 값도 허용 | terrain unknown, height unit/overflow 정책 없음 |
| navgrid block | FORBIDDEN/0 cell로 replace | WATER/FOREST/MOUNTAIN은 route cost나 passability에 아무 영향 없음 |
| navgrid unblock | NORMAL/0 cell을 map에 남김 | prior terrain/height를 소실하고 sparse-default invariant도 깨뜨림 |
| callback | built-in blocked query는 =terrain==FORBIDDEN=만 검사 | enum 확장/unknown behavior 없음 |
| equality/hash | navgrid의 coord_hash callback에 의존 | navcell value equality/hash policy가 header에 없음 |
| wrapper | =navgrid.py= CFFI에 struct layout 그대로 생성 | field/type 변경은 Python ABI/parser에도 즉시 영향 |
| test | navcell 전용 semantic test 없음 | create/init/copy/invalid/layout/allocator 미검증 |

* 업계 관행 비교와 목표 의미

- [[https://docs.godotengine.org/en/stable/classes/class_astargrid2d.html][Godot AStarGrid2D]]는 solid 여부와 weight scale을 별도 property로 다룬다. BYUL의
  =terrain= enum과 passability/cost를 암묵적으로 결합하지 말고 mapping policy를 명시해야 한다.
- C API의 POD value는 opaque resource와 달리 field layout을 ABI로 약속할 수 있다. 현재
  =navcell_t=는 lifecycle resource가 없으므로 stack =init/assign=을 canonical fast path로 두고
  heap create/copy는 convenience로 유지한다.

Target 후보(공통 status 확정 전에는 ABI placeholder):

#+begin_src c
BYUL_API navsys_status_t navcell_validate(
    const navcell_t* cell);
BYUL_API navsys_status_t navcell_create_checked(
    terrain_type_t terrain, int32_t height, navcell_t** out_cell);
BYUL_API navsys_status_t navcell_is_terrain_supported(
    terrain_type_t terrain, bool* out_supported);
#+end_src

새 cost/passability가 필요하면 기존 field 의미를 바꾸지 않고 navgrid policy 또는 다음과 같은
versioned descriptor를 검토한다.

#+begin_src c
typedef struct byul_navcell_properties {
    uint32_t struct_size;
    terrain_type_t terrain;
    int32_t elevation_cells;
    float traversal_cost_scale;
    uint32_t flags;
} byul_navcell_properties_t;
#+end_src
