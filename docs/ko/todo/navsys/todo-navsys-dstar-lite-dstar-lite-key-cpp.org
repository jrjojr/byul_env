* Build compile-definition 불변식

- module static consumer는 =BYUL_STATIC=으로 C key ABI와 private comparator를 컴파일한다.
- root shared source는 같은 helper를 =BYUL_EXPORTS= 경로에서 재컴파일한다. helper 자체는
  export/install하지 않되 두 compile path의 exact ordering이 같음을 검사한다.

* 문서 책임과 결론

=byul/navsys/dstar_lite/dstar_lite_key.hpp=의 C++ helper를 전수 조사한다. 공통 기준은
[[file:todo-navsys-industry-api-baseline.org][Navsys 업계 API 기준]]이다. 이 파일은 현재 D* Lite public header install 목록에 들어가지만
실제 production consumer는 =dstar_lite_pqueue.cpp=뿐이다. 전 선언을 internalize하고
설치 SDK에서 제거한다.

* 실행 소유권과 테스트 위치

- 이 문서는 C key API를 소비하는 최소 C++ comparator/hash helper, namespace/ODR와
  public install에서 private detail로 옮기는 source-compatibility 경계만 소유한다.
- Key 수치 domain과 exact relation은 =dstar_lite_key.h= 문서, queue storage/container
  선택과 operation은 =dstar_lite_pqueue.h= 문서가 소유한다. 이 문서는 pqueue storage를
  직접 전환하지 않고 private helper와 소비 계약만 제공한다.
- 현재 =.hpp= 전용 native test 파일은 없다. Comparator property, two-TU ODR와 실제
  pqueue 소비 assertion은 shared test
  =byul/navsys/dstar_lite/tests/test_module_dstar_lite_pqueue.cpp=에 추가한다.
- Installed SDK exclusion/source-compatibility 검증은 해당 test target의 install fixture와
  root =byul/navsys/tests/test_module_navsys.cpp=에서 합류한다.

* 현재 구조 결함

- project license, =@file/@brief=, BYUL guard와 namespace가 없다.
- 사용하지 않는 =coord.h=, =vector=, =map= include가 있다.
- approximate =DstarLiteKeyEqual=, exact-bit hash, exact pointer less와
  approximate-first C compare가 한 파일에 공존한다.
- pointer functor가 NULL을 dereference하고 key lifetime을 container 밖에 의존한다.
- =std::hash<dstar_lite_key_t>=가 public mutable struct에 process-wide specialization을
  추가한다.

* 현행 선언 전수 목록과 처분

| 선언 | 현재 의미/결함 | 처분 |
|------+----------------+------|
| =DstarLiteKeyHash= | pointer exact hash | ABI 1.x: keep; conformance-block -> ABI-major: internalize |
| =DstarLiteKeyEqual= | pointer approximate equality | ABI 1.x: keep; conformance-block -> ABI-major: internalize |
| =DstarLiteKeyLess= | pointer exact lexicographic less | ABI 1.x: deprecate-forward -> ABI-major: internalize |
| =std::hash<dstar_lite_key_t>= | global mutable-key specialization | ABI 1.x: keep; conformance-block -> ABI-major: internalize |
| =dstar_lite_key_ptr_less= | C compare 기반 pointer less | ABI 1.x: deprecate-forward -> ABI-major: internalize |

이 파일에는 위 다섯 선언 외 public macro/type/function이 없다. 이름을 유지해 public
source contract로 승격하지 않는다.

* 업계 기준과 목표 detail header

- ordered comparator는
  [[https://eel.is/c++draft/associative.reqmts][C++ associative requirements]]의 strict weak
  ordering을 충족해야 한다. approximate equality는 사용할 수 없다.
- queue entry는 key와 coord를 값으로 소유한다. pointer comparator와 heap key allocation을
  함께 제거한다.
- 한 exact canonical C compare를 C/C++ 양쪽이 공유하고 detail functor는 이를 호출한다.

#+begin_src cpp
#ifndef BYUL_NAVSYS_DSTAR_LITE_INTERNAL_KEY_ORDER_HPP
#define BYUL_NAVSYS_DSTAR_LITE_INTERNAL_KEY_ORDER_HPP

namespace byul::navsys::dstar_lite_detail {

struct key_less final {
    [[nodiscard]] bool operator()(
        const dstar_lite_key_t& lhs,
        const dstar_lite_key_t& rhs) const noexcept {
        return dstar_lite_key_compare_exact(&lhs, &rhs) < 0;
    }
};

struct key_equal final {
    [[nodiscard]] bool operator()(
        const dstar_lite_key_t& lhs,
        const dstar_lite_key_t& rhs) const noexcept;
};

struct key_hash final {
    [[nodiscard]] std::size_t operator()(
        const dstar_lite_key_t& key) const noexcept;
};

} // namespace byul::navsys::dstar_lite_detail
#endif
#+end_src

- less/equal/hash는 동일한 canonical ±0/Inf/NaN 정책을 공유한다.
- 실제 pqueue가 ordered container만 사용하면 =key_equal/hash=도 만들지 않는다.
  speculative helper를 보존하지 않는다.
- C++ public convenience layer가 필요하면 별도 versioned header로 source compatibility,
  namespace와 exception policy를 새로 심사한다.

* CMake와 compatibility 계획

- =dstar_lite/CMakeLists.txt=의 public =dstar_lite_HEADERS=에서
  =dstar_lite_key.hpp=를 제거한다.
- internal header는 source list 또는 private file set에만 두고 install/export/wrapper에서
  제외한다. static module과 root shared는 같은 private file을 컴파일한다.
- repository 검색상 production consumer는 pqueue 하나지만 clean installed SDK를 쓰는
  external source fixture와 package history도 확인한다.
- external consumer가 있으면 기존 path를 한 major 동안 deprecated forwarding header로
  유지할 수 있다. 다만 broken approximate hash/equality를 계속 제공하는 대신 migration
  compile error 또는 명시적 legacy namespace를 선택한다.
- ABI 1은 inline C++ helper라 binary export보다 source compatibility가 문제다. 현행 설치
  SDK를 사용한 source fixture를 보존하고, 확인된 소비자에는 deprecated forwarding path와
  exact comparator migration을 제공한다.
- ABI 2에서는 이 header와 다섯 global 선언을 public SDK에서 종료한다. module static과
  root shared에 필요한 private detail만 남기고 wrapper/installed manifest에 exclusion
  reason을 고정한다.

* 검증 항목

- exact comparator: irreflexive, asymmetric, transitive, equivalence transitive.
- exact equality/hash: equal ⇒ same hash, ±0 canonical, 허용된 +Inf, NaN rejection.
- include: C++17 standalone, 최소 include, include order, two-TU ODR.
- inventory: public install에 없음, root export/C wrapper에 없음, static/root source는 동일.
- performance: by-value key의 comparison/allocation 수를 현행 heap-pointer map과 비교한다.
