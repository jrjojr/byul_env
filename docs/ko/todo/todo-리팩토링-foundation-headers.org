* 목적과 범위

Root, core, numal, number theory, RNG, console, tick과 aerial의 29개 header를
리팩터링한다. 이 묶음은 ABI/export macro와 version, 공통 numeric value, allocation,
callback과 module-entry를 제공한다. 전체 리팩터링에서는 전역 inventory/audit와 Navsys
분리 작업 뒤에 실행하며 완료된 Navsys dependency 경계를 보존한다.

* Header 명명 Gate

29개 모두 현행 경로와 별도의 canonical basename/path 및 명명 처분을 가진다. 대표
public type header는 type stem, 대표 type이 없는 public-component는
=<public_module>_<responsibility>.h=를 사용한다. Project명이나 물리 directory chain을
기계적으로 연결하지 않고 SDK public basename의 전역 고유성을 검사한다.

기존 public include 이름이 바뀌면 canonical header를 먼저 추가하고 기존 이름은 제거
version이 있는 compatibility-forwarder로 유지한다. Internal/generated/C++ façade도
각 역할에 맞는 최종 경로를 결정하며, 이름을 유지하는 항목도 중앙 규칙과 일치하는
근거를 manifest에 기록한다. 이 명명 처분이 없는 header가 하나라도 있으면 이 하위
TODO를 완료하지 않는다.

* Navsys 선행과 dependency 금지

이 문서는 전체 리팩터링의 Navsys 단계가 완료된 뒤 실행한다. Navsys에서 제거한
=scalar.h/scalar_equal/M_PI/numal= include·call·link edge를 Numal compatibility를 이유로
복구하지 않는다. Numal과 새 Math Engine의 consumer manifest에서 Navsys를 제외하고,
Foundation 변경 뒤 Navsys dependency scan과 static/root/install consumer를 다시
실행한다.

* 29개 Header Disposition

** Root와 core: 3개

| header | 현재/목표 역할 | 선행 판단 또는 핵심 작업 |
|--------+----------------+--------------------------|
| =byul/byul.h= | public root module-entry | umbrella 범위, =(void)= prototype, version string 수명, ABI version/fingerprint |
| =byul/byul_config.h= | public foundation/config | export macro, 충돌 없는 version encoding, generic =DBG_PRINT= internal 분리 |
| =byul/core/strset.h= | public component/opaque resource | =bool= direct include, callback 계약, owned =pop= destroy, allocation 이름과 CRT 경계 |

제거하고 internal 경계와 외부 소비자 migration을 명시한다.

* 공통 기준선과 위험

- 29개 중 정확한 표준 license block과 =@byul.*= metadata를 가진 파일은 0개다.
- 정확한 =@file=은 =dualnumber.h= 하나지만 license보다 앞서고 예제가 실제 API와 다르다.
- =strset.h=, =dualnumber.h=, =vec3.h=은 =bool= 사용에 필요한 =stdbool.h=을 직접
  include하지 않는다. =geom.h=과 =plane.h= 등도 export/type을 transitive include에 의존한다.
- =strset_pop=은 =malloc= 문자열을 반환하지만 BYUL destroy API가 없다.
- =dstar_lite_console.h=의 =dsl_debug_print_full_state=와
  =dsl_debug_print_state=는 구현이 없는데 Python wrapper가 호출한다.
- =aerial.h=의 네 선언은 구현되지 않았고 =aerial.cpp=에는 header에 없는
  =projectile_launch=가 있다.
- 현재 root integration test module 목록은 RNG, tick과 aerial을 포함하지 않는다.
- public ABI query가 최소 필요한 type은 =strset_t=, =dualnumber_t=, =dualquat_t=,
  =plane_t=, =quat_t=, =euler_order_t=, =vec3_t=,
  =vec3_intersect_result_t=, public 승인 시 =byul_rng_t=, =tick_entry_t=,
  =tick_t=와 =aerial_t=다.
