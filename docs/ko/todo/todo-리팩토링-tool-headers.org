* 목적과 범위

=tools/gpu_comp_tester= 2개와 =tools/unit_1003= 29개, 합계 31개 header의
역할, 권리, build와 BYUL SDK 경계를 정리한다. tool/private header에 public ABI 규칙을
기계적으로 덮어쓰지 않고, public으로 승격되는 새 header만 중앙 BYUL 규칙의 전체 gate를
통과시킨다.

=unit_1003=은
[[file:math-engine/todo-unit-1003-migration.org][unit_1003 → Mathunits migration TODO]]에
명시된 요구사항, 데이터와 regression reference다. 기존 source를 새 BYUL public ABI로
직접 import하거나 원래 license를 BYUL license로 교체하지 않는다.

* Header 명명 Gate

31개 모두 현행 경로, 최종 canonical 경로와 명명 처분을 가진다. Active tool header는
해당 tool의 안정된 module 경계와 responsibility가 드러나는 이름을 사용하되 BYUL SDK
public module 이름을 가장하거나 물리 directory chain을 basename으로 복제하지 않는다.
Public SDK로 승격되는 header만 중앙 public-component 명명 규칙과 SDK basename 전역
고유성 검사를 적용한다.

Immutable deprecated/reference corpus는 provenance와 regression 재현을 위해 원래 이름과
경로를 유지할 수 있다. 이 예외는 role manifest에 =reference-keep=과 근거를 기록하고
SDK install, wrapper와 public aggregate에서 제외한다. Generated 또는 migration output은
산출물만 rename하지 않고 generator/template과 소비 경로를 함께 변경한다. 모든 31개의
명명 처분이 terminal 상태가 아니면 이 하위 TODO를 완료하지 않는다.

* 감사 기준선과 보호 조건

- =gpu_compute=는 create/map/unmap API가 있으나 destroy가 없고 main이 concrete object를
  직접 delete하여 lifecycle abstraction이 분산되어 있다.
- GPU header 둘 다 include를 =extern "C"= 안에 두며 =gpu.h=의 public-looking
  =void* SDL_Window= 반환은 lifetime이 없다.
- =unit_1003=은 별도 CMake와 hardcoded =~/apps/cmake_starter= 의존을 가지며 source의
  =All Rights Reserved= 문구 때문에 복사/변경 전에 권리와 provenance 확인이 필요하다.
- compiler layout에 의존하는 bit-field/union이 =unit.h=, =nibble.h=, =real.h=,
  =si_unit.h=와 deprecated basic SI header에 있다.
- =dec.h=와 =nibble.h=는 platform-width =long=을 ABI에 사용한다.
- 두 =dec_real.h= family가 같은 basename/guard를 사용하고 =quantity.h=가 bare
  ="dec_real.h"=를 include하여 include order에 따라 다른 ABI가 선택될 수 있다.
- current basic/derived SI header는 존재하지 않는 =si_prefix.h=를 include한다.
- =ustring.h=는 owned =char*=와 create/delete/release/get =char**=의 ownership/capacity가
  불명확하다.
- tester =.hpp= 9개는 =Progress/Stopwatch/Logger=와 =BOOST_TEST_MODULE=을 각자 정의해
  하나의 translation unit 또는 통합 test에서 충돌한다.

이 결함은 원본을 즉시 고쳐 BYUL ABI로 쓰라는 뜻이 아니다. known-bug와 useful behavior를
snapshot한 뒤 clean-room BYUL 구현의 negative/compatibility requirement로 옮긴다.
