=numeq_solver.h=의 직접 =BYUL_API= 선언은 정확히 5개다. 모두 canonical 계산은
Math Engine =numeq/polynomial= 또는 =numeq/roots=가 소유하고 기존 이름은 ABI 1.x
forwarder로 남긴다.

| =BYUL_API= 선언 | 정본 분류 | 기존 symbol의 최종 운명 | Canonical 책임 |
|-----------------+-----------+--------------------------+----------------|
| =numeq_solve_linear= | Math Engine pure calculation/value | Balix legacy forwarding | scale-aware degree-1 polynomial result |
| =numeq_solve_quadratic= | Math Engine pure calculation/value | Balix legacy forwarding | degenerate case를 포함한 real polynomial result |
| =numeq_solve_quadratic_stable= | Math Engine pure calculation/value | Balix legacy forwarding | cancellation-safe roots, residual과 error bound |
| =numeq_solve_cubic= | Math Engine pure calculation/value | Balix legacy forwarding | real roots, count/multiplicity candidate와 residual |
| =numeq_solve_bisection= | Math Engine pure calculation/value | Balix legacy forwarding | bracketed root result, tolerance/budget/cancellation |

=numeq_func_f32= callback typedef는 =BYUL_API= function은 아니지만
=numeq_solve_bisection= ABI의 일부다. Canonical Math Engine callback에는 호출 횟수,
userdata lifetime, reentrancy, cancellation과 non-finite 반환 계약을 명시하고 기존 typedef와
calling convention은 forwarding ABI에서 보존한다.

* 구현 식·literal inventory

- linear: absolute =1e-8f=로 =a=를 퇴화 판정하고 =-b/a= 계산
- ordinary quadratic: =scalar_equal(a, 0)=이면 즉시 실패, =D=b^2-4ac=와 직접
  quadratic formula 사용
- stable quadratic: absolute =1e-12f= degree gate, =q=-0.5(B+sign(B)sqrt(D))=,
  =|q| > 1e-20f= 분기와 sorted roots
- cubic: depressed cubic/Cardano/trigonometric branch, =1e-8f= discriminant와 coefficient
  gate, =M_PI=, exact =x1 == x2= repeated-root count
- bisection: =a < b=와 endpoint sign product, 최대 100회, =|f(mid)| < tol= 또는
  interval width =tol= 종료

공식, tolerance와 iteration cap은 Math Engine formula/context manifest로 이동한다. Balix
forwarder는 =bool=, root array/count와 output mutation을 canonical status/result에 mapping할
뿐 solver 식을 보유하지 않는다.

* Build·DLL 이행

현재 =numeq_solver.cpp=는 =balix_SOURCES=에 직접 들어가 root =byul= shared에서도
Balix inventory를 통해 compile된다. =collision= static target은 이 symbol을 호출하지만
자신의 =CMakeLists.txt=에는 solver dependency가 없다.

- Static 과도기: solver source는 단일 Math Engine manifest와
  =byul_math_engine_static=으로 이동한다. Balix/collision/projectile은 target dependency로
  소비하고 기존 5개 이름의 root ABI shim을 유지한다. Root shared는 중앙 규칙이 유지되는
  동안 같은 manifest를 =BYUL_EXPORTS=로 직접 compile한다.
- 독립 DLL 전환: 중앙 규칙과 =rules-ref.org= pin 승인 뒤 canonical solver symbol은
  Math Engine DLL에만 존재한다. BYUL DLL에는 기존 5개 legacy forwarder만 남기고
  Balix, collision과 projectile의 internal call은 canonical imported target을 사용한다.

* 현재 진단

- 일반 quadratic header는 linear fallback을 설명하지만 구현은 =a ~= 0=이면 실패한다.
- stable quadratic도 linear/constant/identically-zero polynomial을 구분하지 않는다.
- stable variant는 output null을 확인하지 않고 q=0의 double root와 signed zero 계약이
  불명확하다.
- cubic은 exact equality로 repeated root count를 판단하고 coefficient scale에 무관한
  =1e-8=을 사용한다. acos argument clamp와 residual refinement가 없다.
- bisection은 =tol <= 0=, NaN/Inf, endpoint root, discontinuity와 callback cancellation을
  충분히 구분하지 않고 100회를 고정한다.
- =bool=과 root count만으로 no real root, degenerate, invalid, non-finite와 max iteration을
  구분하지 못한다.

* 목표 result

Polynomial degree/status, real root count, sorted root, multiplicity candidate, residual,
condition/scale, iterations/evaluations와 error bound를 반환한다. Caller는 absolute와
relative tolerance, max evaluation, cancellation과 backend를 지정한다.
