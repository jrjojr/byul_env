State transition, P/I/D 식, saturation과 anti-windup은 Math Engine =control/pid=의
순수 kernel이다. =pid_controller_t=와 독립 channel 묶음은 Math Engine value/API다.
Balix controller는 target/measurement 수집, frame/tick에서 받은 =dt= 전달, actuator limit
적용과 lifecycle만 소유한다. 기존 scalar PID 8개와 Vec3 PID 8개 이름은 독립 DLL 전환
뒤 Math Engine canonical API를 부르는 Balix legacy forwarding symbol로 유지하거나
명시적으로 deprecate한다.

* BYUL_API 전수 inventory

아래 표는 두 공개 header의 =BYUL_API= 선언 16개를 하나씩 기록한다. =ME-VALUE=는 순수
Math Engine state/options API, =ME-KERNEL=은 순수 계산 API, =LEGACY-FORWARD=는 기존 BYUL
ABI 이름의 최종 역할이다. 와일드카드나 묶음 표기는 symbol inventory로 사용하지 않는다.

=pid_controller_t=와 =pid_controller_vec3_t=도 public value layout이므로 field 순서,
=sizeof/alignof/offsetof=와 =bool= ABI를 versioned compatibility 대상으로 기록한다. 신규
Math Engine ABI가 status, previous-sample-valid flag, derivative filter state를 추가할 때 기존
struct에 field를 끼워 넣지 않고 새 versioned type을 만든다.

* 현재 진단

- =pid_init_full/assign/set_state/update/preview= 일부가 null과 =dt <= 0=을 검사하지 않아
  division by zero와 crash 가능성이 있다.
- Integral/derivative discretization이 backward Euler 한 종류로 고정되고 derivative kick,
  setpoint weighting, filter와 sample-time change 계약이 없다.
- Anti-windup은 saturation 시 현재 적분만 되돌리는 conditional integration이며
  output limit와 integrator limit의 의미가 분리되지 않는다.
- =pid_preview=의 finite input output은 현재 =pid_update=의 반환값과 같지만, would-commit
  state와 anti-windup decision/status를 반환하지 않는다. 특히 Balix가 밖에서 다시 clamp하는
  실제 actuator 경로와의 preview parity는 성립하지 않는다.
- =pid_init_auto=의 gain 공식은 plant model/ultimate gain 없이 고정 상수에서 나오므로
  일반 auto-tune으로 부르기 어렵다.
- Vec3는 축별 scalar wrapper지만 frame, gain per-axis와 coupled system 제한이 없다.
- =controller_create_pid=는 사용자 =kp/ki/kd/dt=를 전달받고도 =pid_impl_init=만 호출해
  기본 gain을 사용한다.
- =controller_pid_compute=는 매 tick =pid.dt=를 덮어쓰고 =pid_impl_t.output_limit=로
  결과를 외부 clamp한다. =pid.output_limit/anti_windup=은 이 saturation을 보지 못하므로
  integral windup이 계속될 수 있다.
- =pid_init_full=, =pid_assign=, =pid_set_state=, =pid_reset=, =pid_update=,
  =pid_preview=와 모든 Vec3 함수에는 null 검사가 없고 =pid_update/pid_preview=는 zero 또는
  non-finite =dt=에서 division-by-zero/NaN을 만든다.
- Vec3 구현에는 =pid_vec3_auto= definition이 전혀 없으므로 16개 공개 선언 중 1개가
  link/export 불가능하다. 나머지 15개 선언은 definition이 존재한다.
