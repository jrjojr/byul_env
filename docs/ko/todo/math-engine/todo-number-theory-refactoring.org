* 목적과 단일 소유권

=byul/number_theory=의 소수 판정, 다음 소수, 인수분해, segmented sieve, 정수 제곱근과
generated base-prime table을 Math Engine 정수론 component로 통합한다. Foundation 또는
consumer TODO는 이 공식과 구현을 다시 소유하지 않는다.

기존 include path와 symbol의 compatibility도 Math Engine이 소유한다. Consumer는
canonical Number Theory target/header를 link/include할 뿐 별도 소수·인수분해 구현을
두지 않는다.

* 기존 Header 5개

| Header | Math Engine disposition |
|--------+-------------------------|
| =byul/number_theory/number_theory.h= | canonical aggregate 또는 versioned compatibility aggregate |
| =byul/number_theory/byul_prime32.h= | primality·next-prime·factorization canonical/forwarding surface |
| =byul/number_theory/byul_prime_segmented_u32.h= | segmented sieve와 callback/status surface |
| =byul/number_theory/byul_base_primes_65535.h= | generated internal table; SDK export 금지 |
| =byul/number_theory/byul_compiler.h= | unused detail 조사 후 internalize 또는 제거 |

* 계약 경계

- 정수론 API는 simulation, Navsys, GUI와 mutable world state를 알지 않는다.
- Callback early-stop, invalid input, capacity 부족과 partial result를 서로 다른 status로
  표현한다.
- Overflow와 범위는 fixed-width integer 기준으로 명시하며 silent wrap을 정상값으로
  취급하지 않는다.
- Generated table은 generator/template/output hash가 함께 재현돼야 한다.
- 암호학적 안전성을 보장하지 않는 알고리즘은 그 제한을 public contract에 기록한다.
