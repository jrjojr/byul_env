* 목적

=byul/balix/collision/collision.h=와 =collision.cpp=에 있는 교차 좌표, 접촉 시각,
normal, barycentric 판정과 연속 충돌(TOI) 공식을 독립 Math Engine의 stateless
geometry query로 옮긴다. Balix는 tick 감시, broad-phase 후보 선택, pair cache, 접촉
지속성, event와 response만 소유하고 선택한 한 쌍의 계산은 Math Engine에 요청한다.

이 문서는 source 이동 전 실행 계획이다. 현재 오류 동작을 새 canonical 계약으로
승인하지 않으며, 분석적 식·고정밀 oracle·잔차와 기하 membership을 기준으로 수정한다.

* 소유권 경계

| 책임 | Math Engine | Balix와 상위 consumer |
|------+-------------+------------------------|
| 단발 공간 질의 | 점/선/면/구/삼각형의 교차 집합, 최근접점, normal, barycentric, feature | 질의할 object pair와 frame 선택 |
| 명시적 시간 구간 질의 | 입력 motion 두 개와 =[t_min,t_max]=에서 TOI/contact 계산 | tick 시작 시각, 주기, substep 정책과 재호출 |
| 결과 | hit/miss/overlap/degenerate, 접촉 좌표·시각·오차 | contact cache, begin/persist/end event와 impulse response |
| 가속 구조 | 단일 primitive query가 사용할 순수 helper | world broad phase, spatial index 갱신과 candidate filtering |
| 허용 오차 | caller가 넘긴 immutable 계산 context | 제품 preset 선택과 world scale 전달 |

Math Engine query는 entity id, tick callback, world container, event dispatcher와 전역
clock을 알지 않는다. 반대로 Balix adapter는 교차 공식, Newton 식, Rodrigues 식과
barycentric 식을 다시 구현하지 않는다.

* 확인된 정확성·안전 결함

** P0 실제 구현 결함

1. =detect_triangle_collision_rotating_alpha=의 =eval_triangle_at=은 =theta=를 계산하지만
   사용하지 않는다. =tri_center=는 =try_candidate=로 전달된 뒤 무시되고 =A0/B0/C0=를
   자기 자신 중심, angle 0으로 반복 회전하는 placeholder만 실행한다.
2. =detect_triangle_collision_moving=은 상대 평면에서 시각을 구한 뒤 world projectile
   좌표를 초기 =A0/B0/C0=에 검사한다. 비영 triangle translation을 candidate triangle이나
   projectile relative coordinate에 반영하지 않아 false hit/miss가 가능하다.
3. precise sphere의 Newton 경로는 bracket, 접촉 전후 부호, 최종 잔차 합격을 확인하지
   않고 clamp된 시각의 위치를 구 surface로 snap한 뒤 success를 반환할 수 있다.
4. 일반 sphere 함수는 chord에서 얻은 시각에 곡선 위치를 평가하고 강제로 sphere에
   snap한다. 반환 시각과 반환 좌표가 같은 운동식의 실제 root라는 보장이 없다.
5. 회전 triangle 함수는 초기 plane normal의 root 두 개만 검사한다. 회전한 plane의
   실제 root가 그 집합에 없을 수 있으므로 “high angular velocity 정확”이라는 설명과
   알고리즘이 맞지 않는다.
6. =detect_plane_collision=은 primary 경로에서는 =pos_curr=를 무시하고 등가속도 model을
   쓰지만 실패하면 =pos_prev-pos_curr= segment로 바뀐다. 두 입력이 같은 motion을
   표현하는지 검사하지 않아 한 호출에서 서로 다른 궤적 의미가 섞인다.
7. =detect_sphere_collision_moving_precise=의 high-curvature/no-segment-root 경로는 두 seed를
   Newton으로 보정한 뒤 최종 residual 합격 없이 =finish_at_time=을 호출하므로 실제 root가
   없어도 true가 될 수 있다.
8. =std::numeric_limits=를 사용하면서 =<limits>=를 직접 include하지 않아 transitive
   include에 의존한다. 독립 Math Engine source에서는 필요한 표준 header를 직접 소유한다.

** 공차와 차원 결함

- =VEC3_ABS_EPS_LEN2=를 제곱 길이, 실제 길이, 속도, 가속도, quadratic coefficient와
  derivative에 같은 숫자로 적용한다. 서로 다른 차원의 값을 같은 threshold로 비교한다.
- =1e-20=, =1e-16=, =1e-8=, =1e-6=, =1e-5=가 함수별로 박혀 있고 좌표 scale,
  시간 scale, triangle extent와 radius를 반영하지 않는다.
- =select_axis_for_1d=은 실제 길이 =l=을 length-squared epsilon과 비교한다.
- =nearly_colinear=은 =la2*lb2=의 overflow/underflow와 cosine 범위 clamp를 처리하지 않는다.
- NaN/Infinity 입력을 선제 거부하지 않아 비교가 false로 흐르거나 NaN output이 남을 수
  있다.
- snap 거리가 허용 오차를 넘어도 hit로 바꾸므로 수치 오차 보정과 실제 geometry 변경이
  구분되지 않는다.

** API와 ABI 결함

- =bool=false=가 miss, invalid pointer, degenerate primitive, unsupported approximation과
  non-convergence를 모두 뜻한다.
- miss/실패 때 output 보존 여부와 hit가 initial overlap/tangent/enter/exit 중 무엇인지
  알 수 없다.
- 접촉 normal, 두 shape의 closest/contact point, barycentric, feature id, penetration,
  residual과 error bound가 없다.
- =target_accel=과 translation velocity의 nullable 규칙이 함수마다 다르며 이름만으로
  알 수 없다.
- C header의 generic =detect_*= prefix와 compile macro가 미래 Math Engine DLL의 독립
  symbol/version 경계를 표현하지 못한다.

* 업계 기준선

- [[https://box2d.org/documentation/md_collision.html][Box2D Collision]]은 geometry primitive,
  distance, shape cast, TOI와 contact manifold를 simulation 바깥에서도 쓸 수 있는
  collision interface로 제공한다. World 갱신과 별개로 단발 query 결과를 구조화하는
  경계가 BYUL의 목표와 맞는다.
- 같은 Box2D 문서는 회전 TOI가 작은 회전에서도 collision을 놓칠 수 있음을 명시한다.
  BYUL도 fixed-plane approximation을 “정확”으로 숨기지 않고 capability, quality와
  error/status로 노출한다.
- [[https://doc.cgal.org/latest/Kernel_d/index.html][CGAL Geometry Kernel]]은 intersection
  검출 predicate와 실제 intersection construction을 구분하고, 교차 결과가 점·선분·빈
  집합처럼 여러 종류일 수 있음을 명시적 result로 표현한다. BYUL C ABI도 단순 bool 대신
  tagged result를 사용한다.
- CGAL의 exact-predicate/inexact-construction 구분을 따라 topology를 결정하는 predicate와
  좌표 construction의 정확도 상태를 분리하고, full precision profile의 oracle을 둔다.
- [[https://box2d.org/files/ErinCatto_ContinuousCollision_GDC2013.pdf][Continuous Collision Detection]]의
  root-finding 관점처럼 TOI는 반환 좌표를 surface에 snap하는 것으로 증명하지 않고
  separation 함수의 bracket, root와 잔차로 검증한다.

* 목표 Math Engine 계약

** 입력 value

- point/vector/plane/sphere/triangle은 caller-owned POD이고 entity나 Balix pointer를
  보관하지 않는다.
- motion은 position [L], velocity [L/T], acceleration [L/T^2]와 local time interval [T]을
  명시한다.
- triangle rigid motion은 translation, rotation center [L], unit axis, angular velocity
  [rad/T]와 angular acceleration [rad/T^2]를 가진다.
- 좌표계 handedness, forward convention과 frame id는 호출 경계에서 명시하고 query 안에서
  world state를 찾지 않는다.
- f32 ABI를 기존 adapter에 제공하되 f64 oracle/canonical profile과 변환 규칙을 둔다.

** 결과와 상태

Canonical result는 최소한 다음 의미를 POD field 또는 versioned accessor로 제공한다.

| field family | 계약 |
|--------------+------|
| function status | success, invalid argument, non-finite, unsupported, no-convergence, insufficient workspace |
| relation | miss, enter, tangent, exit, initial-overlap, coplanar, degenerate |
| time | local =t=, normalized parameter와 interval; Balix absolute tick time은 제외 |
| geometry | point on A, point on B, representative point, normal A-to-B |
| topology | barycentric, feature id, intersection kind point/segment/area |
| metric | signed separation, penetration, residual과 estimated error bound |
| quality | algorithm/profile, iterations/evaluations, approximate/refined flag |

함수 오류에서는 caller output을 변경하지 않는다. 정상 계산의 =miss=는 성공 status와
relation miss로 구분하고 result 전체를 결정적으로 초기화한다. initial overlap을 임의로
surface로 옮기지 않고 overlap 상태와 representative closest/contact를 함께 반환한다.

** 알고리즘 계약

- 점-평면 등가속도는 하나의 motion 식으로 quadratic root를 풀고 별도 segment 질의와
  섞지 않는다.
- 등가속도 점-구의 squared-distance 식은 일반적으로 4차식이다. colinear 특수 경우만
  quadratic exact로 분류하고 일반 경우는 polynomial interval isolation, bracketed
  refinement 또는 보수적 advancement 중 선택한 방법과 보증을 기록한다.
- 이동 구는 상대 motion으로 환원하되 두 물체의 접촉점과 normal을 반환 시각의 각
  motion에서 계산한다.
- translation triangle은 candidate 시각의 세 vertex 또는 일관된 relative frame에서
  plane과 barycentric을 함께 평가한다.
- 회전·각가속도 triangle은 trigonometric separation root다. 고정 초기 normal 근사는
  별도 fast capability로만 제공하고 error bound 없는 결과를 precise라 부르지 않는다.
- snap은 계산된 error bound 안의 좌표 정규화에만 사용하며 hit/miss 판정을 바꾸지 않는다.

** tolerance와 결정성

- absolute length, relative, time, angular, barycentric와 residual tolerance를 구분한다.
- scale은 =max(primitive extent, radius, |position|=의 적절한 local scale)에서 계산하고
  단위가 다른 coefficient에 길이 epsilon을 재사용하지 않는다.
- tangent/discriminant 경계는 uncertainty 상태 또는 interval로 표현한다.
- 동일 입력, resolved context, backend와 floating-point mode에서는 결과와 feature order가
  결정적이어야 한다.
- query는 reentrant이고 전역 mutable threshold, allocation과 callback을 사용하지 않는다.

* Balix compatibility와 감시 경계

- 기존 8개 =BYUL_API detect_*= symbol과 include path는 ABI 1.x에서 유지하고 deprecated
  forwarding shim으로 canonical Math Engine query를 호출한다.
- 기존 =bool=은 canonical status가 정상이고 relation이 legacy hit로 매핑될 때만 true다.
  invalid, miss, degenerate, unsupported와 non-converged의 상세 원인은 새 API에서만 보존되며
  legacy 문서에는 손실을 명시한다.
- 현재 결함과 수치 우연을 보존하지 않는다. 결과가 달라지는 fixture는 corrected behavior,
  compatibility 영향과 release-note 후보로 분류한다.
- =projectile_tick=의 주기 감시, target 순회, pair 선택, event/response와 =t_prev= 합산은
  Balix/consumer에 남는다. 그 내부의 교차 공식만 canonical 호출로 교체한다.
- canonical source는 Math Engine target에서 한 번만 compile한다. Balix와 root shared가
  같은 source를 다시 inventory하지 않는다.

* 필수 검증 corpus

** Plane와 sphere

- zero/non-unit normal, coplanar stationary, endpoint, tangent, enter/exit 두 root
- zero radius, center coincidence, initial overlap, moving-away와 exactly tangent
- chord는 hit지만 곡선은 miss인 case와 chord는 miss지만 곡선은 hit인 case
- 상대 가속도가 있는 moving sphere와 Galilean translation invariance
- 반환 시각에서 motion 위치, signed distance와 root residual 동시 검사

** Triangle

- interior, 세 edge, 세 vertex, outside, winding reversal와 zero-area triangle
- nonzero normal/tangential translation으로 현재 moving-triangle frame 오류를 재현
- rotation center가 =A0=가 아닌 off-origin triangle
- zero omega/alpha parity, 세 축 omega, 양·음 각속도와 큰/작은 angle
- 현재 alpha placeholder가 회전 없는 답을 내는 비대칭 triangle fixture
