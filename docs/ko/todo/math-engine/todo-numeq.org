* 목적

이 문서는 =balix/numeq= 에 섞인 기능을 분리하고 루트 =byul/numeq= 를 재사용 가능한
독립 수치해석 모듈로 구축하기 위한 계획서이다.

=numeq= 는 계산기, 게임, 물리와 GUI를 모르며 callback과 수치 자료형을 사용해
근 찾기, 미분, 적분, ODE, 보간과 최적화를 수행한다.

임베디드 공학용 CAS 그래프 계산기에서는 expression이 compile한 식을 callback으로
받아 solve, derivative, integral, graph 분석과 ODE 기능을 제공한다. =numeq= 자체는
계산기 worksheet, CAS와 renderer를 알지 않는다.

기초 자료형과 공식은 =todo-numal.org=, 표현식과 전체 엔진은
=todo-math-engine.org=, 단위와 quantity는 =todo-mathunits.org= 에서
다룬다.

* 최종 책임

byul/numeq/
  polynomial      다항식 평가와 저차 방정식
  roots           이분법, secant, Newton, Brent
  differential    finite difference, gradient, Jacobian
  integral        trapezoid, Simpson, adaptive quadrature
  ode             Euler, midpoint, RK와 adaptive step
  interpolation   Lagrange, Hermite, spline
  optimization    최소화, 최대화와 수치 최적화
  approximation   회귀와 곡선 근사
  error           tolerance, 수렴과 상태 보고
#+end_src

* 제외 범위

- 문자열 expression parser와 AST
- 중력, 항력과 =motion_state_t=
- Kalman filter, PID와 MPC의 게임·물리 정책
- GUI와 그래프 렌더링
- 기호 적분과 범용 CAS
- 단위 이름, prefix, affine 변환과 quantity 저장

=numeq= callback 경계는 canonical unit의 순수 숫자 배열을 기본으로 한다. expression
또는 =mathunits= 가 입력 quantity의 차원을 검사하고 canonical unit으로 변환한 뒤
=numeq= 를 호출하며, 결과에 적절한 차원을 다시 부여한다. 수치 반복 내부에서 문자열
단위 조회나 registry 탐색을 수행하지 않는다.

=numeq= 알고리즘은 분야별로 복제하지 않는다. 동일 API에 Math Engine 계산 context의
fast/precision/hybrid preset을 적용한다. 세부 budget, tolerance와 승격 규칙은
=todo-math-computation-context.org= 를 따르고 설정 해석은
=todo-math-engine-configuration.org= 를 따른다.

* =balix/numeq= 해체 원칙

- solver와 범용 ODE 알고리즘은 루트 =byul/numeq= 로 이동한다.
- 순수 물리 공식은 Math Engine =physics=로 이동하고 Balix에는 state adapter만 둔다.
- Kalman 계산 kernel은 Math Engine =estimation=으로 이동하고 Balix에는 sensor/tick
  binding만 둔다.
- PID와 MPC 계산 kernel은 Math Engine =control/optimization=으로 이동하고 Balix에는
  target/route/controller orchestration만 둔다.
- 최종적으로 =balix/numeq= 혼합 디렉터리를 제거한다.
- 기존 =numeq_*= public API는 ABI 호환을 유지한다.

* 기능 영역 1: 현재 알고리즘 진단과 회귀 테스트

- linear, quadratic, stable quadratic, cubic과 bisection 조사
- root 개수, 정렬, 중복 root와 퇴화식 규약 확정
- 일반 quadratic header 설명과 구현의 linear fallback 불일치 교정 계획
- stable quadratic의 출력 pointer와 퇴화식 처리 검증
- 가속 표적 요격에서 4차식 상수항이 누락되는 경로의 회귀 테스트
- 기존 Euler, Verlet와 RK4의 상태 결합 부분 조사

현재 오류를 승인하는 테스트를 추가하지 않고, 기대 수학식과 잔차를 기준으로 한다.

* 기능 영역 2: 공통 callback과 결과 상태

** 함수 형태

scalar, vector와 ODE callback을 C ABI로 정의한다.

#+begin_src c
typedef double (*numeq_func_f64)(double x, void* userdata);

typedef void (*numeq_ode_func_f64)(
    double t,
    const double* state,
    double* out_derivative,
    size_t dimension,
    void* userdata);
#+end_src

** 결과 상태

- success
- invalid argument
- no bracket 또는 no root
- max iterations
- non-finite evaluation
- singular 또는 ill-conditioned
- overflow, underflow와 inexact
- cancelled
- insufficient workspace
- unsupported backend

반환값 하나만으로 실패 원인을 숨기지 않고 선택적 통계에 반복 횟수와 함수 평가
횟수를 기록한다.

모든 장시간 알고리즘은 선택적 cancel callback 또는 cancellation token, 최대 함수
평가 횟수와 caller-provided workspace를 받을 수 있어야 한다. thread가 없는 장치에서는
일정 평가 횟수마다 cooperative yield 지점을 제공한다.

* 기능 영역 3: 방정식과 근 찾기

- Horner polynomial evaluation
- 안정적인 linear과 quadratic
- cubic과 quartic
- bisection
- secant
- Newton-Raphson
- Brent 계열
- 구간 scan과 여러 root 탐색
- 연립 비선형방정식은 후속 범위로 분리

가속 요격의 4차식은 검증된 quartic 또는 범용 root 탐색으로 처리하고 입력 구간에서
가장 이른 유효 양의 root를 선택한다.

* 기능 영역 4: 수치 미분

- forward와 backward difference
- central difference
- 5-point stencil
- gradient와 Jacobian
- scale을 고려한 step 크기
- 복소 step 및 자동미분은 backend 확장 후보

알려진 해를 가진 함수로 절대·상대 오차와 step 변화에 따른 수렴을 검증한다.

* 기능 영역 5: 수치 정적분

- midpoint와 trapezoid
- Simpson
- adaptive Simpson
- Gauss-Legendre와 Gauss-Kronrod 후보
- 무한 구간과 특이점 적분은 후속 단계

API는 tolerance, 최대 평가 횟수, 추정 오차와 수렴 상태를 제공한다.

* 기능 영역 6: 범용 ODE

- explicit Euler
- midpoint 또는 RK2
- RK4
- adaptive Runge-Kutta
- stiff ODE와 DAE는 SUNDIALS backend 후보로 분리
- Verlet 계열은 2차 운동식 상태 계약을 별도 정의

=numeq= ODE는 배열 상태와 callback만 다룬다. Balix dynamics adapter가
=motion_state_t= 와 물리 법칙을 callback으로 변환한다.

- speed preset: semi-implicit Euler, Verlet, 고정 step과 작은 평가 budget 중심
- precision preset: RK4, adaptive Runge-Kutta, 엄격 tolerance와 오차 추정 중심

* 기능 영역 7: 보간·근사·최적화

- 선형, Lagrange와 Hermite 보간
- cubic spline
- 선형·다항 회귀
- golden-section search
- Brent minimization
- gradient 기반 최적화는 행렬 backend와 오류 계약 확정 후 도입

* 기능 영역 8: 다중 정밀도

- =f32=: 실시간·GPU·대량 계산
- =f64=: 기본 수치해석
- =f128=: 우주·천문과 고정밀 검증
- arbitrary precision: 기준 결과와 사용자 지정 정밀도
- decimal/fixed-point: 적합한 알고리즘만 명시적으로 지원

모든 알고리즘이 모든 숫자 backend에 적합하다고 가정하지 않는다. 예를 들어 금융
산술은 decimal에 적합하지만 삼각함수와 ODE는 별도 지원 수준을 정의해야 한다.

public C API는 =*_f32=, =*_f64=, =*_f128= 처럼 정밀도를 명시하고 내부 template
또는 trait로 구현을 공유한다.

mpdecimal은 모든 수치해석의 기본 backend가 아니라 decimal-compatible 알고리즘의
선택 backend로 검토한다.

적용 후보:

- polynomial evaluation과 root refinement
- interpolation과 decimal 입력 회귀
- 고정밀 수치 미분·적분의 기준 계산
- 금융·단위 기반 반복 계산

기본 적용에서 제외:

- 실시간 ODE와 대량 graph sampling
- 삼각함수 중심 solver
- SIMD/GPU batch
- geometry collision과 contact

알고리즘별 지원 숫자 backend를 capability로 보고하고, 지원하지 않는 경우 binary
backend로 조용히 변환하지 않고 unsupported를 반환한다.

* 기능 영역 9: 외부 backend 판단

- [[https://www.boost.org/doc/libs/latest/libs/math/doc/html/index.html][Boost.Math]]: root,
  quadrature, differentiation, interpolation과 특수함수
- [[https://computing.llnl.gov/projects/sundials/cvode][LLNL SUNDIALS/CVODE]]: 고급
  ODE, stiff ODE와 DAE
- Eigen: Jacobian, 선형 시스템과 최적화 내부 계산
- Boost.Multiprecision: =f128= 및 임의 정밀도
- mpdecimal/libmpdec: decimal-compatible 알고리즘의 선택 backend

BYUL이 public C API와 결과 계약을 소유하고 외부 라이브러리는 private backend로만
사용한다.

* 기능 영역 10: batch·SIMD·GPU

- 서로 독립적인 다수 방정식
- 많은 함수 지점의 미분과 적분
- 다수 ODE 상태와 trajectory
- 최적화 후보 평가
- 그래프용 1D/2D 함수 표본

단일 문제는 CPU를 기본으로 하고 =*_batch= API와 benchmark로 GPU 임계값을 정한다.

* 기능 영역 11: 그래프 계산 지원

- 화면 구간의 root, 교점, 극값과 변곡 후보 탐색
- adaptive 1D sampling과 오차·불연속 상태 보고
- parametric와 polar 곡선의 parameter sampling
- implicit graph용 2D scalar grid 평가
- pan/zoom에서 재사용 가능한 sample cache key
- 렌더러가 아닌 plot 계층이 수행할 수치 분석 API
- 한 frame의 평가 예산, progressive refinement와 취소

Marching Squares, polyline 연결과 화면 clipping은 plot 계층의 책임이다. =numeq= 는
필요한 표본, bracket, root와 오차 추정만 제공한다.

* 기능 영역 12: 임베디드 실행 profile

- =full=: adaptive 고급 알고리즘, f64/f128와 arbitrary precision 후보
- =compact=: f32/f64, 제한 차원 ODE와 고정 최대 반복
- =minimal=: bisection, 기본 미분·적분과 고정 workspace 중심
- heap 없는 실행 경로와 workspace 크기 조회 API
- 알고리즘별 stack 상한과 callback 재진입 계약
- 동일 입력, context와 backend에서 결정론적 상태·통계 보고
- 기능이 제외된 profile은 명시적 unsupported 반환

* 완료 기준

- 루트 =byul/numeq= 독립 target이 존재한다.
- =numeq= 가 =balix=, expression과 GUI에 의존하지 않는다.
- =balix/numeq= 혼합 디렉터리가 제거된다.
- 근 찾기, 미분, 적분과 범용 ODE가 공통 callback API를 사용한다.
- 기존 SDK 호출은 호환 wrapper 또는 유지된 API로 동작한다.
- 가속 표적 4차식 오류가 테스트와 함께 교정된다.
- 정밀도와 수렴 상태가 명시적으로 보고된다.
- 장시간 계산을 취소할 수 있고 고정 workspace에서 예측 가능한 메모리를 사용한다.
- 그래프 계층에 표본, root, 극값과 오차 정보를 렌더러 독립적으로 제공한다.
- 임베디드 profile별 지원 알고리즘과 자원 상한이 문서화된다.
