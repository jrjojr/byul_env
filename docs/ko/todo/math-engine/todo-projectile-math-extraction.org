* 목적

Projectile 6개 header에 섞인 발사 해, intercept/guidance, trajectory prediction,
time-step, thrust/fuel 계산을 Math Engine physics/control/trajectory kernel로 옮긴다.
Projectile은 projectile state, callback, tick, result storage와 launch orchestration만
소유하는 consumer가 된다.

* 기존 Header 6개와 분리

| Header | Math Engine 계산 | Projectile consumer에 남는 책임 |
|--------+------------------+---------------------------------|
| =byul/projectile/projectile.h= | shell/rocket/missile별 보편 계산 유무 확인 | preset state, lifecycle와 launch dispatch |
| =byul/projectile/projectile_core.h= | launch parameter·inverse solution | projectile state, callback와 update 적용 |
| =byul/projectile/guidance.h= | point/lead/accelerated intercept와 direction query | callback signature, userdata와 target mapping |
| =byul/projectile/projectile_predict.h= | neutral trajectory/impact prediction, initial-force query | result/trajectory allocation, printing과 orchestration |
| =byul/projectile/projectile_tick.h= | suitable dt/max-time heuristic의 canonical policy 또는 kernel | tick lifecycle, collision event와 completion |
| =byul/projectile/propulsion.h= | thrust/fuel/runtime/impulse prediction과 response kernel | mutable propulsion state, controller binding와 serialization |

* 금지 경계

- Math Engine API는 =projectile_t=, =entity_dynamic_t=, =environ_t=, =ground_t=,
  =propulsion_t=, =trajectory_t=와 guidance callback을 직접 받지 않는다.
- Projectile callback/tick/event/result allocation은 Math Engine으로 옮기지 않는다.
- Cardano, launch equation, integration, fuel/thrust 식과 tolerance를 Projectile adapter에
  남기지 않는다.
- Navsys route/heading은 guidance kernel의 입력 정책이나 dependency가 아니다.
