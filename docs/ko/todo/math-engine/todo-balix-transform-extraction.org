* 목적

=byul/balix/xform/xform.h=와 =xform.cpp=의 vector/quaternion/TRS/affine 계산,
composition, inverse, 좌표 적용, 보간, look-at, pivot와 reflection을 Math Engine으로
옮긴다. Balix에는 world 좌표 범위 정책과 debug 출력 adapter만 남긴다.

현재 =xform_t=는 =entity_dynamic_t= 등에 값으로 포함된 공개 ABI다. 따라서 source를
물리적으로 옮기는 것과 기존 layout/symbol을 제거하는 일을 같은 단계에서 하지 않는다.
먼저 수학 계약을 교정하고 canonical API를 추가한 뒤 기존 =xform_*=를 forwarding한다.

* 소유권 경계

| 책임 | Math Engine | Balix |
|------+-------------+-------|
| 값과 계산 | rigid pose, TRS, affine matrix, composition, inverse, point/vector/normal 적용 | entity/world 안의 현재 pose 보관 |
| orientation | quaternion, axis-angle/Euler 변환, look/align, pivot와 보간 | camera/entity가 어느 target/up을 선택할지 결정 |
| 수치 정책 | finite/singular/degenerate 판정, 명시 tolerance와 status | world bounds preset과 out-of-world 처리 |
| 표현 | caller-owned POD, matrix layout와 handedness 계약 | legacy =xform_t= layout와 compatibility |
| 출력 | caller buffer formatter용 값 제공는 별도 utility 후보 | log sink, console/iostream, locale와 debug level |

Math Engine transform 함수는 위치를 임의의 게임 범위로 saturate하거나 stdout/stderr에
쓰지 않는다. Balix는 matrix/quaternion/scale 공식을 다시 구현하지 않는다.

* 공개 surface 전수 분류

** 공개 macro와 type

| symbol | 현재 계약과 위험 | 목표 disposition |
|--------+------------------+------------------|
| =XFORM_MAX_POS= | 모든 축 상한 =99999.0f=, translation unit override 가능 | Math Engine에서 제거, Balix world-bounds policy로 유지 |

Canonical Math Engine은 한 구조에 서로 닫히지 않는 모든 변환을 억지로 담지 않는다.

| 목표 value family | 표현과 용도 |
|-------------------+-------------|
| rigid pose | translation + unit quaternion, scale/shear 없음 |
| decomposed TRS | translation + unit quaternion + scale, 표현 가능성 조건 명시 |
| affine transform | 3x3 linear + translation 또는 3x4 matrix, scale/shear/reflection 포함 |
| mat4 interop | storage order, vector convention과 handedness가 고정된 변환 buffer |

* 확인된 수학·API 결함

** Scale, composition과 inverse

현재 layout을 =p' = t + R(S p)=로 해석하면 다음 문제가 있다.

1. =xform_apply_to_point=와 inverse는 =scale=을 전혀 사용하지 않는다.
2. =xform_apply_to_direction=은 scale을 무시한 뒤 결과를 무조건 unit vector로 만들어 입력
   magnitude 정보까지 잃는다. Free vector와 direction의 의미가 섞였다.
3. =xform_mul=은 child translation에 parent scale을 적용하지 않는다. 또한
   =R_a S_a R_b S_b=는 일반적으로 =R_a R_b(S_a S_b)=가 아니며 non-uniform scale과
   회전의 composition은 shear를 만들 수 있다.
4. =xform_inverse=은 translation에 inverse scale을 적용하지 않는다. 더 근본적으로
   =(R S)^-1 = S^-1 R^-1=은 일반 non-uniform scale에서 같은 =R' S'= 순서의 TRS로
   정확히 표현되지 않는다.
5. =xform_to_mat4=만 scale을 쓰므로 같은 =xform_t=를 point 함수와 matrix 함수에 적용한
   결과가 다르다.
6. zero/near-zero/negative/non-finite scale을 허용하면서 inverse, handedness와 normal
   변환의 실패 계약이 없다.

Canonical API는 rigid와 affine을 정확히 닫힌 연산으로 제공하고, TRS composition/
inverse는 affine 결과를 반환하거나 representability 검사 뒤 status를 반환한다.

** Alias와 초기화

- =xform_mul(out,a,b)=는 =out==a= 또는 =out==b=일 때 rotation을 먼저 덮어 이후 입력을
  바꿀 수 있다. Alias 허용 여부가 선언에 없다.
- =xform_inverse(out,src)=도 in-place가 명시되지 않고 하위 quaternion 함수의 alias
  우연에 의존한다.
- =xform_init_axis_angle=과 =xform_init_euler=은 =xform_init(NULL)=가 반환한 뒤 =out->pos=
  를 계속 읽을 수 있다.
- =xform_equal=, =xform_from_forward=, pivot/look/align/reflect 계열 일부는 NULL 검사가 없거나
  서로 다른 no-op 정책을 가진다.
- =xform_look_at=과 =xform_align_vectors=는 =pos/rot= 일부만 쓰고 =scale=을 초기화하지 않아
  fresh output에 미정 값이 남는다.
- =xform_look_to=는 =xform_from_forward=를 호출해 기존 position을 0으로, scale을 1로
  바꾼다. “orientation만 수정”인지 “새 transform 생성”인지 이름과 구현이 충돌한다.
- =xform_slerp_pivot=은 =b->pos=를 전혀 사용하지 않고 =a->pos-pivot=을 보간된 절대
  rotation으로 돌린다. 따라서 일반 입력에서 =t=1= endpoint가 =b=가 아니며 보간의 기본
  endpoint invariant를 만족하지 않는다.
- =xform_rotate_around_pivot=과 =xform_rotate_local_around_pivot=은 모두 =q*old_rot= 순서를
  사용한다. Local 이름이 다른 multiplication order를 뜻하는지, pivot 위치만 local인지
  정의가 없고 후자는 quaternion normalization도 하지 않는다.

Canonical 함수는 status를 반환하고 모든 output field 초기화, input/output alias matrix,
실패 시 output 불변을 함수 family별로 고정한다.

** Look-at와 orientation 퇴화

- eye와 target이 같을 때 forward가 zero다.
- up이 zero이거나 forward와 평행/반평행이면 right가 zero다.
- local forward가 +Z인지 -Z인지, 좌/우 handedness와 row/column vector convention이 header에
  없다. 현재 test 일부의 주석과 실제 기대 축도 일관되지 않는다.
- =xform_align_vectors=의 parallel/antiparallel 및 zero vector 결과가 하위 quaternion
  구현에 암묵적으로 맡겨진다.
- 반복 rotation 후 normalization이 함수마다 다르다. pivot 한 함수만 normalize하고 다른
  multiply 경로는 하지 않는다.
- Euler round-trip은 gimbal lock, angle canonical range와 여러 등가해 선택을 정의하지
  않는다.

Degenerate look/align은 zero quaternion이나 NaN을 만들지 않고 invalid/degenerate status,
caller fallback-up 또는 deterministic fallback-axis 중 승인된 정책을 사용한다.

** Clamp, matrix, reflection과 출력

- Header는 position이 항상 =[-99999,99999]=라고 말하지만 clamp는 init/set/translate/
  look-at 일부에만 있다. mul, inverse, interpolation, pivot와 reflect는 clamp하지 않는다.
- =fminf/fmaxf= clamp는 NaN을 명시 오류로 보고하지 않고 경계값으로 바꿀 수 있다.
- 같은 public macro를 consumer와 DLL이 다르게 compile하면 world 범위 계약이 갈라진다.
- =xform_to_mat4=의 row/column-major, row/column-vector, handedness와 translation index 계약이
  선언에 없다.
- =xform_reflect=는 plane normal을 unit으로 검증/정규화하지 않는다. 세 basis vector를
  reflect하면 determinant -1의 improper transform인데 이를 quaternion으로 다시 만들며
  reflection/parity 정보를 잃을 수 있다.
- =xform_print=는 public C ABI 구현에서 C++ =std::cout=과 process locale/global stream을
  사용하고 buffer, 오류, thread/interleaving 계약이 없다.

* 업계 기준선

- [[https://libeigen.gitlab.io/eigen/docs-nightly/group__TutorialGeometry.html][Eigen Geometry]]는
  rotation/translation/scaling 같은 추상 변환과 일반 affine matrix를 구분한다. 일반
  affine에서 point는 전체 변환, vector는 linear part, normal은 inverse-transpose를
  적용한다. BYUL도 point/direction/normal을 한 함수 의미로 섞지 않는다.
- Eigen은 일반 affine을 =(Dim+1)^2= matrix로 표현하고 composition/inverse를 그 닫힌
  표현에서 수행한다. Non-uniform TRS가 shear를 만드는 경우 BYUL도 affine으로 승격하거나
  명시적 non-representable status를 반환한다.
- [[https://learn.microsoft.com/en-us/windows/win32/dxmath/ovw-xnamath-reference-functions-matrix][DirectXMath matrix functions]]는 LH/RH look-at/look-to를 이름으로 구분하고 row-major
  convention을 명시하며 affine, inverse, scaling과 reflection matrix를 별도 연산으로
  제공한다. BYUL canonical API도 handedness/storage/reflection 표현을 암묵적으로 두지
  않는다.
- Quaternion은 compact rigid rotation에, matrix/affine은 scale·shear·reflection을 포함한
  일반 선형 변환에 사용한다. Dual quaternion interpolation은 rigid transform에만 적용하고
  별도 scale 선형 보간을 동일한 “slerp” 의미로 숨기지 않는다.

* 목표 Math Engine 계약

** 좌표와 단위

- position과 translation: 선택된 length unit, 같은 frame
- scale: dimensionless; zero/singular, negative/reflection과 finite 규칙 명시
- axis/direction/normal: dimensionless, unit 여부를 type/function 계약으로 구분
- angle: radians canonical, degree convenience API는 명시 suffix와 중앙 상수 사용
- matrix: 저장 순서, vector side, multiplication order, translation slot과 handedness 동결

Math Engine은 world unit을 고정하지 않지만 한 호출의 모든 length가 같은 단위를 쓴다는
조건을 요구한다. Balix가 world frame/unit와 bounds를 선택한다.

** Canonical family

최종 prefix는 Build/ABI master에서 동결하되 의미 family는 다음처럼 분리한다.

#+begin_src text
math_rigid3f_*       translation + unit quaternion, closed rigid algebra
math_trs3f_*         decomposed convenience plus representability status
math_affine3f_*      full affine composition/inverse/application
math_transform_*_point
math_transform_*_vector
math_transform_*_normal
math_transform_*_look_at_lh / math_transform_*_look_at_rh
#+end_src

- Rigid inverse/composition은 exact closed operation이다.
- TRS point 적용은 =t + R(S p)=처럼 승인한 순서를 header와 test에 고정한다.
- TRS composition/inverse가 shear 때문에 표현 불가능하면 affine result를 사용하거나
  =NOT_REPRESENTABLE=을 반환한다.
- Vector는 translation을 제외하되 magnitude를 보존하는 linear transform이고 direction은
  필요할 때 별도 normalize한다. Normal은 inverse-transpose와 singular status를 사용한다.
- All-output 함수는 성공 전에 local temporary로 계산해 승인된 alias 조합을 안전하게
  지원한다.

** Status와 수치 정책

Canonical C API는 최소한 invalid pointer, non-finite, zero axis, degenerate look basis,
singular, not representable와 invalid enum을 구분한다. 정상 계산의 identity/zero 값과 오류를
같은 값으로 표현하지 않는다.

- quaternion unit drift는 context tolerance에서 검사하고 normalize 여부를 policy로 정한다.
- look/align의 angular tolerance와 scale singular tolerance를 분리한다.
- comparison은 absolute/relative component tolerance와 rotation angular tolerance를
  구분한다.
- =t= 보간의 =[0,1]= clamp/extrapolation 정책, quaternion shortest path/hemisphere와 scale이
  zero를 통과할 때 singular 상태를 명시한다.
- 동일 입력/context/backend에서 결과 bit 또는 허용 오차 계약이 결정적이어야 한다.

* Balix compatibility, clamp와 출력

- =xform_t=의 field 순서, =sizeof/alignof/offsetof=와 기존 48개 =BYUL_API= symbol은 ABI
  1.x에서 유지한다. Canonical Math Engine type과 layout이 같더라도 typedef alias 여부는
  ABI/header review 뒤 결정한다.
- 기존 setter/translation의 =XFORM_MIN_POS/MAX_POS= saturation은 Balix legacy adapter가
  canonical raw 계산 뒤 적용한다. Math Engine 값은 숨은 clamp를 하지 않는다.
- clamp가 빠져 있던 기존 함수에 무조건 새 saturation을 넣지 않는다. 함수별 관찰 동작과
  문서 약속의 충돌을 compatibility decision으로 기록한다.
- =xform_print=는 legacy Balix debug shim으로 남기고 canonical formatter가 필요하면
  caller buffer/capacity/actual count, UTF-8/locale와 truncation status를 제공한다. Math
  Engine DLL은 =iostream= 또는 process stream을 link하지 않는다.
- =entity_dynamic_t= 등 public embedding type은 기존 layout을 보존하고 내부 계산부터
  canonical API로 전환한다. Opaque 전환은 별도 ABI-major 계획 없이 실행하지 않는다.

* 필수 검증 corpus

** Algebra와 scale

- identity, pure translation/rotation/uniform scale와 non-uniform scale
- point/vector/direction/normal을 같은 affine matrix oracle과 비교
- =inverse(T)(T(p))=, =T(inverse(T)(p))=와 composition associativity tolerance
- parent scale이 child translation에 미치는 known-answer
- non-uniform scale + rotated child가 만드는 shear와 =NOT_REPRESENTABLE=/affine 결과
- zero, near-zero, negative, NaN와 infinite scale
- =out==a=, =out==b=, =out==src= 및 서로 다른 input/output alias permutation

** Orientation, interpolation과 reflection

- eye==target, zero up, parallel/antiparallel up와 near-degenerate basis
- LH/RH, +Z/-Z forward, matrix/quaternion basis와 look-at known-answer
- from==to, from==-to, zero vector align과 axis enum invalid
- Euler 모든 지원 order, gimbal lock와 canonical angle range
