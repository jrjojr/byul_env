=numeq_model.h=의 직접 =BYUL_API= 선언은 정확히 11개다. 아래 표는 모든 선언을
개별 행으로 기록한다. Canonical 계산은 전부 Math Engine 소유이고 기존
위 표의 기존 11개 이름은 Balix/노출 root의 legacy forwarding ABI다.

| =BYUL_API= 선언 | 정본 분류 | Math Engine component | 기존 symbol의 최종 운명 |

| =numeq_model_accel_except_gravity= | Math Engine pure calculation/value | =physics/dynamics= named force composition | Balix legacy forwarding |
| =numeq_model_accel_predict= | Math Engine pure calculation/value | =physics/dynamics= state evaluation | Balix legacy forwarding |
| =numeq_model_pos_predict= | Math Engine pure calculation/value | =physics/kinematics= constant-acceleration query | Balix legacy forwarding |
| =numeq_model_vel_predict= | Math Engine pure calculation/value | =physics/kinematics= velocity query | Balix legacy forwarding |
| =numeq_model_predict= | Math Engine pure calculation/value | =physics/kinematics= state query | Balix legacy forwarding |

Balix adapter는 =linear_state_t=, =environ_t=, =bodyprops_t=를 neutral state/force/contact
parameter로 변환한다. 위 세 Balix type을 Math Engine public header에 노출하지 않는다.

* Private 계산 inventory

=numeq_model.cpp=의 export 밖 계산도 모두 Math Engine manifest에 포함한다.

| 현재 private/static 구현 | 목표 소유권 | 이유 |
|--------------------------+-------------+------|
| =compute_drag_accel= | Math Engine pure calculation/value | =-0.5*rho*Cd*A*|v|v/m=의 핵심 drag 식이다. |
| =numeq_model_accel_internal= | Math Engine pure calculation/value | gravity, drag와 environment distortion의 합성 순서를 결정한다. |
| =numeq_model_accel_except_gravity_internal= | Math Engine pure calculation/value | named force composition과 gravity 제외 의미를 결정한다. |

그 밖에 =scalar_zero=, =scalar_safe_div=, =vec3_point_plane_distance=,
=vec3_ray_plane_intersect=, =numeq_solve_quadratic=와 integrator 호출은 canonical Math Engine
component 사이 의존성으로 바꾼다. Balix adapter가 이를 직접 조합하지 않는다.

* 공식·상수와 output 계약 inventory

- drag: =0.5 * air_density * |v_rel|^2 * Cd * area / mass=, null environment의
  density fallback =1.225f=
- constant acceleration: =v=v0+a0*t=, =p=p0+v0*t+0.5*a0*t^2=
- bounce: normalized normal에 대해 tangential 보존, normal 성분을
  =-restitution=으로 반전하고 restitution을 [0, 1]에 clamp
- sphere query: =|p_rel+v_rel*t|^2-radius_sum^2=0=의 quadratic; relative acceleration이
  nonzero이면 현재 false
- plane query: distance/radius 초기 overlap과 =plane_point-normal*radius= offset의
  ray-plane intersection; =SCALAR_EPSILON= parallel gate

=1.225f=처럼 물리 표준값으로 보이는 값은 온도·압력·습도 조건과 출처 없이 보편 상수로
승격하지 않는다. Null environment fallback은 Balix legacy preset으로 보존하고 canonical
Math Engine은 명시 parameter 또는 출처가 있는 atmosphere model을 요구한다.

* Build·DLL 이행

현재 =numeq_model.cpp=는 =balix_SOURCES=와 root shared source inventory에 포함되며
Balix 내부 integrator/body/environment type에 직접 의존한다.

- Static 과도기: pure source는 neutral API 기준 단일 Math Engine manifest로 이동하고
  Balix는 11개 DTO/status forwarding shim만 소유한다. Root shared는 현행 중앙 규칙대로
  같은 manifest를 직접 compile하고 기존 export를 보존한다.
- 독립 DLL 전환: 중앙 규칙과 =rules-ref.org= revision 승인 뒤 canonical physics,
  ODE와 CCD symbol은 Math Engine DLL에만 존재한다. BYUL DLL의 11개 기존 symbol은
  imported Math Engine 호출로 forwarding하며 math source를 중복 포함하지 않는다.

* 현재 진단

- Drag 식은 방향상 =-0.5*rho*Cd*A*|v|v/m=이지만 body mass zero/negative, density/Cd/
  area range와 unit status가 없다.
- Environment distortion과 drag/gravity composition 순서가 model 이름에서 드러나지 않는다.
- =accel_predict(time>0)=는 예측 velocity로 internal acceleration을 계산하면서
  =state0.acceleration=을 누락해 =time<=0= 경로와 불연속이다.
- Position/velocity 예측은 acceleration을 초기값에서 한 번 계산하지만 drag가 velocity에
  의존하므로 closed-form처럼 보이는 이름과 실제 근사가 다르다.
- Velocity prediction에 friction factor와 acceleration을 동시에 적용해 model
  interpretation과 energy가 불명확하다.
- Bounce는 tangential velocity를 보존하는 frictionless point response이며 normal
  orientation/normalization과 separating contact 처리 계약이 없다.
- Sphere collision은 상대 acceleration이 0이 아닐 때 문서 기대와 달리 false를 반환한다.
- Plane collision의 radius offset 방향이 motion/normal side에 따라 일반적으로 달라질 수
  있으나 고정 =-normal*radius=를 사용한다.
