* 조사 범위와 경계

공개 선언과 구현을 다음 파일에서 직접 대조한다.

- 공개 header: =byul/balix/numeq/numeq_mpc.h=
- 구현: =byul/balix/numeq/numeq_mpc.cpp=
- Balix binding caller: =byul/balix/controller/controller.h=,
  =byul/balix/controller/controller.cpp=
- 현재 test: =byul/balix/tests/test_module_mpc.cpp=,
  =byul/balix/tests/test_module_controller.cpp=,
  =byul/projectile/tests/test_module_propulsion.cpp=

Cost, dynamics rollout, candidate evaluation와 optimizer는 Math Engine =control/mpc=/
=numeq/optimization=의 순수 kernel이다. Options/state/control/result는 Math Engine value/API다.
Waypoint progression, route cursor/loop, direction intent를 target state로 바꾸는 정책, tick
budget, environment/body snapshot binding과 actuator 적용은 Balix다. 현재 공개 solver는
=motion_state_t=, =environ_t=, =bodyprops_t=, =trajectory_t=에 직접 결합하므로 canonical
Math Engine solver가 아니라 이 둘을 섞은 legacy entry다. 독립 DLL 전환 뒤 기존 이름은
neutral Math Engine API를 호출하는 Balix forwarding/adapter로 남긴다.

* BYUL_API 누락 공개 선언

다음 두 함수는 같은 public header에 선언되고 구현도 있지만 =BYUL_API=가 없다. 따라서
header의 함수 surface는 17개지만 export 계약을 만족하는 선언은 15개다.

| 정확한 symbol | 현재 상태 | 분류와 이관 | 검증 |
|-------------+-----------+-------------+------|
| =mpc_direction_target_init_full= | 선언·definition 존재, =BYUL_API= 누락 | BALIX-BINDING / LEGACY-FORWARD. nullable direction/orientation, normalization과 invalid duration 정책을 가진 Balix intent initializer로 정리한다. | Windows shared export, null direction/orientation, full field mapping, wrapper audit와 clean C consumer를 검사한다. |
| =mpc_direction_target_assign= | 선언·definition 존재, =BYUL_API= 누락 | BALIX-BINDING / LEGACY-FORWARD. value copy와 self-assign/null 계약을 고정한다. | Windows shared export, all fields, self-assign/null, wrapper audit와 clean C consumer를 검사한다. |

=mpc_config_t=, =mpc_target_route_t=, =mpc_direction_target_t=, =mpc_output_t=와
=mpc_cost_func=도 public ABI surface다. 기존 layout에 status, workspace, route cursor나 result
statistics를 끼워 넣지 않고 versioned Math Engine type을 새로 만든다. =mpc_output_t=의
=future_state=와 =trajectory_t= storage는 Balix type이므로 canonical Math Engine result는
neutral state/control buffer와 solution provenance를 사용한다.

* 현재 진단

- =mpc_direction_target_init_full/assign=은 구현됐지만 =BYUL_API=가 없어 Windows export
  hole이다.
- horizon seconds와 step dt의 차원을 step count로 올바르게 변환하지 않아 기본 horizon이
  사실상 한 step이 될 수 있다.
- =max_iter=, candidate step과 angular candidate option 일부가 사용되지 않는다.
- Null cost callback이면 모든 cost가 zero가 될 수 있어 첫 candidate를 성공처럼 선택한다.
- Trajectory output을 요청하지 않을 때 =future_state=가 실제 best prediction이 아니라
  target copy가 되는 경로가 있다.
- Fast solver는 한 축 error 값을 여러 축 acceleration/rotation에 재사용하는 indexing
  defect가 있다.
- Cost는 position/velocity/orientation의 서로 다른 단위를 weight로 합치지만 weight
  dimension과 normalization이 없다.
- Exhaustive candidate search는 fixed 729 candidates를 budget/cancellation 없이 순회한다.
  =Eigen/Dense=와 =vector= include는 현재 미사용이고, trajectory storage allocation도 caller
  workspace/budget 계약과 분리되지 않았다.
- =quat_angle_diff=는 relative quaternion을 normalize하지 않고 =abs(w)=를 =[0,1]=로
  clamp하지 않은 채 =acos=에 전달해 drift·invalid quaternion에서 NaN cost가 가능하다.
  이 식은 공용 quaternion geodesic primitive로 추출하고 clamp된 acos 또는
  =2*atan2(length(v), abs(w))=와 zero/non-finite 상태를 비교해야 한다.
- =numeq_mpc_solve_route=는 route 진행 순서가 아니라 현재 위치에서 가장 가까운 point를
  매 호출 선택하므로 되돌아가기와 waypoint skip이 가능하다. 이는 optimizer가 아니라
  Balix route progression 정책이어야 한다.
- Directional target은 direction 길이를 검증·정규화하지 않고 duration, =step_dt=와
  =max_speed=를 곱해 target displacement를 만든다. Direction magnitude가 숨은 speed
  scale이 되고 duration/horizon 의미도 중복될 수 있다.
- =numeq_mpc_solve=, =numeq_mpc_solve_fast=와
  =numeq_mpc_solve_coarse2fine=은 =horizon_sec= float를 =int=로 truncate한 뒤 seconds가
  아니라 loop count로 사용한다. Header의 =horizon_sec / step_dt= 계약과 구현·현재 test가
  서로 다르다.
- 세 solver는 =max_speed/max_ang_speed=를 실제 rollout에 적용하지 않는다. 이 clamp는
  호출되지 않는 =simulate_trajectory=에만 있다. =candidate_step=,
  =ang_candidate_step=, =max_iter=도 후보 생성/반복에 쓰이지 않는다.
- =numeq_mpc_cost_speed=는 target speed로 velocity magnitude가 아니라
  =target->linear.velocity.x=를 사용하고 speed error에 =weight_velocity=가 아니라
  =weight_distance=를 적용한다.
- Coarse-to-fine의 fine acceleration은 이미 =max_accel=인 coarse 해에 =25%=를 더해
  bound의 125%까지 나갈 수 있고 angular acceleration은 fine phase에서 전혀 탐색하지 않는다.
- =numeq_mpc_solve_route=는 =route->loop=를 사용하지 않고 =route->points=,
  =current_state=, =config=를 검증하지 않는다. =numeq_mpc_solve_directional=도
  =current_state/config= null을 검사하기 전에 역참조하고 =weight_dir/weight_rot=을 사용하지
  않는다.
- =mpc_direction_target_init_full=은 =direction/orientation= null을 검사하지 않고 즉시
  역참조한다. Route assign은 borrowed pointer shallow copy지만 public lifetime 계약이 없다.
- Trajectory sample time은 한 step 적분한 상태를 =t=0으로 기록한다. Sample allocation
  실패나 zero count에서는 =future_state=가 초기화되지 않아도 solver가 =true=를 반환할 수
  있다.
