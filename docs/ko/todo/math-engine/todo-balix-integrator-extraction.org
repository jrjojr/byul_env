* 분리 원칙

28개 공개 symbol을 다음처럼 분리한다.

- Math Engine =numeq/ode=: callback+array Euler, midpoint/RK, adaptive 후보와 error control
- Math Engine =physics/dynamics=: translation/rotation state와 quaternion/dual-quaternion step
- Balix adapter: =motion_state_t/environ_t/bodyprops_t= 변환, integrator selection과 state 보관
- Legacy: 위 28개 기존 public symbol과 enum의 compatibility forwarding

Environment suffix 함수가 수학 공식을 직접 소유하지 않게 하고 acceleration/torque callback을
주입한다.

분류는 최종 책임을 뜻한다. =Math Engine 순수 계산=으로 분류한 기존 심볼도
=motion_state_t=를 노출하므로 neutral state/view를 받는 canonical kernel을 추출하고 기존
=BYUL_API=는 Balix 변환·forwarding 진입점으로 유지한다. 새 canonical 함수명, status enum과
workspace layout은 Math Engine public-header 설계 전에는 확정하지 않는다.

* 현재 진단

- =byul/balix/tests/test_module_integrator.cpp=의 7개 test case가 직접 부르는 공개 심볼은
  =integrator_init=, =integrator_init_full=, =integrator_step=, =integrator_step_euler=,
  =integrator_step_semi_implicit=, =integrator_step_verlet=,
  =integrator_step_motion_rk4=뿐이다. 나머지 21개 공개 심볼은 per-symbol regression이 없다.
- 많은 함수가 =dt <= 0=, null과 non-finite를 일관되게 검사하지 않고 =assert=가 public
  failure 계약을 대신한다.
- =integrator_step_motion_verlet=은 header의 =prev_state const=와 source의 non-const
  signature가 다르다. Source는 previous state를 덮은 뒤 angular previous 값을 읽는다.
- =integrator_step=은 null을 검사하지 않고 =type=을 먼저 읽으며, unknown enum은 public
  failure status가 아니라 =assert(false)=에 의존한다. 일반 position-Verlet branch는
  =prev_state=를 다음 history로 갱신하지 않아 반복 step의 상태 계약도 불완전하다.
- =integrator_step_rk4=는 acceleration이 상수라 RK4가 closed-form 수준의 weighted Euler와
  같고, 이름이 일반 RK4 능력을 암시한다.
- Environment RK4 중간 stage에서 position을 갱신하지 않는 경로가 있어 position-dependent
  force에 일반적으로 맞지 않다.
- =integrator_step_motion_rk4_env=도 intermediate state의 velocity만 바꾸고 position은
  원래 값을 유지하며, angular 쪽은 environment/torque를 stage별 재평가하지 않는다.
- Velocity Verlet에서 velocity-dependent drag를 v0로 평가하여 symplectic/Verlet 가정과
  충돌한다.
- Angular drag가 inertia/torque 없이 mass와 area로 acceleration coefficient를 만들며
  물리 모델이 integrator에 섞였다.
- Quaternion update의 frame convention, multiplication order, normalization drift와
  local truncation error가 API에 없다.
- =integrator_step_attitude_euler=와 =integrator_step_attitude_semi_implicit=은 둘 다
  angular velocity를 먼저 갱신한 뒤 그 값으로 orientation을 갱신해 현재 구현상 같은
  update order다. 두 public method 이름이 서로 다른 알고리즘을 약속하지만 결과가 같다.
- =integrator_t=가 state/env/body를 값으로 보유해 generic numerical integrator와 domain
  lifecycle이 결합됐다.
- =integrator_init=은 =INTEGRATOR_RK4_ENV=를 선택하면서 =env/body=를 단순 zero-init한다.
  이 값은 유효한 기본 환경·질량인지 보장되지 않으며 header가 설명하는 default와 실제
  force 경로가 일치하지 않는다. =integrator_clear=도 같은 invalid-default 상태로 되돌린다.
- =integrator_init_full=은 required =state=를 null 검사 없이 역참조하고 optional
  =prev_state/env/body= assign이 null에서 no-op이면 destination field가 미초기화로 남을 수
  있다. Method별 필수 입력 검증과 canonical zero/default state가 필요하다.

* Industry 기준

비교 기준은 [[https://computing.llnl.gov/projects/sundials/cvode][LLNL SUNDIALS CVODE]]의
explicit =y'=f(t,y)= callback, method 선택, tolerance, one-step/normal 반환과 solver
status 계약이다. BYUL이 CVODE 구현을 복제한다는 뜻이 아니라 책임과 결과 계약의
성숙도를 비교한다.

- ODE callback =f(t,y)=와 caller-owned state/workspace
- method, tolerance, max step/evaluation, cancellation과 dense output 분리
- fixed-step method는 차수와 stability region을 문서화
- velocity-dependent force에는 일반 ODE 또는 명시된 splitting 사용
- rigid rotation은 quaternion derivative frame convention과 normalization policy 명시
- step success/rejected/underflow/non-finite를 status로 반환
