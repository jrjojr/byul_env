| Header | Math Engine 역할 | 핵심 판단 |
|--------+------------------+-----------|
| =numal.h= | aggregate | 승인 component inventory |
| =scalar.h= | scalar | generic macro, pi/angle, tolerance와 checked result |
| =dualnumber.h= | automatic differentiation value | layout와 derivative dimension |
| =vec3.h= | vector value | tolerance, aliasing, buffer/string 분리 |
| =quat.h= | rotation value | enum 고정, normalization와 frame |
| =dualquat.h= | rigid transform value | tag/layout와 output order |

Header별 함수 수준 리팩터링은

declaration, definition, root export, install, wrapper와 test를 연결한다.

각 initial asset과 created successor record에 declaration, definition, export, install,
wrapper, test와 =design_todo=를 연결한다. Numal canonical 이름, 수학 의미와 successor
header뿐 아니라 기존 include/symbol forwarding과 C++ façade compatibility까지 이
문서가 소유하며 별도 manifest를 만들지 않는다.

* Math Engine 완결 소유권

Numal 11개는 다른 consumer module의 stateful DTO가 아니라 Math Engine 자체의 기존
계산 header다. 따라서 canonical API, 기존 ABI/layout, install/export, deprecated
forwarding과 C++ façade disposition을 모두 이 문서에서 완료한다. Runtime/Simulation
consumer TODO는 승인된 Numal/Math Engine API를 사용할 뿐 Numal compatibility를 소유하지
않는다.

* Navsys dependency 제외

Navsys는 Numal/Math Engine consumer가 아니다. 현재 Navsys에서 발견되는 =scalar.h=,
=scalar_equal=, =M_PI=와 CMake =numal= link는 이 문서가 보존할 compatibility surface가
아니라 Navsys 선행 리팩터링에서 제거할 legacy dependency다. 새 scalar/value header,
aggregate, install target과 consumer manifest에 Navsys를 추가하지 않는다.

* 상위 Numal 계획 합류 Gate

이 문서의 네 단계는 [[file:todo-numal.org][Numal component 계획]]의 별도 구현 track이
아니다.
