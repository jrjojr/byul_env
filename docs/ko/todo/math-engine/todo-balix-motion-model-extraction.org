=numeq_model_motion.h=의 직접 =BYUL_API= 선언은 정확히 5개다. 공식 본체는 모두
Math Engine =physics/aerodynamics=, =physics/dynamics= 또는 =numeq/ode= 소유이고 기존
이름은 legacy forwarding ABI로 남긴다.

| =BYUL_API= 선언 | 정본 분류 | Math Engine component | 기존 symbol의 최종 운명 |
|-----------------+-----------+-----------------------+--------------------------|
| =numeq_model_motion_predict= | Math Engine pure calculation/value | =physics/dynamics= state prediction | Balix legacy forwarding |
| =numeq_model_motion_predict_rk4= | Math Engine pure calculation/value | =numeq/ode=와 =physics/dynamics= 합성 | Balix legacy forwarding |
| =numeq_model_motion_accel= | Math Engine pure calculation/value | =physics/aerodynamics= force composition | Balix legacy forwarding |
| =calc_spin_accel= | Math Engine pure calculation/value | =physics/aerodynamics= spin force/acceleration | Balix legacy forwarding |
| =numeq_model_motion_drag_scale= | Math Engine pure calculation/value | =physics/aerodynamics/legacy-empirical= | Balix legacy forwarding |

Balix는 =motion_state_t=, =environ_t=, =bodyprops_t=와 simulation preset을 neutral
parameter/result로 변환한다. Math Engine public header는 이 Balix type을 include하지 않는다.

* Formula·상수 전수 inventory

공식과 epsilon은 Math Engine source/context에 한 번만 존재한다. Legacy heuristic parameter는
Balix versioned preset에서 canonical Math Engine empirical API로 값만 전달하고 adapter가
분기나 clamp 식을 재작성하지 않는다.

* 구현 결합과 결함

- =numeq_model_motion_predict=는 linear prediction 뒤 acceleration 전체에 drag scale을
  적용하고 spin acceleration을 더하지만 angular state는 복사만 한다.
- =numeq_model_motion_predict_rk4=는 초기화되지 않은
  =out_state->linear.acceleration=을 drag scale로 먼저 변경한 뒤, 실제 integration에는
  별도 =temp.acceleration=을 사용한다.
- =numeq_model_motion_accel=은 base acceleration과 spin acceleration을 합성한다.
- =calc_spin_accel=은 =time <= 0=일 때 시간과 무관한 Magnus 항까지 zero 처리한다.
- 모든 export는 외부 state를 소유하거나 tick을 예약하지 않는 단발 계산이므로 Balix
  orchestration으로 남길 근거가 없다.

* Build·DLL 이행

현재 =numeq_model_motion.cpp=는 =balix_SOURCES=와 root shared source inventory에 직접
포함되고, =numeq_model.h=와 Balix DTO에 결합돼 있다.

- Static 과도기: neutral aerodynamic/dynamics source와 위 공식·literal은 단일 Math
  Engine manifest로 이동한다. Balix source에는 5개 DTO/status forwarder만 남고 entity,
  MPC와 Balix가 =byul_math_engine_static=을 target link로 소비한다. Root shared는 현행
  중앙 규칙 동안 같은 manifest를 직접 compile한다.
- 독립 DLL 전환: 중앙 규칙과 =rules-ref.org= revision 승인 뒤 canonical 5개 계산은
  Math Engine DLL에만 존재한다. BYUL DLL의 기존 이름은 legacy forwarding shim이며
  canonical source/symbol을 중복 포함하지 않는다.

* 현재 진단

- =calc_spin_accel=의 Magnus-like =k_magnus*(omega x v)=와
  gyro =k_gyro*time*(alpha x v)= coefficient 차원, density, size와 mass dependency가 없다.
- =time <= 0=이면 spin acceleration 전체를 zero로 만들어 time과 무관한 Magnus 항까지
  사라진다.
- drag scale의 max spin 50, reduction 30%, penalty 15%, alignment 0.5와 clamp [0,2]는
  보편 법칙이 아닌 tuning heuristic이다.
- =predict_rk4=가 초기화되지 않은 =out_state->linear.acceleration=을 먼저 scale한다.
- Predict 함수가 angular orientation/velocity를 적분하지 않고 복사하면서 이름이 full
  motion prediction처럼 보인다.
- Drag scale을 acceleration 전체에 적용하면 gravity와 사용자 acceleration도 scale될 수
  있어 aerodynamic drag correction 의미와 맞지 않는다.
