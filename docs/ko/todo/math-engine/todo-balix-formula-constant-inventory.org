* 목적

=byul/balix=의 18개 production =.cpp=와 18개 public header에서 공개 함수뿐 아니라
private helper, inline 식, tolerance, 상수, empirical coefficient, default tuning과
sentinel까지 전수 추적한다. Public symbol 목록만 옮기고 private 식이나 숫자 literal을
Balix에 남기는 누락을 막는 coverage 문서다.

순수 계산식은 Math Engine owner로 이동한다. Tick, 감시, target 선택, callback binding,
storage와 event 정책은 Balix에 남는다. 숫자라는 이유만으로 모두 공용 상수가 되지는
않으며 출처와 의미에 따라 다음 owner를 고른다.

* Industry 검증 기준

- 공식은 변수 정의, 좌표계, 차원, 가정과 유효 범위를 갖는다.
- Closed form은 분석적 known answer와 residual을, iterative method는 bracket/convergence/
  iteration·evaluation budget과 failure status를 갖는다.
- 미분·적분은 차수, local/global error, stability와 step-halving convergence를 기록한다.
- 물리 모델은 dimensional analysis, limiting case, conservation/dissipation과 empirical
  parameter provenance를 검증한다.
- Geometry/CCD는 degenerate, tangent, coincident, initial overlap, moving/rotating frame,
  feature classification과 scale-aware tolerance corpus를 갖는다.
- Controller/estimator는 discrete-time model, units, saturation/covariance invariant,
  deterministic workspace와 invalid/non-finite status를 갖는다.
- Public C ABI 밖으로 C++ exception, STL/backend type 또는 생성 DLL과 다른 allocator의
  해제가 새지 않는다.

공식 reference baseline:

- [[https://computing.llnl.gov/projects/sundials/cvode][LLNL SUNDIALS CVODE]]: ODE를
  =y'=f(t,y)= callback, method, tolerance와 solver 상태로 분리하는 기준
- [[https://doc.cgal.org/latest/Kernel_23/index.html][CGAL 2D/3D Kernel manual]]:
  exact/filtered predicate와 construction 정확도를 분리하고 robust geometry kernel을
  선택하는 기준
- [[https://physics.nist.gov/cuu/Constants/index.html][NIST/CODATA Fundamental Physical
  Constants]]: 상수값에 edition, 단위와 uncertainty를 함께 보존하는 기준
- [[https://www.boost.org/doc/libs/latest/libs/math/doc/html/index.html][Boost.Math]]: root,
  interpolation, quadrature/differentiation와 precision/error policy 비교 기준
- [[https://cmake.org/cmake/help/latest/command/target_link_libraries.html][CMake
  target_link_libraries]]와
  [[https://cmake.org/cmake/help/latest/command/add_library.html][add_library]]: source 흡수 대신
  target dependency와 static/shared/imported library를 표현하는 build 기준
