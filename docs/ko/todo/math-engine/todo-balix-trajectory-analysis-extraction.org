* 분리 원칙

Trajectory allocation, sample/history storage, impact metadata, formatting과 raw export는 Balix가
소유한다. 보간, 미분, norm, time/arc-length resampling과 explicit cut 계산은 Math Engine의
sampled-series 계산이다. 새 canonical API는 =trajectory_t= 대신 finite monotonic abscissa와
strided point/value view, caller-owned output을 받는다.

분류가 =Math Engine 순수 계산=인 기존 심볼은 =trajectory_t/motion_state_t=를 노출하므로
그 선언 자체를 Math Engine DLL 계약으로 옮기지 않는다. Balix가 neutral view를 구성해
canonical kernel을 호출하고 기존 =BYUL_API=는 legacy forwarding으로 유지한다. 구체적인 새
함수명, interpolation kind와 result status는 public-header 설계 전에 임의로 정하지 않는다.

* 현재 진단

- =byul/balix/tests/test_module_trajectory.cpp=의 3개 test case가 직접 부르는 공개 심볼은
  =trajectory_create_full=, =trajectory_destroy=, =trajectory_add_sample=,
  =trajectory_clear=뿐이다. 나머지 20개 공개 심볼과 모든 보간·미분·resampling 경로는
  direct regression이 없다.
- sample time의 strict monotonicity, duplicate time, NaN과 unit/frame 계약이 없다.
- acceleration의 =(v1-v0)/((dt0+dt1)/2)=는 중앙 sample에서의 표준 비균일 3-point
  second derivative와 같은 식이다. 그러나 임의 query =t=를 evaluation point로 쓰지 않고
  =s0.t <= t <= s2.t= 전 구간에 같은 값을 반환하며 =dt0/dt1=의 zero·negative·non-finite
  검사가 없다.
- boundary에서는 저장된 velocity/acceleration을 반환하고 내부에서는 position difference를
  사용해 estimator 의미가 불연속이다.
- time resampling은 floating loop 누적오차와 impact point 중복/비정렬 가능성이 있다.
- distance resampling은 zero-length segment와 carry 의미가 API에 없고, spacing이 마지막
  점 또는 impact cut에 정확히 닿아 이미 출력했어도 마지막/cut 점을 다시 append할 수 있다.
  floating tolerance와 duplicate 억제 계약도 없다.
- Linear interpolation만 지원하면서 error estimate, interpolation kind와 extrapolation
  상태를 반환하지 않는다.
- =trajectory_init/init_full=은 caller가 초기화하지 않은 구조체의 =samples/capacity=를
  먼저 읽고 =delete[]=할 수 있어 invalid free가 가능하다. 이 lifecycle 결함은 Balix
  container가 고쳐야 하지만 numeric view 전환 전에 sanitizer baseline으로 고정한다.
- C ABI 함수가 =new/new[]=를 직접 사용하고 allocation failure를 status로 바꾸지 않아
  C++ 예외가 ABI 밖으로 전파될 수 있다. =trajectory_create_full=의 object allocation 후
  sample allocation이 실패하면 첫 allocation 회수 경로도 없다. 생성 DLL에서 해제하는
  allocator 계약 역시 없다.
- =trajectory_to_string=은 =snprintf=의 required length가 남은 buffer보다 클 때도
  =offset += n=을 수행한다. 다음 =buffer + offset=과 =size - offset=이 범위를 벗어나거나
  unsigned underflow할 수 있다.
- =trajectory_to_string= header는 null trajectory에서 ="[Trajectory] (null)"=을 반환한다고
  설명하지만 구현은 buffer를 쓰지 않고 그대로 반환한다.
- =impact_time == 0=을 "impact 없음" sentinel로 사용해 실제 =t=0= impact를 표현하지 못한다.
  별도 =has_impact= 상태와 endpoint 포함 정책이 필요하다.

* Industry 기준

- sorted finite abscissa와 duplicate policy
- interpolation/extrapolation 명시
- forward/backward/central 또는 nonuniform finite difference formula 구분
- estimated order/error와 insufficient-neighborhood 상태
- arc-length resampling은 cumulative length를 만들고 bracket+interpolation으로 평가
- caller buffer의 required/written count와 partial result 계약
