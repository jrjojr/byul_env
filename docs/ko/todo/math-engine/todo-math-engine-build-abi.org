* 목적

현재 root =byul= shared 하나와 module static target 구조에서 계산 source 소유권을
먼저 분리하고, 이후 중앙 규칙 변경을 거쳐 하나의 독립 Math Engine DLL로 전환한다.

* 설계 소유권 제한

이 문서는 Math Engine root ABI foundation인 export/version/status/allocator header,
전역 C symbol prefix와 public include namespace 관례, target/install/export graph를
소유한다. Balix Math 추출 master와 세부 extraction TODO가 소유하는 component
=successor_headers[]=, 공식·kernel과 component API를 이 문서에서 다시 설계하지 않는다.

Source/header manifest는 전체 header 리팩터링과 Balix Math 추출 계획의 공동
=asset_id/current_path/disposition/compatibility_path/successor_headers[]=
=canonical_owner/design_todo/install_target/export_target=을 소비한다. Build 편의를 위해
같은 source/header에 새 owner를 만들거나 하나의 Balix header를 하나의 Math Engine
header rename으로 축약하지 않는다.

* 현재 graph

#+begin_src text
numal STATIC
collision STATIC -> numal

balix STATIC
  Balix source + numal source + collision source
  -> core, numal

byul SHARED
  numal source + expanded balix source
#+end_src

이 graph는 Balix가 library를 소비하지 않고 source를 흡수하며 root에서 Numal을 중복
확장한다. =balix_HEADERS=도 Numal과 collision header를 흡수해 install inventory가
중복된다. Collision target은 사용 중인 numeq solver dependency가 없고 그 test도
collision target을 직접 link하지 않는다.

* 목표 graph

과도기:

#+begin_src text
byul_math_engine_static
        ^
        |
balix STATIC

byul SHARED
  동일 Math Engine manifest를 BYUL_EXPORTS로 직접 compile
  + legacy ABI shim
#+end_src

최종:

#+begin_src text
byul_math_engine SHARED  [MATH_ENGINE_EXPORTS]
        ^
        |
balix STATIC
        ^
        |
byul SHARED              [BYUL_EXPORTS, legacy shim only]
#+end_src

* ABI 계약 후보

- 별도 =math_engine_config.h=
- =MATH_ENGINE_API=, =MATH_ENGINE_EXPORTS=, =MATH_ENGINE_STATIC=
- 독립 semantic version, ABI version과 fingerprint
- canonical =byul_math_*= prefix와 =<byul/math_engine/...>= include
- caller storage 또는 생성 DLL에서 destroy하는 opaque handle
- C++ exception, STL, backend allocator와 third-party type 비노출
- desktop shared profile에서 Math Engine static과 shared 동시 link 금지
- embedded-only profile에서 명시적으로 static 선택
- 기존 =vec3_*=, =numeq_*=, =detect_*=, =xform_*=는 ABI 1.x root forwarding shim

Balix public header가 Math Engine value를 노출하면 installed CMake target의 Math Engine
dependency는 =PUBLIC=이다. 구현에서만 쓰면 =PRIVATE=로 축소한다.

이 link visibility는
[[https://cmake.org/cmake/help/latest/command/target_link_libraries.html][CMake
target_link_libraries 공식 계약]]의 =PUBLIC/PRIVATE/INTERFACE= 의미를 따른다. 독립
shared와 installed dependency는
[[https://cmake.org/cmake/help/latest/command/add_library.html][CMake add_library의
SHARED/IMPORTED target 계약]]을 기준으로 Windows import library와 Unix shared object를
구분한다.

* 중앙 규칙 충돌

현재 =external/rules/projects/byul/byul-init-rules.org=와 =build-rules.org=는 공개
shared library를 root =byul= 하나로 고정한다. 사용자 목표가 이 정책을 바꾸므로 최종
DLL cutover 전에 중앙 규칙, runtime/public API/test 규칙과 =rules-ref.org= pin을 같은
승인 변경에서 갱신해야 한다. 그 전 단계는 root DLL을 유지한다.

현재 같은 폴더의 [[file:README.ko.md][legacy Markdown README]]도 최종 목표를 “root 단일
shared library”로 설명한다. Markdown은 규칙이나 계획의 원본이 아니며 대응 Org 원본도
현재 확인되지 않았다. 이 파일을 직접 고치지 않고 document rule에 맞는 canonical Org
source와 출력 mapping을 만든 뒤 DLL cutover 문서와 함께 재생성한다.

반대로 TODO 5~6은 consumer adapter와 중앙 규칙 승인을 앞질러 실행하지 않는다.
