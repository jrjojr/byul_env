* 목적

Entity 7개 header와 source에 섞인 거리, 운동 예측, 충돌, bounce, spring, repulsion,
avoidance와 encirclement 계산을 Math Engine의 physics/geometry/control kernel로 옮긴다.
Entity는 객체 state, owner, update/dispatch와 결과 적용만 담당하는 consumer가 된다.

* 기존 Header 7개와 분리

| Header | Math Engine 계산 | Entity consumer에 남는 책임 |
|--------+------------------+-----------------------------|
| =byul/entity/entity.h= | 계산 없음 여부 확인 | object/resource lifecycle와 owner |
| =byul/entity/entity_dynamic.h= | accel/drag/position/velocity/state prediction, bounce | DTO lifecycle, state update와 mapping |
| =byul/entity/entity_dynamic_coord.h= | distance/range·coordinate conversion primitive | cached/world coordinate commit |
| =byul/entity/entity_interaction.h= | distance/contact/FOV/AOE, collision time/point, bounce response kernel | interaction dispatch와 state mutation |
| =byul/entity/entity_spring.h= | spring/repulsion/push force와 neutral network integration kernel | entity/trajectory adapter와 result 적용 |
| =byul/entity/entity_avoidance.h= | avoidance query/direction | experimental disposition과 entity adapter |
| =byul/entity/entity_encirclement.h= | ring position, occupancy/target/control-point query | experimental disposition과 assignment policy |

* 금지 경계

- Canonical Math Engine header는 =entity_t=, =entity_dynamic_t=, =trajectory_t=와 entity
  callback을 받지 않는다.
- Entity adapter에 공식, iterative solver, hard-coded tolerance와 vector kernel을 복제하지
  않는다.
- Mutable entity/world state, owner pointer, update order와 event dispatch는 Math Engine에
  넣지 않는다.
- 구현 없는 avoidance/encirclement header는 근거 없이 canonical API로 승격하지 않는다.

* Canonical/Consumer Gate

TODO 1~4가 Header 리팩터링 5단계의 canonical work이며 TODO 5는 7단계 Simulation
consumer 결과를 합류한다. Geometry/Physics canonical core와 Balix foundation kernel이
준비되기 전에 TODO 2~4를 시작하지 않는다.
