* 조사 범위와 경계

공개 선언과 구현을 다음 파일에서 직접 대조한다.

- estimator 공개 header: =byul/balix/numeq/filters/numeq_kalman.h=
- estimator 구현: =byul/balix/numeq/filters/numeq_kalman.cpp=
- binding 공개 header: =byul/balix/numeq/filters/numeq_filters.h=
- binding 구현: =byul/balix/numeq/filters/numeq_filters.cpp=
- 현재 test: =byul/balix/tests/test_module_kalman.cpp=

Predict, measurement update, covariance와 projection은 Math Engine =estimation=의 순수
kernel이고 filter state/options는 Math Engine value/API다. Sensor polling, tick schedule,
measurement channel 선택, caller-owned state의 보관 기간과 filter binding lifecycle은 Balix
adapter다. 기존 공개 이름은 독립 DLL 전환 뒤 canonical Math Engine API를 호출하는
legacy forwarding entry로 유지하거나 deprecate한다. 특히 raw =void*=와 function pointer를
묶는 =make_kalman_vec3_interface=는 수학 kernel이 아니라 Balix binding이다.

* BYUL_API 전수 inventory

아래 표는 두 공개 header의 =BYUL_API= 선언 12개를 하나씩 기록한다. =ME-VALUE=는 순수
Math Engine state API, =ME-KERNEL=은 순수 계산 API, =BALIX-BINDING=은 polling/scheduling과
lifetime을 연결하는 adapter, =LEGACY-FORWARD=는 기존 BYUL ABI 이름의 최종 역할이다.

=kalman_filter_t=, =kalman_filter_vec3_t=, =filter_interface_t=와
=filter_time_update_func=, =filter_measurement_update_func=, =filter_get_state_func=도 public
ABI surface다. 기존 struct field를 확장하지 않고 versioned canonical state/result를 새로
만든다. 독립 Math Engine DLL 경계에는 Balix가 소유한 raw function pointer나 수명 불명
=void*=를 그대로 넘기지 않으며, version/size, context retention, calling convention과
thread/reentrancy 계약을 명시한다.

* 현재 진단

- Scalar는 identity process의 1D Kalman update로는 설명 가능하지만 =p/q/r=의 variance
  단위와 non-negative/finite invariant가 없다.
- Vec3는 full constant-velocity state covariance filter가 아니다. 축별 position variance
  하나만 =P+=Q=로 갱신하고 measurement correction 차이를 =dt=로 나눠 velocity로
  추정한다.
- Vec3 이름과 문서가 일반 Kalman capability를 과장할 수 있으며 cross-axis covariance,
  velocity covariance와 model matrix가 없다.
- Joseph-form covariance update가 아니고 roundoff에서 negative/PSD 손실을 진단하지 않는다.
- =dt=, =p+R= zero와 null/non-finite 검사가 불완전하다.
- Filter interface는 =void*=와 cast callback을 public storage에 노출하고 lifetime,
  reentrancy와 measurement shape를 표현하지 않는다.
- =make_kalman_vec3_interface=는 2-인자
  =kalman_vec3_measurement_update(kf, measured_pos)=를 3-인자
  =filter_measurement_update_func(state, measured_pos, measured_vel)=로 cast한다. 서로 다른
  function pointer type을 통해 호출하는 것은 C/C++에서 undefined behavior이며
  =measured_vel=도 실제 모델에 반영되지 않는다. =time_update=의 typed pointer cast도
  trampoline 없이 같은 위험을 가진다.
- =kalman_init_full=은 negative/non-finite =p/q/r=을 그대로 저장하고 scalar correction은
  =p+r=이 zero여도 나눈다. simplified =(1-K)P= 갱신의 error/status surface도 없다.
- =kalman_vec3_init_full=은 =dt<=0=을 조용히 0.1로 바꾸지만 =q/r=은 검증하지 않는다.
  이후 =kalman_vec3_measurement_update=는 =dt=를 public struct에서 직접 바꿀 수 있으므로
  zero division을 다시 허용한다.
- =kalman_vec3_project=는 negative/non-finite =future_dt=를 그대로 사용한다. 이는 순수
  projection 식 자체는 Math Engine 소유라는 사실과 별개로 input-domain 계약이 빠진
  public API defect다.
- 12개 =BYUL_API= 선언은 모두 definition이 존재하지만 현재 test는
  =numeq_kalman.h=의 수치 함수만 호출하고 =make_kalman_vec3_interface=와 callback ABI를
  한 번도 실행하지 않는다.
