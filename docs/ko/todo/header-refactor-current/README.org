#+end_src

Standalone matrix는 C/C++ compiler를 함께 지정한다. MSVC는 =VsDevCmd.bat -arch=x64=
환경에서 =cl.exe=를 두 인수에 같이 전달하고, MinGW/Linux/macOS는 해당 target의
=cc/c++= 또는 =clang/clang++=을 전달한다. Export snapshot은 MSVC =dumpbin=뿐 아니라
GNU/LLVM =nm -P --defined-only --extern-only=를 지원하므로 Windows, Linux와 macOS가
같은 symbol subset gate를 사용한다.

현재 metadata 부채 2,773개와 structural 부채 283개는 해결 완료가 아니다. Exact
fingerprint와 일치하는 기존 항목만 기본 gate에서 grandfather하며 새 오류는 실패한다.
승인 export exclusion 16개도 supported symbol이 아니며 4단계 Navsys, 5단계 Math Engine,
7단계 Simulation owner가 해결하면 exclusion을 같은 변경에서 제거해야 한다. 이미
해결된 exclusion이 남아도 stale exclusion으로 실패한다.

Role manifest는 기존 asset 135개, 새 Math Engine successor 20개와 public resource의
ABI 1.x additive API/ABI 2.0 opaque migration 10개를 서로 다른 record로 승인한다.
생성 successor는 기존 asset count와 중복 계산하지 않고 모든 source asset에 역링크한다.
ABI 2.0 layout 전환은 이 설계 승인만으로 실행하지 않으며 별도 ABI-major release gate가
필요하다.

Fresh MSVC build/install, SDK consumer, build snapshot과 ABI probe는 상위 TODO의 진행
기록에 사용한 =build_header_refactor_current= 격리 build tree에서 실행했다. Build tree는
추적 artifact가 아니며 이 directory의 JSON과 명령·profile 기록만 재현 근거로 보존한다.

MSYS2 MinGW-w64 검증은 =build_header_refactor_mingw= 격리 build tree에서
=MinGW Makefiles= generator로 실행했다. Release 전체 build와 CTest 19/19가 통과했고,
clean install은 header 92개, test executable 19개와 MinGW runtime DLL 3개를 포함한
파일 116개다. =libbyul.dll= SHA-256는
=2fe4b1abd5fb48e660fdb3284a1c32269197518ab086660ff31f206de82534e6=이다. 설치
