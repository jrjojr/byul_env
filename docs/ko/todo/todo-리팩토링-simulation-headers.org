* 목적과 범위

=balix= 18개, =entity= 7개, =projectile= 6개와 =ground= 1개, 합계 32개
header를 리팩터링한다. 공개 simulation value/resource layout과 callback, ownership,
lifecycle 및 numeric unit/error 계약을 implementation과 일치시키고 module static과 root
shared export를 모두 검증한다.

이 문서는 Balix header layout/export/install, lifecycle와 legacy forwarding gate를
소유한다. 공식·상수·계산 kernel과 canonical Math Engine API는
[[file:math-engine/todo-balix-math-extraction.org][Balix 수학 추출 master]]와 아래 header별
세부 TODO가 소유한다.

* Header 명명 Gate

32개 최초 header asset 모두 현행 경로와 명명 처분을 가진다. 단순 rename은 canonical
basename/path를 가지지만 Math Engine으로 책임이 분리되는 header는 기존 Balix
compatibility/adapter 경로와 =successor_headers[]=를 별도로 가진다. 최초 Balix header와
새 Math Engine header를 일대일 rename으로 취급하지 않는다.

대표 public type header는 type stem, 대표 type이 없는 public-component는
=<public_module>_<responsibility>.h=를 사용한다. Project명, 상위 분류 directory와
implementation directory chain을 basename에 기계적으로 연결하지 않고 SDK public
basename의 전역 고유성을 검사한다.

Math Engine successor의 canonical 이름은 Math Engine 소유 TODO에서 결정한다. 이 문서는
그 이름을 중복 설계하지 않고 기존 Balix header의 =keep/rename/forward/split/internalize=
처분, compatibility path와 successor link를 기록한다. 기존 공개 include는
compatibility-forwarder와 제거 version 없이 삭제하지 않는다. 모든 32개와 새 successor가
terminal 상태로 닫히기 전에는 이 하위 TODO를 완료하지 않는다.

* Balix/Math Engine 공동 소유권 Gate

| 대상 | 단일 design owner | 이 문서의 책임 |
|------+-------------------+----------------|
| Math 공식·kernel·neutral value/result | 연결된 Math Engine 세부 TODO | owner와 successor link 검증 |
| 새 canonical Math Engine header/API | 연결된 Math Engine 세부 TODO | 이름·API를 재설계하지 않음 |
| 기존 Balix public header/layout | 이 문서 | ABI/layout/install snapshot과 compatibility |
| Balix DTO/lifecycle/adapter | 이 문서 또는 명시된 adapter TODO 하나 | callback, state와 forwarding |
| legacy Balix include/symbol | 이 문서 | deprecation, forwarding과 제거 version |

공동 manifest는 =asset_id/current_path/current_owner/disposition/compatibility_path=와
=successor_headers[]=를 기록한다. 각 successor에는 =canonical_owner/design_todo=
=install_target/export_target=을 기록한다. 하나의 symbol, source 또는 header에 design
owner가 둘 이상이면 다음 단계로 진행하지 않는다.

실행 순서는 공동 inventory 동결 → symbol/header별 owner 승인 → Math Engine successor
설계 승인 → Balix compatibility/adapter mapping 승인 → 구현 → install/export/wrapper
합류다. 두 오케스트레이션 문서가 독립적으로 같은 단계를 =DOING=으로 시작하지 않는다.

* 감사 기준선

- 32개 모두 표준 license 첫 block, =BYUL_= main guard, conforming file Doxygen와
  =@byul.*= metadata를 충족하지 않는다.
- public size/align/offset ABI query가 없다.
- 16개 header가 project/system include를 =extern "C"= 안에 둔다.
- =BYUL_API=를 쓰면서 =byul_config.h=을 직접 include하지 않는 header가 최소 11개다.
- CMake/install inventory는 30/32이며 =entity_avoidance.h=와
  =entity_encirclement.h=는 구현, test, CMake, =BYUL_API=와 내부 reference가 없는 orphan이다.
- wrapper generator의 module header inventory에는 설치된 대상 30개가 하나도 등록되지 않았다.
- balix CMake가 numal source/header를 자기 inventory에 넣고 root도 numal을 별도 집계해
  ownership과 compile inventory가 중복될 수 있다. Source 이동과 target 방향은
  [[file:math-engine/todo-balix-math-extraction.org][추출 master]]와
  [[file:math-engine/todo-math-engine-build-abi.org][Build·DLL ABI TODO]]에서 닫는다.

* P0 ABI, ownership와 구현 결함

- =numeq_mpc.h=의 =mpc_direction_target_init_full=과
  =mpc_direction_target_assign=만 인접 API와 달리 =BYUL_API=가 없어 Windows DLL export
  hole이 생긴다.
- =projectile_result_free= 구현은 내부 trajectory뿐 아니라 result 자체도 delete하여
  =destroy=와 동일하다. stack/caller storage에 =free=를 호출하면 double delete나 invalid
  free 위험이 있다.
- =projectile_tick=의 assign은 trajectory를 shallow copy하고 일부 state를 누락한다.
  null source에서 기존 resource를 정리하지 않으며 free도 ground/trajectory lifecycle을
  일관되게 사용하지 않는다. 현재 test는 trajectory 없는 경로만 검사한다.
- =ground_assign=은 tables/height를 deep-copy하지만 =ground_free=는 height만 해제하여
  table leak이 발생한다. =void= assign은 partial allocation failure를 보고할 수 없다.
- =trajectory_init=은 기존 =samples=를 읽고 delete하므로 fresh uninitialized caller storage에
  안전하지 않다. zero-init 선행 요구와 일반 init 이름이 충돌한다.
- controller와 filter interface는 =void*=, function pointer, vtable과 inline layout access를
  public ABI에 고정한다.
- environ callback 문서는 =delta_time= parameter가 있다고 설명하지만 실제 typedef에는
  없고 retention/thread/reentrancy/lifetime 계약도 없다.
- projectile core/projectile/predict/tick과 ground는 owning/retained pointer 및 callback을
  public struct에 노출한다.
- =detect_*=, =calc_spin_accel=과 =calc_suitable_*= 등 대표 type prefix가 없는 legacy
  symbol은 직접 rename하지 않고 canonical API와 deprecated forwarding을 추가한다.
