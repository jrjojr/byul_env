# BYUL Header 기반 Python Wrapper 완전 자동 생성 TODO

## 최종 목표

BYUL Python wrapper의 최종 목표는 기존 generated Python 파일이 하나도 없는
상태에서 public C header를 읽어 실행 가능한 wrapper package 전체를 생성하는 것이다.

완료 시 다음 절차가 성공해야 한다.

```text
입력:
  BYUL SDK 또는 source tree의 public C header
  header에 기록된 Doxygen와 @byul.* metadata
  platform의 byul shared library

초기 상태:
  generated byul_wrapper/*.py 파일 없음

생성:
  generate_wrapper_abi.py 실행

출력:
  import 가능한 byul_wrapper package
  ffi.cdef 선언
  c_* class와 module function
  enum과 callback adapter
  소유권과 오류 처리 코드
  symbol coverage report
  generated test 또는 test manifest
```

기존 wrapper Python 구현을 다른 디렉터리의 template, 문자열, 압축 데이터 또는
복사본으로 옮긴 뒤 그대로 복원하는 방식은 완전 자동 생성으로 인정하지 않는다.
Python API에 필요한 판단은 C 선언, 표준화된 이름, 구조화된 `@byul.*` metadata와
명시적인 소규모 generator 설정만으로 결정되어야 한다.

  - 출력 module이 없으면 새 파일을 만들지 못한다.
