#+TITLE: BYUL 문서 출판 매핑
#+LANGUAGE: ko
#+OPTIONS: toc:2 num:nil
#+DOCUMENT_KIND: configuration
#+AUDIENCE: maintainer
#+ACCESS: protected
#+SECTION: document-tooling
#+SLUG: byul-document-map
#+SOURCE_FORMAT_VERSION: 1
#+DOCUMENT_MAP_VERSION: 1

* 목적

이 Org 문서는 BYUL의 출판 대상 Org 원본과 Markdown 산출물 사이의 완전한 1:1 매핑을
소유한다. 행을 추가하거나 바꿀 때는 원본과 산출물의 경로, 문서 metadata와 slug를 함께
검증한다. Markdown을 별도 원본으로 등록하지 않는다.

* 행 계약

- =key=와 =slug=는 출판 집합 전체에서 유일한 영문 소문자 식별자다.
- =source=와 =output=은 저장소 root 기준 POSIX 상대경로다.
- source는 =docs/org/= 아래의 =.org= 파일이고 output은 =docs/= 아래의 =.md= 파일이다.
- =kind=, =language=, =access=는 source preamble의 =DOCUMENT_KIND=, =LANGUAGE=,
  =ACCESS=와 일치해야 한다.
- source preamble에는 =TITLE=, =AUDIENCE=, =SECTION=, =SOURCE_FORMAT_VERSION=과
  =GENERATED_TARGETS=도 있어야 하며, =GENERATED_TARGETS=는 해당 행의 =output= 경로와
  정확히 같아야 한다.
- =docs/org/**/*.org= 원본은 빠짐없이 이 표에 있어야 한다.
- 이 publisher marker를 가진 =docs/**/*.md= 산출물도 빠짐없이 이 표에 있어야 한다.
- Org source block은 BEGIN/END가 균형을 이루어야 한다. source block 밖의 상대
  =file:= link는 저장소 안의 실제 대상을 가리켜야 한다.

* 출판 매핑

아래 이름과 열 순서는 parser 계약이다. 표 이름이나 열을 임의로 바꾸지 않는다.

#+NAME: byul-document-map-v1
| key                     | source                                        | output                                | slug                     | kind          | language | access |
|-------------------------+-----------------------------------------------+---------------------------------------+--------------------------+---------------+----------+--------|
| build-environment-ko    | docs/org/ko/build-environment.ko.org          | docs/ko/build-environment.ko.md       | build-environment-ko     | configuration | ko       | public |
| build-environment-en    | docs/org/en/build-environment.org             | docs/en/build-environment.md          | build-environment-en     | configuration | en       | public |
| runtime-execution-ko    | docs/org/ko/runtime-execution.ko.org           | docs/ko/runtime-execution.ko.md       | runtime-execution-ko     | configuration | ko       | public |
| runtime-execution-en    | docs/org/en/runtime-execution.org              | docs/en/runtime-execution.md          | runtime-execution-en     | configuration | en       | public |

* 생성 산출물 표시

Python publisher는 각 Markdown 앞에 다음 정보를 HTML comment로 기록한다.

#+begin_src text
GENERATED FILE - DO NOT EDIT
Source: 저장소 상대경로
Source-SHA256: 줄바꿈을 LF로 정규화한 Org 원본의 SHA-256
Content-SHA256: marker 뒤 Markdown 본문의 SHA-256
Generator: tools/docs/publish_docs.py
#+end_src

source와 본문 hash를 함께 검사하므로 Emacs가 없는 환경에서도 =--check=는 원본 변경과
산출물 직접 수정을 검출할 수 있다. Emacs가 있으면 staging export도 수행해 현재
publisher가 만드는 bytes와 공식 산출물이 정확히 같은지 추가로 검사한다.

* 실행

#+begin_src text
tools/docs/publish_docs.bat
tools/docs/publish_docs.bat --check
tools/docs/publish_docs.bat --apply

tools/docs/publish_docs.sh
tools/docs/publish_docs.sh --check
tools/docs/publish_docs.sh --apply
#+end_src

기본 실행은 write 없는 dry-run이다. =--check=는 현재 mapping, source metadata, marker와
hash를 검사한다. =--apply=만 저장소 소유 Emacs exporter로 staging 산출물을 만든 뒤 공식
Markdown을 교체한다. marker가 없거나 본문 hash가 바뀐 기존 파일은 검토 후 =--force=를
함께 명시해야 교체할 수 있다.
